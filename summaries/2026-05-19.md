### Vite Version 8: Unified Rust-Based Bundler and Up to 30x Faster Builds - (InfoQ)

https://www.infoq.com/news/2026/05/vite-v8-rust/

# Vite Version 8: Unified Rust-Based Bundler and Up to 30x Faster Builds

Vite 8.0 represents a major architectural overhaul, replacing its dual-bundler system (esbuild for development, Rollup for production) with Rolldown, a single unified Rust-based bundler. This consolidation eliminates inconsistencies from maintaining two separate transformation pipelines and delivers dramatic performance improvements—10 to 30x faster builds with real-world examples showing production build times cut from 46 seconds to 6 seconds (Linear), a 57% reduction (Ramp), and up to 8x improvements for some developers.

Beyond performance, Vite 8 introduces developer experience improvements including built-in tsconfig paths support, native decorator metadata support for frameworks like NestJS, and browser console forwarding to terminals. While a compatibility issue exists with Yarn's Plug'n'Play on Windows, the release maintains full plugin API compatibility with existing Vite plugins. As a framework-agnostic alternative downloaded over 65 million times weekly, Vite 8 positions itself competitively against Turbopack and Rspack in the rapidly Rust-ifying JavaScript toolchain ecosystem.

### Swiggy Improves Search Autocomplete Using Real Time Machine Learning Ranking - (InfoQ)

https://www.infoq.com/news/2026/05/swiggy-autocomplete-rt-ranking/

# Summary

Swiggy has implemented a real-time machine learning ranking system to improve search autocomplete suggestions, replacing traditional hand-tuned heuristic approaches. The system operates in two stages: candidate generation using OpenSearch lexical retrieval and embedding-based search, followed by ML-based ranking that reorders results using real-time signals like user interaction history, click behavior, and item popularity. The architecture integrates learning-to-rank models directly into OpenSearch to avoid additional network latency, which is critical since each keystroke can trigger a new query.

The system maintains high performance through strict latency optimization while incorporating a continuous feedback loop that retrains ranking models using live user interaction data. By separating candidate generation and ranking into independent stages, and leveraging feature stores for efficient feature serving, Swiggy achieves both responsiveness at scale and the ability to adapt to evolving user behavior without manual rule updates.

### Article: Building a Secure MCP Server on AWS for a Million-Company B2B Platform - (InfoQ)

https://www.infoq.com/articles/secure-mcp-server-aws/

# Summary

This article describes how to build a production-ready MCP (Model Context Protocol) server on AWS that safely connects large language models to a B2B intelligence platform containing over one million company profiles. Rather than treating the MCP server as a simple demo wrapper, the team designed it as a first-class production interface with its own security contracts, testing strategies, and operational controls. The architecture uses a Go-based MCP server that translates user requests into narrowly scoped tools, which then map to GraphQL operations on AWS AppSync, ensuring the backend remains the system of record and preventing prompt-level ambiguity from leaking into production data.

Key lessons include separating read and write operations at the tool level, implementing a default-deny approach for mutations, validating against actual backend systems rather than relying solely on unit tests, and designing explicit tool contracts to make the system testable and observable. This approach reduces risk when connecting LLMs to production data by maintaining clear boundaries between the LLM layer, the MCP interface, and the underlying business systems.

### Podcast: Context is the Key to the Agentic Architecture Revolution: A Conversation with Baruch Sadogursky - (InfoQ)

https://www.infoq.com/podcasts/context-key-agentic-architecture-revolution/

# Summary

In this podcast conversation, Baruch Sadogursky discusses how agentic AI is revolutionizing software architecture by making specifications the source of truth rather than code. Large Language Models can function as reasoning machines capable of interpreting human ambiguity, but they require "context engineering"—a rigorous discipline using artifacts like skills, rules, scripts, and feedback—to constrain their stochastic nature and guide code generation. Unlike prompt engineering, context engineering provides clear intent and enables AI agents to ask clarifying questions until requirements are fully understood, allowing quality validation before code is written.

Humans remain essential in this new paradigm, responsible for ensuring requirements are correct, providing proper context, and validating results. Given current LLM context window limitations, agentic AI development requires a microservices architecture where each service can be regenerated from updated specifications, with human architects orchestrating these services. Written documentation becomes the definitive source of truth while code becomes a disposable intermediate language, fundamentally shifting how software is developed and maintained.

### Cloudflare and Stripe Let AI Agents Create Accounts, Buy Domains, and Deploy to Production - (InfoQ)

https://www.infoq.com/news/2026/05/cloudflare-stripe-agent-commerce/

# Summary

Cloudflare and Stripe have launched a new protocol enabling AI agents to autonomously handle cloud infrastructure tasks—including creating accounts, purchasing domains, setting up subscriptions, and deploying applications to production—without human intervention beyond legal and financial approval points. The system uses three key components: Discovery (querying available services via REST API), Authorization (Stripe as identity provider with automatic account creation), and Payment (tokenized billing with a default $100/month spending cap). The protocol is designed to be open and extensible, similar to how OAuth enabled delegated access.

However, the capability raises significant risk concerns. Documented failure modes include agents purchasing wrong domains, exhausting budgets through API retry loops, and making irreversible spending decisions based on ambiguous specifications. Security experts emphasize the need for strict runtime budget enforcement, audit logs, idempotency keys, and fast kill switches. While the protocol represents an ambitious step toward agentic automation, practical safeguards remain critical to prevent costly autonomous errors.

### Navigation API Reaches Baseline Newly Available as Replacement to the History API - (InfoQ)

https://www.infoq.com/news/2026/05/navigation-api-browser/

# Summary

The Navigation API has reached Baseline Newly Available status as of January 2026, offering a modern replacement for the long-outdated History API with support across Chrome, Edge, Firefox 147, and Safari 26.2. The new API provides a centralized interface specifically designed for managing client-side navigation in single-page applications (SPAs), addressing decades of developer frustration with the History API's fragmented approach, inconsistent event behavior, and inability to detect all navigation types or inspect the full history stack.

The Navigation API introduces significant improvements including a unified `navigate` event that fires for all navigation triggers (link clicks, form submissions, back/forward buttons, or programmatic calls), the `event.intercept()` method for automatic URL and history management, built-in scroll restoration, and new history entry inspection capabilities via `navigation.entries()` and `traverseTo(key)`. While popular SPA frameworks like React Router and TanStack Router are exploring adoption, the API primarily serves as a lower-level platform primitive that higher-level routing abstractions can build upon, with developers already describing it as "a game-changer" for SPA development.

### OpenAI Open-Sources Symphony, a SPEC.md for Autonomous Coding Agent Orchestration - (InfoQ)

https://www.infoq.com/news/2026/05/openai-symphony-agents/

# Summary

OpenAI has open-sourced Symphony, a specification for orchestrating multiple autonomous coding agents using project management tools like issue trackers. Rather than requiring developers to manage individual coding sessions (limited to 3-5 simultaneous sessions before context switching becomes painful), Symphony assigns each task to a dedicated agent that works autonomously to completion, with humans reviewing the final output. This significantly reduces the cost of agent mistakes since errors are caught at the completion stage rather than requiring continuous human supervision.

Symphony structures work around project deliverables (issues, tasks, tickets, milestones) rather than individual PRs, allowing agents to generate implementation plans, break work into subtasks, and even create new issues autonomously—all subject to human review before execution. Notably, Symphony is released as a SPEC.md document (a reference specification) rather than a standalone product, enabling organizations to build their own customized orchestrators. The reference implementation uses Elixir for its strong concurrency primitives.

### Agent Skills Work but the Research Shows Most Teams Are Building Them Wrong - (Radar)

https://www.oreilly.com/radar/agent-skills-work-but-the-research-shows-most-teams-are-building-them-wrong/

# Summary

While agent skills have become widely adopted across major platforms like Atlassian, Canva, and Figma, recent research reveals that most teams are implementing them incorrectly. Four empirical studies show that curated, human-written skills improve task completion rates by 16.2% on average, whereas AI-generated skills provide no consistent benefits. Additionally, as skill libraries grow, flat organizational structures cause similar skills to collide and become indiscoverable—a problem solved by organizing skills hierarchically instead.

Beyond performance issues, a security analysis of 31,000 community skills found that over 25% contain exploitable vulnerabilities including prompt injection, data exfiltration, and privilege escalation attacks. The research emphasizes that skills—modular folders containing descriptions, instructions, and optional scripts—are most effective when carefully curated and strategically organized, rather than auto-generated or stored in unstructured libraries.

### Agent Harness Engineering - (Radar)

https://www.oreilly.com/radar/agent-harness-engineering/

# Summary

This article introduces "agent harness engineering," a discipline focused on the scaffolding and infrastructure built around AI models rather than the models themselves. The author argues that a coding agent consists of the model plus everything constructed around it—including prompts, tools, context policies, feedback loops, and recovery mechanisms. The key insight is that a decent model with excellent harness engineering outperforms a superior model with poor harness design, making the real engineering challenge lie in designing effective scaffolding rather than selecting the best model.

The article emphasizes that most agent failures are configuration and design issues ("skill issues") rather than model limitations. It breaks down what comprises a harness: system prompts, tools and integrations, infrastructure, orchestration logic, hooks for deterministic execution, and observability features. By treating agent failures as opportunities to engineer solutions preventing those mistakes in the future, developers can continuously tighten and improve their harness design—shifting focus from the loud model-selection debate to where actual leverage and differentiation exist.
