## https://pydantic.dev/articles/inline-snapshot

# Summary

The article introduces **inline-snapshot**, a Python testing library that significantly improves how developers write and maintain tests for complex data structures. Traditional snapshot testing requires manually writing expected values or storing them in separate files, which is tedious and creates maintenance headaches. Inline-snapshot elegantly solves this by allowing developers to write empty `snapshot()` placeholders and automatically populating them with actual values using `pytest --inline-snapshot=fix`, making it easy to update multiple tests at once when data structures change.

The article demonstrates practical techniques for handling real-world testing challenges, such as combining inline-snapshot with **dirty-equals** (another Pydantic library) to handle dynamic values like timestamps or random IDs using matchers like `IsInt()` or `IsNow()`. It also showcases advanced patterns like converting data to built-in types before snapshotting for robustness, and nesting snapshots within conditional assertions for complex nested structures. The author illustrates these concepts with real examples from the OpenAI Agents SDK, where this approach enabled comprehensive test coverage with minimal effort while improving both test quality and developer velocity.
