### Inside SPy part 2: Language semantics - (Python Weekly)

https://antocuni.eu/2026/03/25/inside-spy-part-2-language-semantics/

# Summary

This blog post is the second installment in Antonio Cuni's "Inside SPy" series, focusing on the language semantics of SPy, a statically-typed Python variant designed for performance comparable to C, Rust, and Go. SPy operates through three execution phases: import time (module-level code execution), redshift (optional partial evaluation), and runtime (program execution). The compiler pipeline uses AST (Abstract Syntax Tree) as its internal representation rather than bytecode, which simplifies implementation, particularly for redshifting, while still targeting performance. SPy supports both an interpreter for development and a C-code compiler backend for speed, with the key design principle that both have identical semantics.

The post demonstrates SPy's core concepts through examples like "Hello World," showing how code can be executed in multiple modes: interpreted (untyped AST), doppler (redshifted but interpreted), and compiled (to native executables or WebAssembly). A significant departure from standard Python is that all module imports are resolved eagerly at compile-time rather than dynamically, which is essential to SPy's design philosophy of balancing performance with maintainable Pythonic syntax while explicitly rejecting 100% Python compatibility.

### Django Custom Managers Are Silently Leaking Data - (Python Weekly)

https://johnnymetz.com/posts/django-custom-managers-data-leak/

# Summary

Django custom managers, commonly used to exclude inactive or soft-deleted records by default, only apply to queries rooted in the manager's model—not when that model is accessed through joins or relationships. This creates silent data leaks where inactive records unexpectedly appear in results from ManyToManyField access, cross-relation filtering, and aggregations/annotations. The article demonstrates these vulnerability patterns and provides fixes using explicit queryset methods that duplicate the manager's filtering logic.

To catch these leaks before production, the author provides a runtime check that inspects SQL queries and raises errors when the filtered table is accessed without the required active filter. This check can be enabled in test suites to surface unsafe queries. For cases where inactive data is legitimately needed (analytics, admin tools), the author recommends explicit access through an escape-hatch manager, ensuring that accessing sensitive data is always intentional rather than accidental.

### Fixed Python autocomplete - (Python Weekly)

https://matan-h.com/better-python-autocomplete

# Summary

The author identified a flaw in Python autocomplete functionality across popular editors like VSCode—autocomplete results were sorted alphabetically rather than by actual usage frequency, surfacing obscure functions while hiding commonly-used ones. The author argues that alphabetical sorting, while visually clean, is poor UX for programming where developers type predictably and expect relevant suggestions based on frequency.

To fix this, the author analyzed 5.2K Python repositories using the ManyTypes4PyDataset to count function and attribute usage frequencies, then created an optimized binary format (Hash Score Table) that stores this data efficiently using 24-bit FNV-1a hashes and compressed frequencies. By applying thresholds for minimum usage across projects and raw call counts, the author reduced file bloat and significantly improved autocomplete relevance. The solution has been integrated into the Ty LSP and demonstrates how data-driven, frequency-based sorting produces more useful autocomplete than arbitrary alphabetical ordering.

### Reducing Pydantic's memory footprint using bitsets - (Python Weekly)

https://pydantic.dev/articles/pydantic-bitset-performance

# Summary

Pydantic reduced memory usage by 55% by replacing Python sets with bitsets for tracking which model fields were explicitly set during validation. The investigation revealed that the `model_fields_set` attribute—which stores field names as a Python set—was consuming roughly 800 MB of memory across 1 million model instances. Since this attribute only needs to track on/off states for ordered fields, the team implemented a bitset-based solution using the `FixedBitSet` crate, which efficiently represents multiple boolean flags in a compact binary format. For models with extra fields, these are stored separately in a regular set.

The implementation improves both memory efficiency and validation speed, particularly for models with many fields, though it comes with challenges around backward compatibility. Since `model_fields_set` is exposed as a public API typed as `set[str]`, users may expect it to be an actual Python set with all associated behaviors (pickling, isinstance checks, etc.). These compatibility concerns may require waiting for Pydantic V3 to fully implement the change, highlighting the open-source maintenance challenge of not breaking existing code that relies on undocumented API behavior.

https://techcrunch.com/2026/03/26/netflix-confirms-its-raising-prices-again/

# Netflix Raises Prices Again

Netflix has announced another price increase across all subscription tiers, effective March 26, 2026. The ad-supported plan rises from $7.99 to $8.99 per month, the standard ad-free plan increases by $2 to $19.99, and the premium plan also goes up $2 to $26.99. Additionally, the cost of adding extra household viewers has increased on both ad-supported and ad-free plans.

The company justified the hikes by citing improvements to its entertainment offerings and service quality, including recent additions like video podcasts, livestreaming content, and mobile app updates. New members will see the updated pricing immediately, while existing subscribers will be notified via email one month before the changes apply to their accounts. This marks Netflix's second price increase in roughly 14 months, following a similar adjustment in January 2025.

### 16 of the most interesting startups from YC W26 Demo Day - (TechCrunch)

https://techcrunch.com/2026/03/26/16-of-the-most-interesting-startups-from-yc-w26-demo-day/

# Summary

Y Combinator's Winter 2026 cohort featured nearly 190 startups at Demo Day, with AI remaining a dominant theme across industries including law, transportation, and healthcare. TechCrunch highlighted 16 particularly interesting startups, ranging from ARC Prize Foundation (which creates benchmarks for measuring AGI progress) to companies like Asimov (training humanoids with human movement data), Button Computer (wearable AI devices), and CodeWisp (AI-powered game creation).

The standout startups address various market gaps and emerging needs: Avoice automates tedious work for architects, Crosslayer Labs detects website spoofs amid rising deepfake threats, Doomersion teaches languages through social media-style scrolling, and several companies tackle underserved sectors like library management (Librar Labs) and language barriers in healthcare (Opalite Health). Other notable entries include defense tech solutions like Milliray's drone-tracking radar system and tools for fraud detection and cross-market trading, reflecting how AI is being applied to enhance both security and operational efficiency across diverse industries.

### Google Translate’s real-time headphone translations feature expands to iOS and more countries - (TechCrunch)

https://techcrunch.com/2026/03/26/google-translates-real-time-headphone-translations-feature-expands-to-ios-and-more-countries/

# Summary

Google has expanded its "Live Translate" feature to iOS and 12 additional countries, making real-time translation in headphones available across both platforms in the U.S., India, Mexico, Germany, Spain, France, Nigeria, Italy, the United Kingdom, Japan, Bangladesh, and Thailand. The AI-powered feature, which leverages Google's Gemini AI, preserves speakers' tone and emphasis while translating over 70 languages in real-time, allowing users to understand conversations and announcements in different languages through any standard headphones.

The announcement comes alongside Google's expansion of its conversational search feature, "Search Live," to over 200 countries and territories globally. This broader rollout demonstrates Google's commitment to making its AI translation and search capabilities more accessible worldwide.

### OpenAI abandons yet another side quest: ChatGPT’s erotic mode - (TechCrunch)

https://techcrunch.com/2026/03/26/openai-abandons-yet-another-side-quest-chatgpts-erotic-mode/

# Summary

OpenAI has indefinitely paused development of an "erotic mode" for ChatGPT that CEO Sam Altman proposed in October, following significant backlash from tech watchdog groups and internal staff. The feature had been delayed multiple times, with advisers warning the company could be creating a "sexy suicide coach," and the company has now provided no timeline for its release.

This decision is part of a broader strategic shift by OpenAI to abandon side projects and refocus on core business priorities: business users and coders. In the past week alone, the company has also deprioritized its Instant Checkout e-commerce feature and shut down Sora, its AI video generator. The consolidation appears driven by competitive pressure from Anthropic and OpenAI's success in securing a $200 million Pentagon contract, signaling that the future of AI development is shifting toward business and defense applications rather than consumer entertainment features.
