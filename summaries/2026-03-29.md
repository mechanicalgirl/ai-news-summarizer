### We Rewrote JSONata with AI in a Day, Saved $500K/Year - (Simon Willison's Weblog)

https://simonwillison.net/2026/Mar/27/vine-porting-jsonata/#atom-everything

# Summary

A team called Reco used AI to rewrite JSONata (a JSON expression language similar to jq) from its original implementation into Go in just one day, spending only $400 in AI tokens. The project succeeded largely because JSONata had an existing comprehensive test suite, which allowed AI to generate a working implementation in just 7 hours. The team then validated the new version through a week-long shadow deployment, running both versions in parallel to ensure identical behavior before fully switching over, reportedly saving $500K annually.

This case study exemplifies "vibe-porting"—using AI to port code between languages based on understanding the general approach rather than line-by-line translation. The existing tests were crucial for both rapid development and confidence in correctness, demonstrating how good test coverage can enable faster AI-assisted rewrites.

### Quantization from the ground up - (Simon Willison's Weblog)

https://simonwillison.net/2026/Mar/26/quantization-from-the-ground-up/#atom-everything

# Summary

This post highlights Sam Rose's interactive essay on LLM quantization—the process of reducing model size by converting high-precision numbers to lower precision. The essay includes excellent visual explanations of floating-point binary representation and covers an important but poorly understood phenomenon: "outlier" values that are rare but critical to model quality. Removing even a single such "super weight" can cause models to produce gibberish, so production quantization schemes often preserve these outliers through special handling rather than compressing them normally.

The essay also measures real-world accuracy impacts using perplexity and KL divergence metrics on the Qwen 3.5 9B model. Rose's key finding: reducing precision from 16-bit to 8-bit causes almost no quality loss, while 16-bit to 4-bit quantization results in roughly 90% of original performance—a significant but acceptable trade-off for the dramatic reduction in model size.

### Thoughts on slowing the fuck down - (Simon Willison's Weblog)

https://simonwillison.net/2026/Mar/25/thoughts-on-slowing-the-fuck-down/#atom-everything

# Summary

Mario Zechner, creator of the Pi agent framework, warns against the reckless speed enabled by AI coding agents. He argues that while humans naturally act as bottlenecks that limit code production and mistakes, orchestrated armies of agents remove this constraint, allowing small errors to compound rapidly into unsustainable complexity. When developers delegate all oversight to agents, they lose understanding of their own codebase and only discover problems after "cognitive debt" has accumulated beyond recovery.

Zechner recommends deliberately slowing down by limiting daily agent-generated code output to match human review capacity, and manually writing critical architectural decisions rather than letting agents generate them. Simon Willison agrees that developers need new discipline to balance the newfound speed of AI-assisted coding with the mental thoroughness required to maintain system coherence—since code generation is no longer the bottleneck in software development.

### Package Managers Need to Cool Down - (Simon Willison's Weblog)

https://simonwillison.net/2026/Mar/24/package-managers-need-to-cool-down/#atom-everything

# Summary

A recent supply chain attack on LiteLLM has renewed interest in "dependency cooldowns"—a security practice where package managers delay installing updated dependencies for several days to allow the community time to detect compromised packages. An article by Andrew Nesbitt reveals that this mechanism has gained significant traction across major package managers, with recent implementations in pnpm, Yarn, Bun, Deno, uv, pip, and npm between September 2025 and February 2026.

Each tool offers slightly different approaches to cooldowns, typically allowing developers to set a minimum release age before packages can be installed, with options to exempt trusted packages from the delay. While pip currently only supports absolute timestamps rather than relative durations, a workaround exists using scheduled cron jobs. The widespread adoption across diverse package ecosystems suggests growing recognition of dependency cooldowns as an important security best practice.

### Malicious litellm_init.pth in litellm 1.82.8 — credential stealer - (Simon Willison's Weblog)

https://simonwillison.net/2026/Mar/24/malicious-litellm/#atom-everything

# Summary

LiteLLM version 1.82.8 published to PyPI contained a malicious credential stealer hidden in base64 within a `litellm_init.pth` file. The exploit was triggered simply by installing the package, without even needing to import it, and would steal a vast array of sensitive credentials including SSH keys, AWS credentials, Kubernetes configs, Docker credentials, cryptocurrency wallets, and shell history files.

PyPI quarantined the package within hours of discovery, limiting exposure. The attack originated from a prior exploit against Trivy, a security scanner tool used in LiteLLM's CI pipeline, which resulted in stolen PyPI credentials that attackers used to directly publish the compromised packages to the repository.

### Streaming experts - (Simon Willison's Weblog)

https://simonwillison.net/2026/Mar/24/streaming-experts/#atom-everything

# Summary

Simon Willison reports on rapid advances in "streaming experts," a technique that enables running extremely large language models on consumer hardware by streaming expert weights from SSD storage rather than loading entire models into RAM. Recent achievements include running a 397-billion parameter model on 48GB RAM, a 1-trillion parameter Kimi K2.5 model on 96GB of RAM on a MacBook Pro, and even getting the same model functional on an iPhone (albeit slowly at 0.6 tokens/second).

These breakthroughs demonstrate the practical viability of the streaming experts approach, with researchers like Dan Woods continuing to optimize the technique through iterative experimentation. The ongoing efforts suggest this method could democratize access to state-of-the-art large language models by making them runnable on standard consumer devices, even those with limited RAM.
