# Summary

This Stack Overflow article argues that AI-generated backend code creates a new security problem: developers ship insecure backends that "work" (return HTTP 200) but contain critical vulnerabilities like missing request body limits, overly permissive CORS settings, and unvalidated user input. The author illustrates common vulnerabilities found in typical Express.js code that AI assistants generate, showing how these issues pass tests but expose systems to denial-of-service attacks, cross-site request forgery, and server-side request forgery exploits.

The solution proposed is to **invert security defaults**: frameworks should make safe practices the default behavior and require developers to deliberately opt-in to dangerous configurations. The author demonstrates this philosophy through DaloyJS, a TypeScript framework that enforces security by default (strict body limits, request timeouts, schema validation, automatic secure headers) and refuses to start when misconfigurations are detected. This approach ensures that "making code work" and "making code secure" become the same action, protecting against both AI agents and human developers from shipping vulnerable code.

### Vulnerability reports are not special anymore - (Hacker News)

https://words.filippo.io/vuln-reports/

# Summary

For years, open source maintainers have treated vulnerability reports as special—distinct from regular issues—because security researchers provided scarce insight and crucial confidentiality, allowing projects to patch vulnerabilities before attackers could exploit them. However, the author argues this premise no longer holds in 2026: LLMs can now find vulnerabilities as effectively as any human researcher, confidentiality and embargoes are less critical since attackers have equal access to these tools, and the real bottleneck has shifted from discovering vulnerabilities to triaging them. Consequently, vulnerability reports deserve no special status, and maintainers should focus efforts on rapid remediation and prevention through automated LLM analysis in CI/CD pipelines rather than servicing external reports.

The author acknowledges nuances in follow-up discussion, noting that some extremely high-severity reports and those from highly trusted sources may still warrant special treatment, and security teams should develop processes to quickly classify reports into "special" and "routine" categories.

### Show HN: An ASCII 3D Rendering Engine - (Hacker News)

https://glyphcss.com

# Summary

**glyphcss** is a JavaScript library that renders 3D polygon meshes as ASCII art directly in the DOM using a single `<pre>` element, eliminating the need for WebGL or canvas. It supports multiple 3D file formats (OBJ, glTF, GLB, STL, VOX) with textures and materials, and works seamlessly with vanilla JavaScript, React, and Vue through custom elements and framework-specific bindings.

The library achieves interactivity by projecting 3D coordinates onto a character grid and optionally overlaying absolutely-positioned DOM elements for hotspots at specific 3D anchors, enabling real hover states and accessibility features without creating a DOM node per polygon. Installation is available via npm or CDN, with a simple API for creating interactive 3D scenes.

# Summary of Jerry's Map

Jerry Gretzinger began drawing an imaginary city map in 1963 as a casual doodle during work breaks, continuing until 1983 when he set it aside. After his son discovered it decades later in their attic, Jerry resumed the project, which has evolved into a massive two-dimensional "virtual world" comprising over 4,000 individual 8x10-inch panels arranged in a circular formation. The artwork is created using acrylic, markers, colored pencils, ink, collage, and inkjet printing on heavy paper.

The creative process is uniquely driven by a custom deck of approximately 100 cards that provide randomized instructions, allowing Jerry to feel like an observer rather than the sole creator. Each card directs specific tasks across five categories: panel work, collage activities, creating new panels, modifying the card deck itself, and documentation. The map evolves through successive layers—from painted color bands to collaged city squares representing different population densities, progressing through abstract phases like "The Void," "The Red Dimension," and "The Ziggurat." This systematic approach, guided by explicit artistic principles, has transformed what began as simple doodling into an elaborate, continuously evolving collaborative process between artist, randomness, and intentional rules.

### FUTO Swipe – A new swipe typing model - (Hacker News)

https://swipe.futo.tech/

# FUTO Swipe Summary

FUTO Swipe is an open-source swipe typing system designed to provide fast, accurate mobile keyboard functionality while prioritizing privacy and accessibility. Available in FUTO Keyboard (an offline Android app) and as downloadable models for developers, it breaks the traditional lock on quality swipe typing that was previously limited to privacy-invasive commercial apps. The project includes three complementary model types: a universal encoder, a lightweight language model for context, and a layout-specific decoder that together achieve approximately 96% accuracy (top-4 fail rate of ~4%).

The system is remarkably efficient, built on models totaling just 2.5 million parameters that run on low-end devices in milliseconds. FUTO released a dataset of 1 million voluntarily-collected swipes under the MIT license and offers a C++ inference library under GPL, enabling developers to implement swipe typing in various applications. The models' small footprint and low environmental training costs (requiring only a single GPU workstation) make this a sustainable alternative to proprietary keyboard solutions.

### In memory of the man who put red and green squiggles under words - (Hacker News)

https://devblogs.microsoft.com/oldnewthing/20260622-00/?p=112451

# Summary

This article by Raymond Chen is an obituary for Tony Krueger, a software engineer who worked on multiple versions of Microsoft Word and is credited with inventing the red and green squiggles that underline spelling and grammar errors. Krueger improved Word's spell-check feature by making it non-intrusive—instead of blocking user work with interruptions, the spell checker ran in the background and displayed potential errors with colored squiggles in real-time, a feature so beloved that even Penn and Teller praised it.

Krueger's innovation became ubiquitous across word processors and other applications, fundamentally changing how users interact with digital text. The article celebrates his legacy as a designer whose work touched millions of people, despite remaining largely anonymous. Chen notes that this feature appeared before its popularization in "Weird Al" Yankovic's parody video and represents a subtle but profound improvement to user experience that seemed obvious only in retrospect.

### Qwen-AgentWorld: Language World Models for General Agents - (Hacker News)

https://arxiv.org/abs/2606.24597

# Qwen-AgentWorld: Language World Models for General Agents

This paper introduces Qwen-AgentWorld, a suite of language-based world models (35B and 397B parameters) designed to simulate agent environments across seven domains. Trained on over 10 million environment interaction trajectories through a three-stage pipeline (continuous pre-training, supervised fine-tuning, and reinforcement learning), these models can predict environment dynamics and state transitions through chain-of-thought reasoning. The authors also present AgentWorldBench, a comprehensive evaluation benchmark constructed from real-world interactions, demonstrating that Qwen-AgentWorld outperforms existing frontier models.

Beyond serving as foundation models, the work explores two complementary applications: Qwen-AgentWorld can function as a decoupled simulator for scalable reinforcement learning of agents (outperforming real-environment training alone), and as a pre-training warm-up that improves performance across seven downstream agentic benchmarks. This research demonstrates how language-based world modeling can enhance general agent capabilities for reasoning, planning, and decision-making.
