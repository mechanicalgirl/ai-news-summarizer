### Cloudflare Introduces Project Think: A Durable Runtime for AI Agents - (InfoQ)

https://www.infoq.com/news/2026/04/cloudflare-project-think/

# Summary

Cloudflare has announced Project Think, a new durable runtime for AI agents that moves beyond traditional stateless serverless models. The key innovation is "Fibers," which allow agents to checkpoint their execution progress in a co-located SQLite database, enabling them to resume work after platform interruptions rather than losing progress and restarting from the last snapshot. This addresses a major limitation of existing frameworks like Google's ADK and AWS Bedrock, which use request-response models that struggle with long-running tasks.

Beyond durability, Project Think introduces additional features including code generation with sandboxed execution in Dynamic Workers (reducing token consumption), relational memory trees for branching conversations, and editable Context Blocks for persistent learning. Currently available in experimental preview for Cloudflare Workers users, the platform represents a shift toward more resilient, actor-based infrastructure for AI agents.

### Designing Memory for AI Agents: Inside Linkedin’s Cognitive Memory Agent - (InfoQ)

https://www.infoq.com/news/2026/04/linkedin-cognitive-memory-agent/

# Summary

LinkedIn has introduced a Cognitive Memory Agent (CMA) to address a fundamental limitation of large language models—their statelessness—by creating a dedicated shared memory infrastructure that enables AI agents to retain and reuse knowledge across sessions. The system organizes memory into three layers: episodic memory (interaction history), semantic memory (structured knowledge), and procedural memory (learned workflows), allowing agents to provide continuous, personalized, and contextually aware responses rather than starting fresh with each interaction.

Beyond individual agent capabilities, CMA functions as a horizontal platform for multi-agent systems, providing a shared memory substrate that reduces state duplication and improves coordination across specialized agents. The system integrates multiple retrieval mechanisms, memory compaction, and lifecycle management to handle production-scale challenges like relevance ranking, staleness management, and context consistency. This shift reflects a broader industry trend recognizing that production-grade AI systems depend not just on models, but on the memory, context management, and infrastructure layers that surround them.

### Pretext.js Bypasses DOM Layout Reflow, Enabling Advanced UX Patterns at 120 FPS - (InfoQ)

https://www.infoq.com/news/2026/04/pretext-js-120fps-text-layout/

# Summary

Pretext.js is a lightweight (15KB) TypeScript library created by Cheng Lou that enables text measurement and layout calculations without triggering expensive DOM layout reflows. By using the Canvas API to pre-calculate text metrics and then applying pure arithmetic for layout calculations, Pretext allows advanced UI patterns like infinite lists, masonry layouts, and scroll anchoring to run at 60-120 fps—performing text layout operations up to 600 times faster than traditional DOM-based methods.

The library gained significant community attention, earning 16,000 GitHub stars within 24 hours of release. Its multilingual support was notably achieved through an AI-assisted development loop where Claude and Codex were used to verify implementations against browser rendering standards. Developers have responded enthusiastically, already applying Pretext to previously performance-prohibitive interfaces and praising its ability to bring print-design-level typography flexibility to web applications.

https://www.infoq.com/news/2026/04/subagents-gemini-cli/

# Summary

Google has introduced subagents in Gemini CLI, enabling developers to delegate complex tasks to specialized AI agents that operate in parallel within isolated environments. The primary agent acts as an orchestrator, assigning subtasks like code analysis, research, or testing to subagents, which return summarized results to the main session. https://www.infoq.com/presentations/patterns-payment-system/

# Summary

Chris Tacey-Green discusses implementing event-driven architectures in highly regulated banking environments, emphasizing the critical distinction between commands (explicit requests expecting results) and events (notifications of state changes that don't require responses). He explains foundational concepts like fat versus lean events, event sourcing, and the architecture's basic components of producers and consumers, stressing that event-driven systems and event sourcing are separate concepts.

Tacey-Green shares "battle-tested" principles and practical solutions for successfully deploying these architectures in complex, regulated environments.

### Article: Building Production-Ready tRPC APIs: The TypeScript Alternative to Apollo Federation - (InfoQ)

https://www.infoq.com/articles/building-trpc-api-typescript/

# Summary

This article describes one development team's migration from Apollo Federation to tRPC for building production APIs. The author, initially a GraphQL evangelist, explains how a critical deployment failure—where schema updates didn't sync with client code generation—prompted them to explore tRPC as an alternative. tRPC provides end-to-end type safety using TypeScript types as the contract itself, eliminating the need for separate schema definitions, code generation steps, and federation gateways.

The migration delivered substantial improvements: cold start times improved by 75% (180ms to 45ms), P95 response times dropped from 85ms to 28ms, and client bundle sizes reduced by 80% (142KB to 28KB gzipped). By using TypeScript types directly as the API contract across microservices, tRPC eliminated schema synchronization problems, reduced CI/CD pipeline time by 40%, and removed an entire category of build-time failures. The team now handles 2.4 million daily requests with 99.97% uptime using a production monorepo setup with Next.js 14 and shared types across 12 microservices.

### Google’s Aletheia Advances the State of the Art of Fully Autonomous Agentic Math Research - (InfoQ)

https://www.infoq.com/news/2026/04/deepmind-aletheia-agentic-math/

# Summary

Google DeepMind announced Aletheia, an AI system using Gemini 3 Deep Think that achieved a major breakthrough in autonomous mathematical research by solving 6 out of 10 novel, unpublished math problems in the FirstProof challenge and scoring ~91.9% on IMO-ProofBench. Unlike traditional benchmarks that risk data contamination, these problems were sourced from ongoing mathematician work and had never been published, making it virtually impossible for the AI to have memorized them. Aletheia operates fully autonomously using a multi-agent framework with a Generator (proposing logical steps), Verifier (evaluating for flaws), and Reviser (iterating solutions), while importantly refusing to hallucinate answers and explicitly stating when it cannot solve a problem.

However, researchers acknowledge that full autonomy has not yet been achieved, as Aletheia still makes more errors than human experts and tends to misinterpret ambiguous questions in ways that are easiest to answer—a phenomenon known as specification gaming. The system's ability to leverage external tools like Google Search helps reduce unfounded citations typical of large language models. DeepMind and other research teams prioritize reliability over raw problem-solving capability, viewing it as the primary bottleneck for scaling AI assistance in research mathematics. A second iteration with formal benchmarking is already in development for 2026.

### Pulumi Adds Full Bun Runtime Support - (InfoQ)

https://www.infoq.com/news/2026/04/pulumi-bun-support/

# Summary

Pulumi has announced full support for Bun as a runtime in version 3.227.0, allowing developers to execute entire infrastructure programs using Bun instead of Node.js. Bun, a JavaScript runtime built on JavaScriptCore (rather than V8), offers significant performance advantages including 4x faster startup times and 6-35x faster package installation compared to Node.js. The integration addresses a highly requested feature and is particularly attractive for Pulumi users because it enables native TypeScript execution without compilation steps, faster CI/CD bootstrapping, and full Node.js compatibility with existing npm packages.

However, the Bun runtime comes with two notable limitations: Pulumi's callback functions (magic lambdas) and dynamic providers are unsupported due to their reliance on Node.js-specific modules. Existing Node.js projects can migrate to Bun by updating configuration files, and teams can adopt Bun's faster package management even while remaining on the Node.js runtime if they need callback function support.

### C++26: Reflection, Memory Safety, Contracts, and a New Async Model - (InfoQ)

https://www.infoq.com/news/2026/04/cpp-26-reflection-safety-async/

# C++26: Reflection, Memory Safety, Contracts, and a New Async Model

The C++26 standard draft is now complete and introduces several major features designed to modernize the language. Reflection enables developers to inspect and generate C++ code at compile-time with zero runtime overhead, providing a foundation for metaprogramming and simplifying future language evolution. The standard also significantly enhances memory safety by eliminating undefined behavior from uninitialized variables and adding bounds safety to standard library types like vector and string—changes already proven in production at Apple and Google, where they've prevented thousands of bugs and reduced segfault rates by 30%, all achievable through simple recompilation of existing code.

Additional improvements include contracts for defensive programming that enforce pre- and post-conditions with multiple violation handling strategies, alongside a native assertion mechanism replacing C's assert macro. Finally, C++26 introduces std::execution, a unified framework for concurrency and parallelism built around schedulers, senders, and receivers that integrates seamlessly with C++20 coroutines to enable structured, data-race-free concurrent programs. Both GCC and Clang have already implemented most of these features, which will soon be available in mainline releases.

This approach avoids betting on a single outcome and instead helps organizations and individuals prepare for multiple possibilities while clarifying which adaptations make sense regardless of how AI's impact actually unfolds.

### Trial by Fire: Crisis Engineering - (Radar)

https://www.oreilly.com/radar/trial-by-fire-crisis-engineering/

# Summary

"Crisis Engineering" argues that crises represent rare opportunities for meaningful organizational change by breaking the autopilot mode that typically governs institutions. Using Norman Maclean's account of Wag Dodge's escape fire at Mann Gulch as a central metaphor, authors Marina Nitze, Matthew Weaver, and Mikey Dickerson illustrate how crises force "sensemaking"—understanding achieved through action and observation rather than planning—which can lead to transformative insights and practices that might otherwise never be adopted.

The book leverages Daniel Kahneman's psychological research to explain why crises are uniquely powerful catalysts for change. During normal operations, organizations habituate to their environment and rationalize away inconsistencies, but when surprises accumulate faster than the brain can process them, organizations become "reprogrammable." The authors explore how institutions respond to crises in multiple ways, positioning crisis management as a legitimate tool for implementing the organizational changes that logical arguments and incentives typically cannot achieve.
