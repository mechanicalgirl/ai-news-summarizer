### Peter Steinberger – WhatsApp CLI: sync, search, send - (Hacker News)

https://github.com/steipete/wacli

# Summary

**wacli** is an open-source WhatsApp command-line interface tool built on the whatsmeow protocol that enables users to sync message history locally, search messages offline, send messages, and manage contacts and groups. It functions as a third-party WhatsApp Web client and is not affiliated with WhatsApp.

The tool can be installed via Homebrew or built locally with Go, and features core functionality including interactive authentication, continuous message syncing, fast offline search, file sharing capabilities, and history backfilling. It stores data locally (defaulting to ~/.wacli) and supports both human-readable and JSON output formats, with additional features like media downloading and group management commands.

### Dependency cooldowns turn you into a free-rider - (Hacker News)

https://calpaterson.com/deps.html

# Summary

The article argues against dependency cooldowns—a popular practice of waiting N days before adopting new package versions to let others discover potential supply chain attacks—as a flawed security approach. While individually modestly beneficial, cooldowns essentially rely on free-riding: they depend on others who don't implement cooldowns serving as unpaid beta testers who will inevitably get hacked, exposing vulnerabilities before the cautious adopt the updates. The approach is also incomplete, requiring implementation across multiple package managers and projects, and is easily circumvented by accidental installations outside configured settings.

The author proposes an alternative: centralized upload queues (separation of package publication from distribution) similar to Debian's system, where new packages wait in a queue before public distribution. This eliminates free-riding, requires no per-project configuration, allows automated security scanning and human review, and reduces the surprise element of releases—giving maintainers advance notice to prevent compromised versions from spreading. The author argues this approach is particularly critical for AI systems, where markdown files are now executable formats, and addresses funding concerns by noting that well-funded organizations like Microsoft (npm) and the Python Software Foundation could sustain such systems, potentially through paid expedited-review services for commercial projects.

### Claude Code Routines - (Hacker News)

https://code.claude.com/docs/en/routines

# Summary

Claude Code Routines are automated workflows that run on Anthropic-managed cloud infrastructure, allowing you to execute pre-configured Claude Code sessions without manual intervention. A routine consists of a prompt, repositories, connectors, and environment settings that can be triggered on a schedule (hourly, daily, weekly), via API calls, or in response to GitHub events like pull requests. Routines are available on Pro, Max, Team, and Enterprise plans and can be created and managed through the web interface at claude.ai/code/routines, the CLI, or the Desktop app.

Routines are designed for repeatable, unattended tasks such as backlog maintenance, alert triage, code review, deploy verification, and documentation updates. When creating a routine, you define its prompt (which determines what Claude does), select repositories it can access, configure an environment with necessary variables and network access, and set up triggers. Each routine belongs to an individual account, counts against daily run limits, and executes actions (commits, PRs, messages) under that user's identity. Runs create new sessions where you can review Claude's work and manage pull requests.

### My adventure in designing API keys - (Hacker News)

https://vjay15.github.io/blog/apikeys/

# Summary

The author, now a Product Developer, explores the design and implementation of API keys, particularly for a multi-tenant system with database sharding. API keys are authentication tokens formatted with a prefix (metadata), random hex string, and checksum. The main challenge was routing requests to the correct database shard without session cookies. The author tested three initial approaches: directly mapping key hashes to account IDs, using unique prefixes for each company, and encoding hashes with base62/base70 algorithms. However, benchmarking revealed that the encoding approaches performed worse than expected due to inefficient BigInt operations in JavaScript and the effectiveness of B-Tree indexing on larger strings.

The author ultimately selected the SHAKE256 hash algorithm (from the SHA3 family) as the final solution. SHAKE allows generating fixed-size outputs of any desired length, reducing both computation costs and index size while maintaining strong performance. Using a 10-character base64url-encoded output of SHAKE256 provides approximately 72 quintillion possibilities, making hash collisions virtually impossible while being more efficient than full SHA-256 hashes. This approach balances security, performance, and practical implementation needs for the multi-tenant API infrastructure.

### Not all elementary functions can be expressed with exp-minus-log - (Hacker News)

https://www.stylewarning.com/posts/not-all-elementary/

# Summary

Robert Smith critiques a viral paper by Andrzej Odrzywołek claiming that the function E(x,y) = exp x - log y can express all elementary functions. While Smith acknowledges the result is "neat and thought-provoking" under Odrzywołek's specific definition of "elementary" (36 fixed symbols), he argues the paper's title is misleading because it contradicts the standard mathematical definition of elementary functions developed in the 19th century.

Smith demonstrates using Khovanskii's topological Galois theory that EML terms cannot express arbitrary polynomial roots, which are part of the standard definition of elementary functions. Specifically, he proves that all EML terms have solvable monodromy groups, while the generic quintic equation has a non-solvable monodromy group (S₅). Therefore, EML terms form a strict subset of standard elementary functions, meaning the paper's core claim fails under conventional mathematical terminology.

### Founders Need to Be Ruthless When Chasing Deals - (Hacker News)

https://steveblank.com/2024/04/16/founders-need-to-be-ruthless-when-chasing-deals/

# Summary

The article, based on Steve Blank's blog post, emphasizes that startup founders must be ruthless about qualifying potential customer deals before investing significant engineering resources. Blank illustrates this with a cautionary tale from his own enterprise software experience, where he spent months having his engineering team build a functional prototype for an excited potential customer, only to be ghosted after the demo. He made critical mistakes by never asking about budget, purchase authority, decision timelines, or success criteria—essentially wasting precious startup resources on an unqualified opportunity.

To address this problem, Blank advocates implementing a rigorous sales qualification process that separates genuine customer interest from "tire-kicking." Before committing to building demos, startups should ask detailed questions about decision-making processes, budgets, timelines, and approval chains. Blank's key solution was sending a fully cancelable purchase order to prospective buyers, forcing them to validate their genuine interest and intent to buy before the startup invested in prototype development. This approach protects limited engineering resources and ensures efforts are focused on high-probability deals with real potential for conversion.
