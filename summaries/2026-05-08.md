### How we replaced Ingress-NGINX at Stack Overflow - (Stack Overflow Blog)

https://stackoverflow.blog/2026/05/06/how-we-replaced-nginx-ingress-at-stack-overflow/

# Summary

Stack Overflow needed to replace Ingress-NGINX after its retirement was announced, and decided to use this as an opportunity to migrate to the newer Gateway API standard. After evaluating options based on conformance, cloud compatibility, and feature sets, they narrowed testing down to three Gateway API implementations (NGINX Gateway, Traefik, and Istio) and two Ingress backups. They created a comprehensive testing environment using HTTPBin and a custom Go server to validate their existing use cases and performance requirements.

In their testing, all three implementations successfully handled Stack Overflow's use cases and could manage their required traffic load of 10,000 RPS across various latency scenarios. However, they found that Gateway API features often lacked the depth needed in practice, requiring fallback to implementation-specific extensions. Notably, Traefik was significantly slower at route creation with thousands of HTTPRoutes, while Istio's advanced features came with greater syntactic complexity. The team ultimately determined that all three solutions met their scalability requirements, though each had different trade-offs regarding feature depth, performance, and integration complexity.

### Dirtyfrag: Universal Linux LPE - (Hacker News)

https://www.openwall.com/lists/oss-security/2026/05/07/8

# Dirty Frag: Universal Linux LPE Summary

"Dirty Frag" is a critical local privilege escalation (LPE) vulnerability that affects all major Linux distributions, allowing unprivileged users to gain root access. The vulnerability chains two separate kernel vulnerabilities in the networking subsystem (ESP4/ESP6 and RXRPC modules) and has a similar impact to the previously disclosed "Copy Fail" vulnerability.

Because the responsible disclosure embargo was broken before patches were developed, no fixes currently exist across distributions. The researcher has provided a mitigation command to disable the vulnerable kernel modules and released full technical details and exploit code at dirtyfrag.io. Users are advised to immediately disable the ESP4, ESP6, and RXRPC kernel modules to prevent exploitation.

### Blaise – A modern self-hosting zero-legacy Object Pascal compiler targeting QBE - (Hacker News)

https://github.com/graemeg/blaise

# Blaise: A Modern Object Pascal Compiler

Blaise is a next-generation Object Pascal compiler designed to modernize the language by eliminating decades of legacy complexity. Built from scratch for contemporary development, it features a single unified language mode, one UTF-8 string type with automatic reference counting, clean zero-GUID interfaces, reified generics, and first-class debugger support via the OPDF format. The compiler is self-hosting, cross-platform, and targets native code through QBE and LLVM backends.

The project removes many classic Pascal features deemed problematic (multiple string types, the `with` statement, COM-style GUIDs, and multiple language dialects) in favor of simplicity and consistency. Currently in active development with over 1,200 tests, Blaise has completed phases for bootstrapping, type systems, generics, and debug info, with ongoing work on self-hosting, LLVM support, and IDE integration planned for future phases. The codebase is organized with a modular PasBuild structure and is licensed under Apache 2.0.

### Pinocchio is weirder than you remembered - (Hacker News)

https://storica.club/blog/pinocchio-in-italian/

# Summary

The original *Pinocchio* serialized in 1881 is far darker and weirder than most people remember, ending abruptly with the puppet hanged from a tree until Italian children demanded the author continue. Carlo Collodi, a satirist-turned-children's author, reluctantly resumed the story and filled the subsequent chapters with grotesque imagery—a cricket hammered to death, a fairy introduced as a corpse, and a donkey skinned to make a drum—written with deadpan irony that mocked the sentimental moralizing of contemporary children's literature.

Beyond its satirical legacy, *Pinocchio* played an unexpected role in Italian history: it became a foundational schoolbook that helped unify the linguistically fragmented nation by introducing generations of children to standard Tuscan Italian. When Italy unified in 1861, only 2.5% of the population spoke standard Italian; Collodi's clear, accessible prose helped raise that figure to 87% by 1951, making the reluctantly-extended puppet story one of the most translated books in human history and an accidental instrument of linguistic nation-building.

### The map that keeps Burning Man honest - (Hacker News)

https://www.not-ship.com/burning-man-moop/

# Summary

Each year, Burning Man's 70,000 participants build a temporary city on a Nevada lakebed, then leave behind a team of 150 people who meticulously search the 3,800 acres for MOOP—"Matter Out of Place"—cataloging every scrap of debris in a color-coded map. This forensic cleanup effort is essential because the Bureau of Land Management requires the site to pass strict inspections, allowing no more than one square foot of debris per acre, with only 12 of 120 test points permitted to exceed this limit.

Over its 20-year history, the MOOP Map has become a powerful accountability tool that drives community improvement, with data showing that debris per person has steadily declined even as the event has grown. By providing camps with breakdowns of what was found on their footprint and publicly highlighting cleanup patterns, the map transforms Burning Man's "Leave No Trace" principle from an abstract ideal into a tangible responsibility, demonstrating that transparency and shared accountability can effectively change behavior over time.

### Agents need control flow, not more prompts - (Hacker News)

https://bsuh.bearblog.dev/agents-need-control-flow/

# Summary

The article argues that AI agents need structured, deterministic control flow built into software rather than relying on increasingly complex prompt chains. The author contends that prompting has inherent limitations—it's non-deterministic, weakly specified, and unreliable at scale—much like a programming language where statements are merely suggestions. True reliability demands treating LLMs as components within a formally orchestrated system with explicit state transitions and validation checkpoints.

Beyond orchestration, the article emphasizes that agents prone to silent failures require aggressive error detection and programmatic verification. Without these safeguards, systems are forced into one of three unsatisfactory options: keeping humans in the loop, performing exhaustive post-execution audits, or simply accepting outputs on faith. The core message is that scaling AI agents requires engineering discipline through code-based control flow, not prompt elaboration.

### A polynomial autoencoder beats PCA on transformer embeddings - (Hacker News)

https://ivanpleshkov.dev/blog/polynomial-autoencoder/

# Summary

This article introduces a **polynomial autoencoder** for compressing neural network embeddings by combining PCA encoding with a quadratic (degree-2 polynomial) decoder. Rather than using a linear decoder, the method adds a nonlinear polynomial layer fitted via Ridge regression to capture variance that standard linear PCA misses—particularly the "cone effect" in transformer embeddings where data is concentrated in a narrow cone on the unit sphere. The approach is closed-form (no SGD training required) and implemented in ~150 lines of numpy.

**Key results show that on the BEIR retrieval benchmark, this method achieves 4× memory compression (256→512 bytes per vector) while losing only 0.7-1.4 NDCG points compared to raw 1024-dimensional embeddings.** The quadratic decoder consistently improves over standard PCA by 1-4.4 percentage points, with the largest gains (+2.3-4.4 p.p.) on non-Matryoshka-trained models at d=256 compression. However, the method requires corpus-side statistics fitting, making it unsuitable for multi-tenant or streaming scenarios.
