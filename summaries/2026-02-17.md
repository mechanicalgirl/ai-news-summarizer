# Summary

Eclipse Foundation has released GlassFish 8.0.0, a lightweight open-source application server that serves as a reference implementation for Jakarta EE 11. The release requires JDK 21 minimum and supports JDK 25, bringing several major enhancements including virtual thread support for handling concurrent requests efficiently, improved data access through Jakarta Data 1.0 specification integration with Eclipse JNoSQL, and enhanced security via MicroProfile JWT Authentication 2.1 and Jakarta Security 4.0 specifications.

## https://www.infoq.com/news/2026/02/agoda-api-agent/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global

# Summary

Agoda has developed API Agent, a zero-code system that automatically converts any REST or GraphQL API into a Model Context Protocol (MCP) server, enabling AI assistants to query internal services without building individual MCP servers for each API. The system uses schema introspection to dynamically understand API structures, automatically generates natural language queries, and can manage multiple APIs through a single deployment with just configuration updates.

The architecture employs DuckDB for intelligent response handling, storing full API results and using SQL post-processing to filter data before sending concise summaries to language models, thereby managing context limits and avoiding arbitrary code execution. Security is enforced by default in read-only mode, with mutations blocked unless explicitly whitelisted, and the project is open-sourced for REST and GraphQL experimentation.

## https://www.infoq.com/news/2026/02/google-agent-scaling-principles/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global

# Summary

Google Research has published the first quantitative scaling principles for multi-agent AI systems based on evaluating 180 agent configurations. Their findings challenge the common assumption that "more agents are better," showing that multi-agent coordination only improves performance for specific task types. Parallelizable tasks like financial reasoning benefit significantly (80.9% improvement), while sequential reasoning tasks suffer performance degradation of 39-70% due to communication overhead and reduced "cognitive budget."

The research identifies key factors in choosing the right architecture: parallelizable tasks benefit from multi-agent systems, while sequential tasks perform better with single agents. The study also reveals that multi-agent systems can amplify errors up to 17 times compared to 4.4 times with centralized coordination, and that tool-use bottlenecks increase coordination costs. The researchers developed a predictive model achieving 87% accuracy in recommending the optimal architecture based on task properties like sequential dependencies and tool density.

## https://www.infoq.com/news/2026/02/claude-built-c-compiler/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global

# Summary

Anthropic researcher Nicholas Carlini used sixteen Claude AI agents working in parallel to build a Rust-based C compiler from scratch, demonstrating the potential of autonomous multi-agent software development. The agents coordinated changes through a shared Git repository using a simple lock-based task assignment system, ultimately producing a 100,000-line compiler capable of compiling the Linux kernel and major open-source projects like FFmpeg and PostgreSQL, with 99% compatibility with GCC's torture tests. The project consumed approximately 2,000 AI sessions and $20,000 in API costs over two weeks.

However, the achievement sparked significant online debate about its practical significance. Additionally, some questioned whether the agents were largely reproducing code from their training data and whether the comparison to GCC's 37-year development timeline was fair, since GCC has continuously evolved with language standards rather than being built from scratch. The experiment raises important questions about the nature of autonomous software development and the future role of human engineers in the AI era.
