### Local AI needs to be the norm - (Hacker News)

https://unix.foo/posts/local-ai-needs-to-be-norm/

# Summary

The article argues that developers should prioritize local AI models running on users' devices rather than relying on cloud-based AI services like OpenAI or Anthropic. The author contends that this trend of outsourcing AI to remote servers creates fragile, privacy-invasive software that depends on network connectivity, vendor uptime, and billing—turning simple features into complex distributed systems. Modern devices have powerful processors and neural engines capable of handling tasks like summarization, classification, and text extraction locally, without transmitting user data to third parties.

The author illustrates this approach with a concrete example: an iOS news app that summarizes articles on-device using Apple's local language models, eliminating privacy concerns and vendor dependencies. Modern tooling—particularly Apple's FoundationModels framework—makes local AI implementation straightforward, and local models excel at transforming user-owned data rather than requiring general intelligence. The author concludes that while local models may be less powerful than cloud alternatives, they're more than adequate for most real-world app features, and developers should reserve cloud AI only for cases where it's genuinely necessary.

### I'm going back to writing code by hand - (Hacker News)

https://blog.k10s.dev/im-going-back-to-writing-code-by-hand/

# Summary

The author of k10s, a GPU-aware Kubernetes dashboard, abandoned their AI-assisted "vibe-coding" approach after seven months and 234 commits. They discovered that while Claude could rapidly generate individual features (achieving 10x velocity initially), the lack of architectural oversight led to a catastrophic "god object"—a 1690-line Model struct with 500-line update methods that became unmaintainable. When features like the GPU fleet view were added without proper isolation, state bled between views, breaking core functionality.

The key lesson is that AI excels at implementing features within existing architecture but cannot design sound architecture itself. The author identifies five tenets for sustainable AI-assisted development, with the primary recommendation being to write explicit architectural constraints (interfaces, ownership rules, message types) in advance and document them in a CLAUDE.md file. This prevents the AI from defaulting to monolithic structures and ensures each subsequent feature respects established patterns rather than eroding them. Human judgment on architecture remains essential; AI should be constrained to execute within that framework.

### 7 lines of code, 3 minutes: Implement a programming language (2010) - (Hacker News)

https://matt.might.net/articles/implementing-a-programming-language/

# Summary

This article demonstrates how to implement a functional programming language interpreter in just 7 lines of code using the lambda calculus, a minimalist Turing-equivalent language with only three expression types: variable references, anonymous functions, and function calls. The author explains the historical context (Church's lambda calculus from 1929 and its equivalence to Turing machines) and shows how the eval/apply design pattern creates a scalable architecture that can interpret this tiny but powerful language.

The article goes beyond the minimal example by providing a cleaner Racket implementation and then scaling up to a 100-line interpreter supporting practical features like conditionals, variable bindings, recursion, mutation, and top-level function definitions. This demonstrates that the same eval/apply design pattern used in the 7-line interpreter can be extended to support much richer programming languages, making it an excellent pedagogical example for understanding both language implementation fundamentals and how complex language features build on simple foundations.

### Obsidian plugin was abused to deploy a remote access trojan - (Hacker News)

https://cyber.netsecops.io/articles/obsidian-plugin-abused-in-campaign-to-deploy-phantom-pulse-rat/

# Summary

Cybersecurity researchers have discovered a sophisticated social engineering campaign targeting finance and cryptocurrency professionals that weaponizes the Obsidian note-taking application to deliver a new Remote Access Trojan called PHANTOMPULSE. Attackers pose as venture capitalists on LinkedIn and Telegram, building trust before luring victims into opening a malicious shared Obsidian vault. Once victims enable the "community plugins" synchronization feature, compromised versions of legitimate plugins execute malicious code that deploys the RAT on both Windows and macOS systems.

PHANTOMPULSE demonstrates advanced capabilities, including a novel command-and-control mechanism that uses the Ethereum blockchain to dynamically resolve server addresses, making it highly resistant to takedowns. A successful compromise grants attackers full system access, enabling them to steal sensitive data, intellectual property, trading strategies, and critically, cryptocurrency credentials and wallet keys. Defenders are advised to implement process monitoring for Obsidian spawning command-line tools, educate users about social engineering risks, restrict unapproved plugin installations, and deploy endpoint detection and response solutions to detect suspicious script execution.

### An AI coding agent, used to write code, needs to reduce your maintenance costs - (Hacker News)

https://www.jamesshore.com/v2/blog/2026/you-need-ai-that-reduces-your-maintenance-costs

# Summary

James Shore argues that AI coding agents are only worthwhile if they proportionally reduce maintenance costs. While these tools may increase code output significantly, they often increase the cost of maintaining that code through lower quality, harder-to-understand implementations. Using a mathematical model, Shore demonstrates that without corresponding reductions in maintenance burden, teams will initially gain productivity but eventually become worse off than before adopting AI—trapped in a cycle of escalating maintenance obligations they can't escape.

The core principle is simple: if AI doubles your code output, it must halve your maintenance costs to break even. Currently, evidence suggests AI-generated code increases rather than decreases maintenance expenses, making widespread adoption a risky proposition. Shore concludes that teams must focus equally on reducing maintenance costs alongside improving coding speed, or risk trading temporary productivity gains for permanent loss of efficiency.

### Mythos Finds a Curl Vulnerability - (Hacker News)

https://daniel.haxx.se/blog/2026/05/11/mythos-finds-a-curl-vulnerability/

# Summary

Anthropic's AI model Mythos generated significant hype for its alleged ability to find security vulnerabilities in source code. Daniel Stenberg, the lead developer of curl, was offered access to scan curl's 176,000-line codebase. After Mythos analyzed the code, it reported five "confirmed security vulnerabilities," which curl's security team reviewed and ultimately reduced to just one actual confirmed vulnerability (a low-severity flaw) plus three false positives and one minor bug. The report also identified about twenty additional bugs for investigation.

Stenberg concludes that while Mythos found legitimate issues and is certainly useful, the hype surrounding it was largely marketing-driven. The model performed no better than previous AI code analyzers (AISLE, Zeropath, OpenAI's Codex Security) that had already helped curl fix hundreds of bugs. However, Stenberg reiterates that AI-powered code analyzers in general are significantly superior to traditional tools and represent a major advancement—any project not using them risks leaving exploitable flaws undiscovered. The key difference with AI analyzers is their ability to understand context, protocols, APIs, and generate explanations and patches for found issues.
