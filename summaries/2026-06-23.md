### Article: Understanding ML Model Poisoning: How It Happens and How to Detect It - (InfoQ)

https://www.infoq.com/articles/understanding-ml-model-poisoning/

# Summary

Data poisoning is a serious and growing threat to machine learning systems where adversaries deliberately manipulate training data with maliciously crafted examples that blend seamlessly into datasets, making detection difficult. The article explores various poisoning techniques—including label flipping, backdoors, clean-label poisoning, and gradient manipulation—and emphasizes that these attacks can remain hidden for months or years after model deployment, causing models to misbehave in unpredictable or attacker-controlled ways.

To defend against data poisoning, organizations must implement layered defenses combining detection techniques with traditional cybersecurity measures such as securing data storage and protecting system integrity. The article stresses that detecting poisoned data is challenging but achievable, and provides practical guidance on best practices for safeguarding ML pipelines, particularly as companies increasingly rely on public or crowdsourced datasets that are more vulnerable to poisoning attacks.

### Podcast: How eBPF Empowers Developers to Observe Inside the Linux Kernel in a Safe and Unintrusive Way - (InfoQ)

https://www.infoq.com/podcasts/empowers-developers-inside-linux-kernel/

# Summary

eBPF has evolved beyond its origins in packet filtering to become a safe, unintrusive mechanism for extending the Linux kernel and observing its inner workings. The technology uses a strict "verifier" security mechanism that ensures eBPF programs cannot crash the kernel or access invalid memory, enabling developers to deploy observability and security tools without the risks and delays associated with traditional kernel modules or the lengthy Linux upstreaming process.

The real power of eBPF extends far beyond networking (where it's used in tools like Cilium) into deep observability and proactive security enforcement across system calls, file systems, and device drivers. Tools like Tetragon leverage eBPF to intercept and block malicious behavior before kernel execution occurs, enabling "front-foot" security rather than post-incident monitoring. As eBPF expands to support Windows and other operating systems, it promises to unify observability and security policies across heterogeneous environments.

### AWS Graviton5 Reaches General Availability with 192 Cores and Formally Verified VM Isolation - (InfoQ)

https://www.infoq.com/news/2026/06/aws-graviton5-ga/

# Summary

AWS has released the Graviton5 processor, available in M9g and M9gd instances with 192 cores built on TSMC's 3nm process. The chip features 192 MB of L3 cache (5x the previous generation), DDR5-8800 memory, PCIe Gen 6, and notably, formally verified hypervisor isolation—the first production cloud instance to offer mathematically proven security boundaries between virtual machines. While AWS markets Graviton5 for agentic AI workloads, it's fundamentally a high-performance general-purpose ARM processor that benefits any CPU-intensive application.

Customer benchmarks show significant real-world improvements: ClickHouse achieved 36% performance gains, Honeycomb saw 36% better throughput per core, and HubSpot observed up to 60% lower query durations for MySQL. At 9% higher pricing than the previous M8g generation but delivering 25% better compute performance, Graviton5 offers approximately 15% better price-performance. Major customers including Meta (planning to deploy tens of millions of cores), Uber, and Snowflake are already adopting the new instances, with compute-optimized (C9g) and memory-optimized (R9g) variants expected later in 2026.

### Apple Launches Core AI for Apple-Silicon Optimized On-Device Generative AI - (InfoQ)

https://www.infoq.com/news/2026/06/apple-core-ai-wwdc/

# Summary

At WWDC 26, Apple announced Core AI, the official successor to Core ML, designed to enable developers to run large language models and generative AI entirely on Apple Silicon devices. The framework supports both custom-converted PyTorch models and pre-optimized open-source models ranging from 3B to 70B parameters, ensuring user privacy, zero server dependencies, and no cloud costs. Key features include unified hardware access across CPU, GPU, and Neural Engine, a memory-safe Swift API, and ahead-of-time compilation for near-instant load times.

Apple provides developers with tools to convert PyTorch models to Core AI format, including model compression techniques like quantization and palettization to optimize performance on Apple hardware. The company now offers three distinct approaches for ML/AI deployment: Core ML for traditional machine learning, Core AI for neural networks and transformers, and MLX Swift for custom model weights. Community reception has been positive, though the framework's long-term value will depend on its continued growth and developer adoption.

### AWS Adds Multi-Region Replication to Amazon Cognito Identity Service - (InfoQ)

https://www.infoq.com/news/2026/06/cognito-replication-aws/

# Summary

AWS has introduced multi-region replication for Amazon Cognito, its managed identity service, allowing user identities and pool configurations to automatically replicate from a primary region to a secondary read-only region. This enables applications to continue authenticating users during regional outages without requiring custom replication solutions, while maintaining active sessions across regions since both regions recognize access tokens from either location.

The feature supports all authentication methods including federated sign-in and multi-factor authentication, though it currently has some limitations: it operates in active-passive mode (no new sign-ups or password resets in the secondary region), TOTP MFA isn't supported on replicas, and failover requires DNS configuration and custom health checks. The feature is available for Cognito Essentials and Plus tier customers at $0.0045-$0.006 per monthly active user per replica region, with availability across select AWS regions.

### Behind the Scenes:  Block 450 JVM Repositories into Monorepo to Reduce Dependency Drift - (InfoQ)

https://www.infoq.com/news/2026/06/block-450-jvm-monorepo-migration/

# Summary

Block, Inc. successfully migrated approximately 450 JVM-based repositories from a polyrepo architecture to a monorepo to address dependency management challenges and improve developer experience across its Cash App and Square engineering organizations. The previous polyrepo model suffered from dependency version drift, duplicated upgrade efforts, and coordination complexity requiring coordinated deployments. The new monorepo supports ~8,800 builds per week with p90 CI times around 10 minutes and enables atomic updates across services through single commits.

To manage the scale, Block implemented several key innovations: a custom IntelliJ plugin that loads only necessary projects while swapping external dependencies with published JARs to maintain IDE performance, merge queues to keep the main branch stable, dependency graph-based build scoping to optimize which projects need rebuilding based on file changes, and git performance improvements. This consolidation transformed the developer experience by providing a cohesive codebase with faster CI times and a foundation for long-term velocity gains.

### TSRX: a Framework-Agnostic Alternative to JSX - (InfoQ)

https://www.infoq.com/news/2026/06/tsrx-alternative-jsx/

# Summary

TSRX is a new TypeScript language extension created by Dominic Gannaway (former React and Svelte core engineer) that serves as a framework-agnostic alternative to JSX for building declarative user interfaces. It compiles a single .tsrx component file to multiple runtime targets through plugins for React, Preact, Solid, Vue, and Ripple, allowing developers to write UI code once while targeting different frameworks without changing the language itself.

The extension introduces statement-level control flow (@if, @for, @switch, @try), prop shorthands, and component-scoped styles directly into template syntax, moving common UI patterns from convention into syntax. Currently in alpha, TSRX can be adopted incrementally on a per-file basis, with support for major build tools and linters. While the announcement generated significant enthusiasm, there is some debate in the community about whether it represents progress or continues an existing pattern of mixing XML-like syntax with JavaScript.

### Article: Designing Continuous Authorization for Sensitive Cloud Systems - (InfoQ)

https://www.infoq.com/articles/continuous-authorization-cloud/

# Summary

This article discusses how traditional cloud authorization models fail to protect sensitive data because they make authorization decisions only once at login, leaving high-risk operations unchecked during active sessions. The author illustrates this vulnerability with real-world scenarios where employees with legitimate access credentials can still perform unauthorized bulk data exports or abnormal queries that should trigger alerts.

The solution proposed is **continuous authorization architecture**, which treats every sensitive data operation as an independent authorization checkpoint. Rather than relying solely on role-based access control (RBAC), this approach evaluates whether users *should* perform specific actions based on behavioral baselines, location, time of day, and data sensitivity in real-time. The system balances security with performance by using cached decisions for routine operations while triggering deeper evaluation for suspicious patterns, ultimately shifting authorization from static login-time permissions to dynamic runtime decision-making to reduce large-scale data exposure in distributed cloud systems.

### Loop Engineering - (Radar)

https://www.oreilly.com/radar/loop-engineering/

# Summary

Loop engineering represents a fundamental shift in how developers interact with AI coding agents—moving from manual prompting to designing automated systems that manage agents themselves. Instead of manually writing prompts and reviewing outputs turn-by-turn, developers now build recursive loops that handle work discovery, task delegation, quality checking, and decision-making autonomously. This approach is becoming standardized across major AI coding platforms like Codex and Claude Code.

Effective loops require five core components: automated scheduling systems for discovery and triage, parallel worktrees to prevent conflicts, documented project knowledge (skills), integrations with existing tools (plugins), and subagents for independent verification. Additionally, persistent external memory (like Markdown files or Linear boards) is essential since AI models lose context between runs. While still in early stages and requiring careful attention to token costs, loop engineering is emerging as the likely future of developer-agent collaboration, replacing the direct prompting paradigm that has dominated for the past two years.
