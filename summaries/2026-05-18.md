### GenCAD - (Hacker News)

https://gencad.github.io/

# GenCAD Summary

GenCAD is an AI-based generative model that creates parametric CAD programs and 3D solid models directly from images. Unlike existing approaches that use simplified representations like meshes or point clouds, GenCAD generates complete, editable CAD command sequences that preserve the accuracy and modifiability essential for engineering and manufacturing applications.

The model combines four key components: a transformer encoder for learning CAD command representations, contrastive learning to align CAD sequences with images, a latent diffusion model for conditional generation, and a decoder that converts learned representations back into executable CAD commands. This approach enables precise 3D modeling from images while maintaining full parametric control, advancing automated design processes and design space exploration.

### Prolog Coding Horror - (Hacker News)

https://www.metalevel.at/prolog/horror

# Summary

This article warns Prolog programmers against common coding practices that produce defective programs. The main "horrors" discussed are: losing solutions through impure constructs like cut (!), if-then-else (->), and type-checking predicates; introducing implicit dependencies via global database modifications (assertz/retract); printing output directly to the terminal instead of returning pure Prolog terms; and relying on low-level arithmetic predicates (is, =:=, >) rather than modern constraint-based alternatives. Each practice undermines the declarative nature of Prolog and prevents programs from working correctly with general queries.

The author advocates for staying within Prolog's pure, monotonic subset by using clean data structures, constraint predicates like dif/2, and high-level constructs like CLP(FD) constraints. The article illustrates this with a "horror factorial" example that loses solutions due to the cut operator, then demonstrates how using constraint-based arithmetic (#>, #=) and removing the cut enables the same function to work correctly for general queries. The core message is to embrace Prolog's declarative potential through purity rather than clinging to outdated, imperative-style features.

### Jank now has its own custom IR - (Hacker News)

https://jank-lang.org/blog/2026-05-08-optimization/

# Summary

Jank, a Clojure dialect compiler, has developed its own custom intermediate representation (IR) to enable better optimizations and compete with the JVM. Unlike previous versions that delegated optimization work to LLVM, jank's new IR operates at Clojure's semantic level, handling concepts like vars, persistent data structures, and lazy sequences that LLVM couldn't optimize effectively. The IR is SSA-based (single static assignment) and represented as a control flow graph, allowing for more targeted optimizations specific to jank rather than general-purpose compiler platforms.

With the IR now merged into the main codebase after six weeks of development, the jank team is beginning optimization work, starting with a recursive fibonacci benchmark. This benchmark tests important real-world compiler challenges including polymorphic arithmetic, recursion handling, garbage collection, and runtime efficiency. The goal is to make jank competitive with or faster than Clojure JVM.

### Show HN: Semble – Code search for agents that uses 98% fewer tokens than grep - (Hacker News)

https://github.com/MinishLab/semble

# Summary

**Semble** is a fast and efficient code search library designed specifically for AI agents. It enables natural language code searches that return only relevant code snippets, using approximately 98% fewer tokens than traditional grep-and-read approaches. The tool indexes codebases in ~250ms and answers queries in ~1.5ms entirely on CPU, with no API keys, GPU, or external services required. It achieves 99% of the retrieval quality of specialized transformer models while being significantly faster and smaller.

Semble can be deployed as an MCP (Model Context Protocol) server for integration with agents like Claude Code, Cursor, and Codex, or used via command-line tools and bash integration. Users can search local repositories or remote git URLs, discover related code snippets, and track token savings across all searches. The tool requires minimal setup and works entirely locally, making it an accessible solution for agents that need instant access to specific code without reading entire files.

### kharp – k version 3 Language Interpreter in C# - (Hacker News)

https://github.com/ERufian/ksharp

# K# Summary

K# is a comprehensive implementation of the K programming language (version 3), a vector programming language from the APL family, written in C#. The project is currently in beta status with full core language implementation including native verbs, adverbs, functions, conditionals, I/O operations, and a Foreign Function Interface for .NET integration. All 1,549 unit tests pass with 100% success rate.

The project remains under active development with goals to improve debugging capabilities, parsing robustness, code simplification, and performance optimization. While the developer acknowledges that K# won't compete with native K implementations in raw performance due to reliance on the .NET VM and managed memory, it offers benefits like garbage collection, crash prevention, and access to the rich .NET ecosystem. The interpreter can be run directly via command line, and the project provides detailed documentation and examples for users of the K language.

### WriteUp: 16 Bytes of x86 that turn Matrix rain into sound - (Hacker News)

https://hellmood.111mb.de//wake_up_16b_writeup.html

# Summary

"Wake Up! 16b" is a 16-byte x86 assembly program that generates a Sierpinski fractal by exploiting video memory as both a calculation space and audio output. The code initializes DOS video mode, then uses a clever loop that reads, XORs, and writes values to memory while simultaneously sending those values to the PC speaker port. The XOR operation isolates Bit 1 of the data, which controls the speaker cone, while the remaining bits create visual ASCII characters on screen. This dual-purpose exploitation allows the geometric pattern of the fractal to simultaneously generate both a visual display and a sonic representation.

The program's ingenuity lies in its mathematical foundation: the iterative XOR operations across memory produce patterns that follow Lucas's Theorem and map directly to elementary cellular automata Rule 60. By stepping backward by 56 bytes instead of 16, the code modulates the audio frequency by one octave and shears the visual fractal diagonally across the screen into ten vertical columns. However, this extreme constraint makes the output environment-dependent, as the algorithm's results vary based on the initial memory state left by the BIOS across different machines and emulators.
