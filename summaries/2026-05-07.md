### Leading Open Source Author Calls for Verification over Trust in Software Supply Chains - (InfoQ)

https://www.infoq.com/news/2026/05/stenberg-curl-verification-trust/

# Summary

Daniel Stenberg, creator of the widely-used curl software component, argues that the software industry should shift from trusting well-known components to actively verifying them. He contends that even major projects like curl—which runs on billions of devices—remain vulnerable to compromise through various attack vectors, including malicious contributors, breached committers, or hacked distribution servers. To address this, curl implements extensive controls such as strict code reviews, compiler settings that treat warnings as errors, mandatory two-factor authentication, and continuous fuzzing, while also providing signed releases that independent users can verify.

Stenberg advocates for a broader verification ecosystem where multiple independent users verify software integrity rather than relying solely on reputation-based trust. This position aligns with industry lessons from the 2024 XZ Utils backdoor attack, which demonstrated how attackers can exploit trust in established maintainers. To facilitate verification, tools like Software Bill of Materials (SBOMs) are becoming essential—and increasingly mandatory—with regulations such as the EU Cyber Resilience Act and US Executive Order 14028 now requiring SBOM compliance for software procurement and sales.

### Attacker Bought 30 WordPress Plugins on Flippa and Backdoored All of Them - (InfoQ)

https://www.infoq.com/news/2026/05/wordpress-plugins-supply-chain/

# Summary

An attacker purchased 30 WordPress plugins from Flippa marketplace and installed a sophisticated backdoor in all of them, affecting over 400,000 installations. The malicious code remained dormant for eight months before activating in April 2026 to inject cloaked SEO spam into compromised websites. WordPress.org responded by permanently closing all 31 plugins and forcing auto-updates, though this did not fully clean previously compromised sites.

This attack highlights a critical vulnerability shared across all software package ecosystems—npm, PyPI, browser extensions, and others—where maintainers can transfer ownership without triggering additional code review or change-of-control notifications. The attacker exploited this structural weakness by inheriting commit access and the implicit trust of automatic updates. Similar supply chain attacks have occurred before (event-stream in 2018, XZ Utils in 2024), following the same pattern of building trust, gaining access, waiting, and striking. The attacker used sophisticated techniques like PHP deserialization backdoors and Ethereum smart contracts for command-and-control to evade traditional takedowns, demonstrating that the risk cannot be eliminated by switching platforms—only the vendor changes.

### Article: Beyond the Benchmark: A Metrics-Driven Approach to Sustained iOS Performance on Real Devices - (InfoQ)

https://www.infoq.com/articles/metrics-driven-approach-ios-performance/

# Summary

This article challenges the common practice of relying solely on isolated performance benchmarks to validate iOS applications, arguing that passing benchmarks does not guarantee real-world performance under sustained use. The author illustrates this with a real-world example of a cabin crew application that passed all performance metrics (cold start under 2 seconds, API latency under 400ms, zero crashes) yet froze during an 18-hour flight. The core issue is that point-in-time sampling misses cumulative degradation that occurs across extended sessions due to CPU load, memory state, thermal throttling, and OS lifecycle enforcement.

The article emphasizes that performance validation must be conducted on physical iOS devices rather than simulators, as simulators cannot reproduce thermal throttling, memory pressure, and battery dynamics that cause real failures. The author advocates for a metrics-driven methodology using Xcode Instruments profiling tools to monitor warm start latency, thermal states, memory management, and frame rates across extended 8-hour test protocols. Ultimately, the article argues that performance should be treated as an architectural requirement with system-level testing from the first sprint, rather than a post-development concern measured only through short benchmark windows.

### Grafana's Kubernetes Monitoring Helm Chart v4 Brings Multiple Fixes - (InfoQ)

https://www.infoq.com/news/2026/05/kubernetes-monitoring-helm/

# Summary

Grafana Labs released version 4 of its Kubernetes Monitoring Helm chart in April 2026, representing a major update after nearly six months of development aimed at solving pain points encountered by users managing large, complex deployments. The release introduces several structural improvements: destinations are converted from lists to maps for more reliable configuration management across multiple clusters and GitOps tools; collectors are restructured to remove hard-coded names and make feature routing explicit rather than hidden in source code; and backing services are now deployed explicitly rather than automatically, eliminating surprise duplicate deployments.

Additional refinements include reorganizing cluster metrics into three separate, focused features (clusterMetrics, hostMetrics, and costMetrics) for cleaner configuration, and significantly improving memory usage in the pod log pipeline by eliminating bulk label application in favor of explicit label declaration. Overall, the chart is described as "more predictable, more flexible, and much easier to maintain" whether managing a single cluster or hundreds.

By shifting from code-centric customization to data-driven configuration, GitHub enables organizations to encode knowledge about their own systems, reducing false positives and improving vulnerability detection accuracy. This approach aligns with broader industry trends toward making security practices more maintainable and scalable, allowing teams to adopt and adapt security models without specialized training while closing coverage gaps for custom frameworks and internal libraries.

### Figma Builds In-House Redis Proxy to Hit Six Nines Uptime - (InfoQ)

https://www.infoq.com/news/2026/05/figma-redis-figcache/

# Summary

Figma has developed an in-house Redis proxy service called FigCache to replace its fragmented caching infrastructure and achieve six nines of uptime. The system, in production since mid-2025, addresses scalability and reliability issues that arose from connection limit problems, thundering herds during service scale-ups, and inconsistent observability across multiple client libraries. Rather than adopting existing open-source Redis proxies, Figma built a custom solution to gain the semantic awareness needed for implementing runtime guardrails, custom command handling, and support for diverse client configurations.

FigCache is a stateless Go service built on ResPC (a custom Redis Serialization Protocol framework) that sits between client applications and AWS ElastiCache clusters. Its architecture separates frontend connection management from backend multiplexing, enabling extensibility. Notable design choices include using Starlark programs for runtime configuration and a fanout filter engine that transparently handles multi-shard operations that would normally return errors to clients, converting them into parallelized scatter-gather queries instead.

### How AI Swarms Are Disrupting Democracy - (Radar)

https://www.oreilly.com/radar/how-ai-swarms-are-disrupting-democracy/

# Summary

The article describes how artificial intelligence has transformed disinformation campaigns from labor-intensive troll farms into highly efficient "AI farms" capable of generating millions of pieces of synthetic content daily at industrial scale. Modern disinformation operations, such as Russia's CopyCop network, now use autonomous AI agents running on local servers to automatically generate propaganda, fake posts, and deepfakes across hundreds of websites without traceable watermarks or logs—making detection and attribution nearly impossible.

These "malicious AI swarms" represent an emerging threat to democracy by exploiting fundamental human cognitive biases. The coordinated agents create synthetic consensus by flooding platforms with seemingly independent but actually coordinated messages, triggering the "bandwagon effect" and "illusory truth" phenomena. When people see the same viewpoint repeated across multiple apparent sources and contexts, they perceive it as credible and widespread, making them more likely to adopt the position—all orchestrated by a single operator controlling thousands of AI-generated personas.
