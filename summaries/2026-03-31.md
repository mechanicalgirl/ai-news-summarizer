### Discord Open Sources Osprey Safety Rules Engine Processing 2.3 Million Rules per Second - (InfoQ)

https://www.infoq.com/news/2026/03/discord-osprey/

# Summary

Discord has open-sourced Osprey, a real-time safety rules engine capable of processing 2.3 million rules per second across 400 million daily actions. Built with a Rust coordinator and stateless Python worker nodes, Osprey evaluates JSON event payloads against dynamically loadable rules written in SML (a Python-like domain-specific language), then routes verdicts to configurable output sinks. The system uses Apache Kafka and Druid for data processing and has already been adopted by networks like Bluesky and Matrix.org.

The architecture exemplifies a modern polyglot pattern where Rust handles high-concurrency data plane operations while Python manages the control plane business logic and machine learning integrations. Osprey's scalability comes from stateless worker nodes that parse rules into an Abstract Syntax Tree at startup, support dynamic rule updates via ETCD, and can be horizontally scaled through containerization—allowing organizations to efficiently handle sustained event throughput and traffic spikes.

### Article: Optimization in Automated Driving: from Complexity to Real-Time Engineering - (InfoQ)

https://www.infoq.com/articles/optimization-in-automated-driving/

# Summary

This article explores the technical architecture and optimization challenges of autonomous vehicle (AV) systems, which must process massive amounts of sensor data (from LiDAR, cameras, radar, etc.) and convert it into safe control commands within millisecond deadlines. Rather than simple linear pipelines, AV stacks function as distributed, real-time feedback loops of perception, prediction, planning, and control using middleware like ROS 2, requiring sophisticated optimization techniques throughout.

The key optimization strategies discussed include context-aware sensor prioritization that dynamically adjusts sensing and processing effort based on driving scenarios (e.g., prioritizing forward-looking sensors on highways versus peripheral coverage in urban environments), Model Predictive Control (MPC) solvers for planning, and treating the compute budget itself as an engineering optimization problem. Rather than hard-coding rules, engineers define cost functions that solvers minimize, managing resources, time, and physical constraints simultaneously to balance accuracy, latency, and computational efficiency.

### FOSDEM 2026: Intro to WebTransport - the Next WebSocket?! - (InfoQ)

https://www.infoq.com/news/2026/03/fosdem-webtransport-vs-websocket/

# Summary

WebTransport is an emerging protocol presented at FOSDEM 2026 that aims to address performance limitations of WebSockets by leveraging HTTP/3 and QUIC. Unlike WebSockets, which rely on TCP and suffer from head-of-line blocking (where a single lost packet halts the entire stream), WebTransport supports both reliable byte streams and unreliable datagrams, enabling faster connections with reduced latency, transparent network switching, and connection establishment in as little as one round-trip time (RTT).

However, developer community reception has been mixed, with many questioning whether WebTransport is truly a WebSocket replacement or a specialized tool for specific high-performance use cases like cloud gaming and live streaming. Key concerns include the protocol's strict HTTPS requirement, which restricts its use in local network scenarios, and API incompatibility with WebSockets. While some developers view WebTransport as overengineered for simple messaging needs, others recognize the fundamental architectural differences that make it better suited for modern, low-latency applications.

# Summary

The most significant improvement moves test environment setup from setupFiles to the beforeAll hook, which fixes long-standing issues where mocks couldn't reliably intercept composables in middleware or plugins. This change requires some code migration—calls to Nuxt composables at the describe block level must now be moved inside beforeAll. Microsoft has launched the Azure Copilot Migration Agent, an AI-powered tool integrated into the Azure portal designed to streamline cloud migration planning and assessment. Organizations should view this as a planning enhancement to existing Azure Migrate workflows rather than a comprehensive migration solution, positioning it in competition with AWS Transform, which offers broader automation including execution capabilities.

### Discord Engineers Add Distributed Tracing to Elixir's Actor Model Without Performance Penalty - (InfoQ)

https://www.infoq.com/news/2026/03/discord-elixir-actor-tracing/

# Summary

Discord engineers developed a custom distributed tracing solution for their Elixir infrastructure that adds observability to the actor model without performance penalties. The solution introduces an "Envelope" primitive that wraps messages with trace context, allowing trace propagation between Elixir processes—a gap that existing tools like OpenTelemetry couldn't fill. The implementation supports gradual migration through backward compatibility, handling both instrumented and non-instrumented code simultaneously.

To handle Discord's massive scale, the team implemented dynamic sampling that adjusts based on fanout size and optimized context propagation to only include trace data for sampled operations, eliminating serialization overhead. Additional optimizations included preventing sessions from independently sampling fanned-out messages and implementing efficient trace flag parsing for gRPC requests. The solution proved its value during production incidents, revealing user-impacting delays that traditional metrics and logs alone couldn't expose, demonstrating that distributed tracing has become essential for debugging complex distributed systems at scale.

### "Pick and Mix" Custom Regions: Cloudflare Introduces Fine-Grained Data Residency Control - (InfoQ)

https://www.infoq.com/news/2026/03/cloudflare-custom-regions/

# Summary

Cloudflare has introduced Custom Regions, an extension of its Regional Services that allows customers to define precisely where their data is processed and stored. Unlike pre-defined geographic regions, Custom Regions enable customers to create arbitrary geographic groupings—such as specific countries, unique country combinations, or exclusions—to ensure TLS termination and application-layer processing occur only within chosen boundaries for compliance and data sovereignty purposes.

The feature operates through three key components: defining region membership using expressions (like country codes), selecting optimal in-region data center destinations, and enforcing boundaries at the edge. This approach differs from traditional cloud providers like AWS and Azure by leveraging Cloudflare's global edge network while still enabling regional constraints where compliance requirements demand it. Currently, Custom Regions requires manual setup through customer account teams rather than self-service configuration.

### Software, in a Time of Fear - (Radar)

https://www.oreilly.com/radar/software-in-a-time-of-fear/

# Summary

This O'Reilly article by Ed Lyons draws parallels between hiking difficult mountain trails and navigating the AI revolution in software development. Using his experience climbing Precipice Trail in Acadia National Park—a steep, dangerous-looking path that thousands safely traverse annually—Lyons argues that fear about AI replacing developers is often overblown and based on secondhand opinions rather than firsthand experience. He offers six key lessons for coping with technological change: stop listening to fearful people, seek direct evidence, find enthusiastic mentors, avoid dwelling on worst-case scenarios, adapt with new tools, and focus on the journey rather than the endpoint.

Lyons contends that while the future of software development amid AI coding agents is uncertain and challenging, the existential threat to developers is likely exaggerated. He advocates for embracing the climb ahead with realistic caution rather than paralyzing fear, encouraging developers to develop new skills and perspectives to remain relevant in an evolving industry.
