## https://www.infoq.com/news/2026/02/uber-uforwarder-kafka-push-proxy/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global

InfoQ

# Summary

Uber Engineering has open-sourced uForwarder, a push-based Kafka consumer proxy designed to improve scalability and efficiency for event-driven microservices. Rather than having individual services act as direct Kafka consumers, uForwarder serves as an intermediate layer using gRPC to manage message consumption, offset tracking, and delivery. This architecture addresses challenges that arose from Uber's massive internal Kafka deployment, which processes trillions of messages daily across 1,000+ consumer services.

The system introduces several key innovations to overcome limitations of standard Kafka consumer groups: context-aware routing that uses message headers for intelligent workload isolation and delivery; an out-of-order commit tracker that prevents head-of-line blocking by rerouting problematic messages to dead letter queues; an auto-rebalancer that dynamically redistributes partitions based on CPU, memory, and throughput metrics; and DelayProcessManager for partition-level backpressure handling. These features collectively reduce consumer lag, improve hardware utilization, and simplify consumer logic across microservices while providing better operational control at scale.

# TSSLint 3.0 Summary

TSSLint, a lightweight TypeScript linting tool positioned as a successor to the deprecated TSLint, has released version 3.0 as its "final major release." The tool distinguishes itself by running as a TypeScript Language Server plugin rather than as a separate process, enabling near-instant diagnostics and fixes without the overhead of tools like ESLint. Version 3.0 eliminates the esbuild dependency in favor of Node.js's native .ts file support, improving startup speed and reducing complexity—though it requires Node.js 22.6.0 or newer.

The release introduces several improvements including a TSL compatibility layer for unified rule handling, a new importTSLintRules feature to simplify migration from legacy TSLint configurations, and better developer tooling with automated documentation generation. However, developers migrating will encounter breaking changes such as renamed CLI parameters and the removal of the multi-threading option, which the team determined consumed twice the energy for only 1.5x speed gains. The tool remains incompatible with TypeScript 7's native Go compiler and focuses on type-aware analysis deeper than syntactic-only alternatives like Biome and oxlint.

# Summary

This InfoQ article presents a security framework for safely deploying AI agents in infrastructure automation by implementing a "Least-Privilege AI Agent Gateway" that acts as a control boundary between agents and infrastructure systems. The gateway uses three key components: Model Context Protocol (MCP) for agent communication, Open Policy Agent (OPA) for policy-based authorization, and ephemeral runners for isolated, short-lived execution environments. Rather than granting agents direct API access, every request is validated, authorized based on defined policies, and executed in isolated containers with full observability via OpenTelemetry.

The article emphasizes that autonomous AI agents pose unique security risks compared to traditional CI/CD automation because they make dynamic decisions across multiple systems with potentially broad permissions, creating audit and accountability challenges. The proposed solution treats agents as "untrusted requesters" whose actions must be authorized, constrained, observed, and contained through a dedicated governance layer, preventing misconfigurations or compromised agents from causing widespread infrastructure damage.

# Summary

Anthropic's randomized controlled trial of 52 junior engineers found that developers using AI coding assistants scored 17% lower on comprehension tests than those coding manually, with no statistically significant productivity gains. The study revealed a critical distinction: developers who used AI strategically for conceptual questions scored 65% or higher, while those delegating code generation entirely to AI scored below 40%, suggesting that *how* developers interact with AI matters far more than whether they use it.

The research highlights a concerning trade-off between short-term productivity and long-term skill development, particularly for learning new technologies. While independent academic research confirms these findings, experts debate whether the study measures learning new skills specifically rather than general programming ability. To address these concerns, major LLM providers now offer intentional "learning modes" designed to prioritize comprehension over delegation, recognizing that developers need debugging and validation skills to effectively oversee AI-generated code.

# Summary

Rivet has launched the Sandbox Agent SDK to address fragmentation in coding agent APIs. ## https://www.infoq.com/news/2026/02/aws-lambda-rds-trigger-events/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global

InfoQ

# Summary

AWS has introduced an event-driven pattern enabling Lambda functions to be triggered by Amazon RDS for SQL Server database events through CloudWatch Logs and SQS. The solution uses stored procedures that write formatted messages to the SQL Server error log via RAISERROR statements, which CloudWatch subscription filters detect and route through SNS topics to SQS queues that trigger Lambda functions. This approach decouples database operations from application logic, allowing databases to focus on data while specialized functions handle workflows and integrations.

The AWS approach differs from Azure's native SQL triggers by relying on log parsing rather than built-in change tracking, offering trade-offs in flexibility versus simplicity. The solution includes security considerations around plaintext data in logs and comes with a complete Cloud Development Kit template for implementation. This pattern supports event-driven architectures for use cases like data pipelines, real-time notifications, and microservice orchestration, with practitioners reporting significant cost reductions by eliminating polling and enabling selective, batch-based processing.

# .NET 11 Preview 1 Summary

Microsoft has released .NET 11 Preview 1, introducing significant updates scheduled for November 2026 as a Standard Term Support release. The headline feature is **Runtime Async**, a fundamental shift where the runtime itself now handles asynchronous method suspension and resumption instead of relying solely on compiler-generated state machines. Additional runtime improvements include foundational WebAssembly support (migrating from Mono), native Zstandard compression support via ZstandardStream, a new BFloat16 floating-point type for AI/ML workloads, and JIT enhancements for startup performance.

## https://www.infoq.com/news/2026/02/cloudflare-local-uploads/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global

InfoQ

# Summary

Cloudflare has launched Local Uploads for its R2 cloud storage service in open beta, introducing a feature that significantly reduces write latency for globally distributed users. The new capability allows data to be written to storage infrastructure closest to the client while automatically replicating it to the bucket's primary region in the background. According to Cloudflare's testing, this approach achieves up to 75% reduction in Time to Last Byte (TTLB) when upload requests originate from a different region than the bucket's location.

The system works by decoupling physical storage from logical metadata through Cloudflare's Durable Objects technology, ensuring data remains immediately accessible and consistent throughout the replication process. There is no additional cost to enable Local Uploads, as upload requests incur standard Class A operation costs. While AWS offers a similar solution through S3 Transfer Acceleration, Cloudflare's approach uses local writes with asynchronous replication, positioning it as a competitive option for workloads requiring fast, reliable uploads from multiple geographic regions without changing the bucket's primary location.
