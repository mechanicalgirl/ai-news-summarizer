### Microsoft Open-Sources PostgreSQL Extension for In-Database Durable Execution - (InfoQ)

https://www.infoq.com/news/2026/06/postgresql-pg-durable/

# Summary

Microsoft has open-sourced pg_durable, a PostgreSQL extension that enables durable workflows to execute natively within the database, eliminating the need for external orchestration systems like cron jobs, message queues, and schedulers. Developers can define long-running, fault-tolerant workflows directly in SQL, with the extension automatically handling retries, checkpointing, and recovery—allowing execution to resume from the last checkpoint if the database crashes or a step fails.

The extension uses a domain-specific language with special operators for sequential and parallel execution, and persists workflow state in PostgreSQL tables. Built on minimal architecture with only a Postgres extension and background worker (no external control plane), pg_durable is designed for use cases like vector embedding pipelines, scheduled maintenance tasks, and workflows dependent on external APIs. This approach to durable execution simplifies distributed systems by automating state recovery and eliminating complex manual intervention.

### IBM Vault Enterprise 2.0 Brings Automated LDAP Secrets Management to Enterprise Identity Security - (InfoQ)

https://www.infoq.com/news/2026/06/ibm-hashicorp-vault-ldap-secrets/

# Summary

IBM Vault Enterprise 2.0, the first major release under IBM's ownership of HashiCorp, introduces automated LDAP secrets management capabilities designed to address long-standing enterprise identity security challenges. The update migrates LDAP static roles into Vault's centralized rotation framework, enabling organizations to automate credential management, support password rotation, and automate identity lifecycles while reducing reliance on privileged administrative accounts.

A key architectural innovation is the "self-managed flow" model, which allows individual LDAP accounts to authenticate and rotate their own passwords under controlled policies rather than relying on highly privileged administrative accounts. The platform also supports automatic migration from legacy LDAP configurations, with upgrades designed to minimize disruption through background processing and dedicated monitoring APIs.

### Pinterest Uses Content Fingerprints for URL Deduplication across Millions of Domains - (InfoQ)

https://www.infoq.com/news/2026/06/pinterest-miqps-url-dedup/

# Summary

Pinterest engineers developed a URL normalization system called Minimal Important Query Param Set (MIQPS) to deduplicate content across millions of domains in their ingestion pipeline. Rather than relying on manual allowlists or domain-specific rules, MIQPS uses a data-driven approach that renders pages and generates content fingerprints to determine which query parameters affect page identity and should be retained versus which are non-essential tracking parameters that can be safely removed.

The system operates by collecting URLs from Pinterest's pipeline, grouping them by parameter patterns, and analyzing whether removing individual parameters changes the rendered content. If changes exceed a defined threshold, parameters are marked as important; otherwise they're classified as noise. This approach addresses the limitations of traditional methods—such as unreliable canonical tags—when scaling across heterogeneous domains with inconsistent URL structures. MIQPS separates expensive offline analysis from runtime processing, using precomputed parameter importance maps to streamline URL processing at scale while protecting against anomalous updates.

# Summary

This O'Reilly article by Gaurav Savla outlines a product manager's practical guide for successfully launching AI features in production environments. The author highlights that the gap between impressive AI demos and functional production systems—termed "Death Valley"—is primarily an engineering discipline problem rather than a model quality issue. Common production challenges include latency spikes, hallucinations on edge cases, ineffective A/B testing due to AI output variance, and numerous safety failures that traditional software engineering practices don't adequately address.

The article begins with concrete strategies for managing latency, the most critical production constraint. Rather than global latency measurements, Savla recommends setting specific budgets by interaction type: synchronous interactions (under 1 second), progressive token-streaming responses (first token under 500ms), and asynchronous interactions (up to 20 seconds). He emphasizes measuring p90 latency rather than p50, accounting for cold starts separately, optimizing the entire pipeline beyond just model inference, and leveraging streaming to improve perceived performance. This framework helps teams design AI features that actually meet user expectations in production.

The article suggests that visibility of these costs will force organizations to rethink how they design and manage agentic systems.

### Long-Running Agents - (Radar)

https://www.oreilly.com/radar/long-running-agents/

# Summary

This article discusses "long-running agents"—AI agents that can maintain progress on tasks over extended periods (hours, days, or weeks) across multiple sessions and sandboxes, unlike the current chat-based agent paradigm that typically operates within a single sitting. Long-running agents can recover from failures, leave structured records of their work, and resume tasks where they left off by maintaining state outside the model's context window.

The article distinguishes between three related but distinct concepts: long-horizon reasoning (planning over many dependent steps), long-running execution (the agent's process running for extended periods), and persistent agency (an agent identity that accumulates memory across tasks). The author argues this capability matters because it represents a phase change in what tasks can be economically delegated to AI, moving beyond the limitations of current single-session approaches where models forget context, declare tasks complete prematurely, or reintroduce previously fixed bugs.

### The AI Agents Stack (2026 Edition) - (Radar)

https://www.oreilly.com/radar/the-ai-agents-stack-2026-edition/

# Summary

This article presents an updated "AI Agents Stack" framework for 2026, replacing a 2024 reference diagram that has become outdated. The stack comprises six layers of infrastructure needed to move AI agents from development to production—covering state management, tool connectivity, memory, frameworks, and reasoning capabilities. The author emphasizes that the stack has evolved significantly due to three key developments: the standardization of tool connectivity through MCP (Model Context Protocol), the rise of reasoning models that reduce the need for multi-step chains, and memory becoming a first-class architectural priority rather than an afterthought.

The core principle is that teams should only add complexity when a specific problem breaks, starting with the simplest solution that solves their use case. Rather than viewing agents as merely scaled chatbots, the framework recognizes that production agents require fundamentally different infrastructure than inference-based systems—particularly for managing state across multi-step execution, governance of tool access, persistent memory, autonomous reasoning loops, and behavioral guardrails. When evaluating each layer, teams should assess their state management needs, acceptable vendor lock-in, and the effort required to move from prototype to production.
