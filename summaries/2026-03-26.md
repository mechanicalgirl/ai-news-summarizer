### Uber Launches IngestionNext: Streaming-First Data Lake Cuts Latency and Compute by 25% - (InfoQ)

https://www.infoq.com/news/2026/03/uber-streaming-date-lake/

# Summary

Uber has launched IngestionNext, a streaming-first data lake platform that replaces traditional batch processing with continuous event streaming. The new architecture uses Apache Kafka and Flink to process data streams in real-time before committing to the data lake, reducing ingestion latency from hours to minutes and improving data freshness for analytics and machine learning workloads.

The transition to streaming ingestion achieved significant performance improvements, including a 25% reduction in compute usage and faster data availability across the organization. However, the engineers faced technical challenges such as managing small file creation and ensuring data correctness in distributed systems, which they addressed through compaction mechanisms and robust checkpointing strategies. While IngestionNext improves raw data freshness, the team acknowledges that downstream transformation pipelines may still introduce additional latency, with future work focused on extending streaming capabilities throughout the entire analytics stack.

### AWS Load Balancer Controller Reaches GA with Kubernetes Gateway API Support - (InfoQ)

https://www.infoq.com/news/2026/03/aws-gateway-api-ga/

# Summary

AWS has released general availability support for Kubernetes Gateway API in its Load Balancer Controller, enabling teams to manage Application Load Balancers and Network Load Balancers through the standardized Gateway API specification. This replaces the outdated Ingress API and eliminates the previous approach of cramming configuration into Kubernetes annotations by providing type-safe Custom Resource Definitions with proper validation and GitOps-friendly YAML. The Gateway API also supports both Layer 4 (TCP, UDP, TLS) and Layer 7 (HTTP, gRPC) routes, with features including automatic TLS certificate discovery from AWS Certificate Manager and cross-namespace routing capabilities.

The release reflects broader industry adoption, with 20+ conformant controllers (including Google Cloud, NGINX, Envoy, and Istio) implementing the specification. The solution improves developer experience through clear role separation—platform teams define infrastructure templates, operators configure listeners and security, and developers create routes without needing cluster-admin access. While existing Ingress resources remain supported, new projects can benefit from superior validation, clearer error messages, and better integration with Kubernetes-native workflows.

### .NET 11 Preview 2 Updates MAUI with Performance Improvements and Platform Refinements - (InfoQ)

https://www.infoq.com/news/2026/03/net-11-preview2-maui/

# Summary

.NET 11 Preview 2 introduces targeted improvements to MAUI (Multi-platform App UI), focusing on reducing XAML verbosity, enhancing binding performance, and improving API consistency. Key updates include new TypeConverter support for the Map control that allows direct coordinate definition in XAML, enhanced visibility and click event controls for map shapes, and significant performance optimizations to TypedBinding and SourceGeneratedBinding that reduce execution time by ~25% and memory usage by approximately 50%.

The release also addresses developer pain points such as correcting nullable value type binding behavior in Entry.Text controls and formalizing immutability guarantees for commonly used types like Color and Font. While these changes are incremental rather than revolutionary, they reflect Microsoft's strategy of refining existing functionality based on developer feedback rather than introducing new surface area, with one notable breaking change to the VisualStateManager API that may require updates in existing code.

### QCon London 2026: Shielding the Core: Architecting Resilience with Multi-Layer Defenses - (InfoQ)

https://www.infoq.com/news/2026/03/shielding-the-core-parra/

# Summary

Anderson Parra from SeatGeek presented a multi-layered defense strategy for handling massive traffic spikes that can overwhelm infrastructure. The approach addresses three key challenges: the "traffic stampede" (traffic arriving faster than systems can adapt), the "noisy neighbor problem" (one tenant consuming shared resources), and the "scaling gap" (lag between demand and system scaling). SeatGeek implements a three-part strategy: Absorb the Burst, Control the Flow, and Protect the Core.

The defense system consists of three shields: the Edge Shield (caching, queuing, and bot filtering), the Gateway Shield (rate limiting, fair access controls, and request validation), and the Platform Shield (resource isolation, prioritization, and observability). The key to resilience is implementing early warning signals across all layers that trigger scaling responses before failures occur, combined with resource isolation to prevent cascading failures. Parra concluded that while traffic spikes are inevitable, system collapse is preventable through proper layered defenses and careful monitoring.

### Uber Automates Design Documentation with Agentic Systems - (InfoQ)

https://www.infoq.com/news/2026/03/uber-ai-design/

# Summary

Uber has developed uSpec, an agentic AI system that automates the creation of component design specifications, dramatically reducing documentation time from weeks to minutes. The system functions as a "Visual-to-Technical Spec" compiler, using AI agents integrated with Figma to extract design data and automatically generate technical specifications across Uber's seven platform stacks and three accessibility frameworks. The system encodes domain expertise through "Agent Skills"—structured files that handle platform-specific requirements like accessibility mappings (VoiceOver, TalkBack, ARIA) and density logic across SwiftUI, React, and Android Compose.

A key advantage of uSpec is its privacy-first architecture: all processing remains local on users' machines with no proprietary design data sent to cloud APIs. The system integrates with Uber's Michelangelo AI platform through a GenAI Gateway that provides security measures like PII redaction before requests reach external models. However, the article highlights an emerging architectural tension—while enterprises like Uber maintain Figma as their visual "source of truth," individual developers increasingly bypass design tools entirely, using AI to generate production-ready code directly from requirements.

### Revenium Unveils Tool Registry to Expose the True Cost of AI Agents - (InfoQ)

https://www.infoq.com/news/2026/03/revenium-ai-tooling-costs/

# Summary

Revenium has launched its Tool Registry, a platform designed to provide enterprises with comprehensive visibility into the true cost of AI agents by tracking all expenses beyond just token usage. While most organizations focus on LLM token costs (often less than 1% of total spend), real-world AI workflows incur significant expenses from external services like credit bureaus, identity verification, and fraud detection tools. For example, a loan origination workflow might cost only $0.30 in tokens but $50-85 total when including third-party service fees. The Tool Registry maps every expense—including API calls, SaaS platforms, and human review steps—back to specific AI decisions and business outcomes, enabling organizations to measure whether their AI investments actually deliver ROI.

The timing of the release reflects urgent industry needs: Gartner projects 40% of enterprise applications will include AI agents by end of 2026, yet Forrester reports organizations are deferring a quarter of planned AI spending due to ROI uncertainty. Unlike existing tools (Langfuse, LangSmith, Helicone) that primarily track token consumption and LLM metrics, Revenium's approach provides full-stack visibility by including human-in-the-loop costs and linking all spending to measurable business outcomes, addressing a critical gap in enterprise AI cost accountability.

### AI Coding Assistants Haven’t Sped up Delivery Because Coding Was Never the Bottleneck - (InfoQ)

https://www.infoq.com/news/2026/03/agoda-ai-code-bottleneck/

# Summary

While AI coding assistants have increased individual developer productivity, they haven't meaningfully accelerated overall project delivery because coding was never the actual bottleneck in software development. Research shows that teams with high AI adoption complete more tasks and merge more pull requests, but code review time increases by 91%, revealing that the constraint has shifted upstream to specification and verification—areas requiring human judgment that AI cannot replace.

This finding has significant implications for team structure and engineering workflows. Rather than minimizing communication overhead through small teams, the focus should shift toward collaborative specification and architectural alignment as the primary value-creating work. The article advocates for a "grey box" approach where engineers maintain accountability at two critical points: writing precise specifications that guide AI code generation and verifying results through evidence evaluation rather than line-by-line code inspection. This represents a fundamental migration of human authority from code implementation to defining and governing intent.

### QCon London 2026: Ethical AI is an Engineering Problem - (InfoQ)

https://www.infoq.com/news/2026/03/ethical-ai-problem/

# Summary

At QCon London 2026, Clara Higuera from BBVA presented the argument that ethical risks in AI systems are fundamentally engineering problems rather than purely governance or policy issues. Using real-world examples like wrongful arrests due to facial recognition failures, she demonstrated how ethical issues arise from technical and design choices—such as unrepresentative training data, lack of model explainability, and inadequate bias detection—made during development. Rather than treating ethics as an afterthought, engineers must embed ethical considerations throughout the AI lifecycle by evaluating datasets, measuring model behavior across demographic groups, and ensuring system observability.

The presentation emphasized that addressing key ethical dimensions—fairness, transparency, security, sustainability, and accountability—requires concrete engineering practices rather than abstract principles. By incorporating fairness evaluations, explainability analysis, security testing, and production monitoring early in system architecture, organizations can prevent ethical issues from emerging after deployment. Drawing parallels to how aviation and automotive industries evolved safety standards alongside technological advancement, Higuera suggested that AI is entering a similar phase where developers must proactively establish engineering practices for ethical AI, positioning themselves to build trustworthy systems in an environment where formal regulations have not yet been established.

### Article: Lessons from Adopting SwiftUI in an App with 50 Million Users - (InfoQ)

https://www.infoq.com/articles/adopting-swiftui-at-scale/

# Summary

This article shares practical lessons from adopting SwiftUI in a large-scale iOS app with 50 million users across a team of 20+ engineers. The key insight is that SwiftUI adoption isn't primarily a technical challenge—it's an organizational and cultural one. The author highlights critical lessons including: implementing SwiftUI only for new features rather than rewriting existing UIKit code, using feature flags to support multiple iOS versions simultaneously, redesigning the design system to leverage SwiftUI's capabilities, and treating adoption as a culture shift rather than a mandate. The article reveals that senior UIKit engineers were hesitant not due to fear of change, but because their existing solutions were stable and optimized, while newer engineers viewed UIKit as outdated.

The core recommendation is to avoid forcing SwiftUI adoption through technical mandates. Instead, teams should invite engineers to embrace it gradually, allowing senior developers to remain productive on critical systems while newer developers build experience with SwiftUI on lower-risk projects. This balanced approach prevents the team from becoming fragmented and ensures confident shipping to users across different iOS versions.

### Spotting and Avoiding ROT in Your Agentic AI - (Radar)

https://www.oreilly.com/radar/spotting-and-avoiding-rot-in-your-agentic-ai/

# Summary

The article draws a parallel between rogue traders in investment banking and autonomous AI agents, introducing the concept of "ROT" (Rogue Operator Threat) as a novel business risk. Just as rogue traders like Nick Leeson accumulated massive hidden losses by covering mistakes with new trades, AI agents operating with broad autonomy and insufficient oversight could inflict significant, sometimes catastrophic damage to their deploying companies before detection. The key difference is scope: while rogue traders are confined to investment banking, agentic AI will be deployed across numerous industries, potentially creating exponentially greater risks.

The article emphasizes that both rogue traders and AI agents are able to exploit loopholes in control systems because organizations prioritize operational speed over adequate safeguards. As companies rush to deploy autonomous AI with expansive permissions (evident in cases where bots have deleted emails or wiped production databases), they're creating conditions similar to those that enabled historic trading scandals—insufficient oversight, inadequate audit trails, and excessive operational freedom that allows problems to compound undetected until they cause severe financial and reputational damage.

### The Mythical Agent-Month - (Radar)

https://www.oreilly.com/radar/the-mythical-agent-month/

# Summary

This article by Wes McKinney explores how AI coding agents are changing software development and whether the lessons from Fred Brooks's classic book *The Mythical Man-Month* still apply. McKinney notes that while AI agents can dramatically increase productivity, the quality of the output depends entirely on human oversight—developers must carefully curate specifications, approve features, and manage the complexity that agents naturally generate. He warns that agentic development may suffer from the same pitfalls that have plagued software teams for decades: scope creep, architectural drift, and coordination overhead.

McKinney suggests that the future of software engineering will likely require talented developers acting as "orchestrators" of AI agent swarms, maintaining rigorous supervision over parallel agent sessions. The key challenge is whether the short-term productivity gains from AI agents will translate into long-term project success, or whether we'll simply create a new version of Brooks's "tar pit"—a messy tangle of code bloat and unnecessary complexity generated by unsupervised agents.

### The Missing Mechanisms of the Agentic Economy - (Radar)

https://www.oreilly.com/radar/the-missing-mechanisms-of-the-agentic-economy/

# Summary

Tim O'Reilly and economist Ilan Strauss outline their evolving work on AI regulation, starting with the AI Disclosures Project in 2024. They argue that regulators struggle to understand AI technology and that disclosure standards should focus not just on model safety but on how AI systems are actually deployed and operated. Drawing lessons from platforms like Google and Amazon—which initially advanced market coordination but later degraded service quality for profit—they advocate for transparency in the operating metrics that shape AI product behavior before these systems become exploitative.

The authors propose moving beyond traditional disclosure frameworks toward comprehensive mechanisms including protocols and market design. They emphasize that understanding AI's impact requires examining business models and real-world deployment, not just factory-level capabilities testing. Their work aims to establish shared understanding between regulators and industry through focused convenings, with the ultimate goal of building safeguards and market mechanisms that preserve the beneficial potential of AI systems before they become corrupted by misaligned incentives.
