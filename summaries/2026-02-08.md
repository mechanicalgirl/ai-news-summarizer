## https://simonwillison.net/2026/Feb/7/vouch/#atom-everything

# Summary

Mitchell Hashimoto has created Vouch, a new system designed to combat the growing problem of low-quality AI-generated pull requests flooding open source projects. The system works by requiring users to be "vouched" by existing contributors before they can contribute, while also allowing contributors to explicitly "denounce" problematic users. Users can be vouched or denounced through GitHub issues, discussions, or via a CLI tool.

The system is intentionally flexible and project-agnostic, integrating easily into GitHub through published GitHub Actions while remaining forge-independent. Hashimoto emphasizes that individual projects have full autonomy to define their own vouching criteria and community standards, rather than imposing a universal standard for what constitutes a legitimate contributor.

## https://simonwillison.net/2026/Feb/7/claude-fast-mode/#atom-everything

# Summary

Anthropic has launched a "fast mode" for Claude Opus 4.6 that delivers 2.5x faster response speeds but at significantly higher costs: $30/million input tokens and $150/million output tokens (compared to the standard $5/$25 rates). Until February 16th, a 50% discount brings this down to a 3x price multiplier, but after that discount expires, users will pay 6x more for the speed improvement.

Additionally, Claude Opus 4.6 now offers an extended context window of up to 1 million tokens (up from 200,000), with pricing that increases to $10/million input and $37.50/million output tokens once input exceeds 200,000 tokens. These premium pricing tiers also apply to fast mode, potentially resulting in extremely high costs for users combining both speed and extended context features.

## https://simonwillison.net/2026/Feb/7/software-factory/#atom-everything

# Summary

StrongDM's AI team has pioneered a "Software Factory" approach where AI coding agents write, test, and validate software without human code review—a radical strategy made possible by recent advances in LLM reliability (Claude 3.5 Sonnet in late 2024). Rather than relying on traditional testing, they solve the critical problem of ensuring untested code works through "scenario testing" (treating test cases as holdout sets that agents cannot see) and a "Digital Twin Universe"—behavioral clones of third-party services like Okta and Slack that allow agents to validate software at massive scale without hitting rate limits or API costs.

The approach demonstrates how AI agents can even build high-fidelity clones of complex SaaS applications, something previously economically infeasible. However, the strategy comes with significant costs—around $1,000 per engineer per day in token usage—raising questions about economic sustainability. The article suggests this represents one potential future of software development, where engineers transition from writing code to building and monitoring the systems that generate code, though the viability of these patterns at lower costs remains an open question.

## https://simonwillison.net/2026/Feb/6/pydantic-monty/#atom-everything

# Summary

Pydantic's Monty is a lightweight sandbox that runs a Python-like subset in Rust, offering a faster and simpler alternative to container-based sandboxes for executing LLM-generated code. It supports basic Python operations with microsecond startup times while blocking access to the filesystem, environment variables, and network—allowing developers to safely control what functions untrusted code can call.

Simon Willison successfully compiled Monty to WebAssembly in two forms: a standalone WASM module callable from JavaScript, and a Python wheel compatible with Pyodide. He created interactive browser demos for both approaches, showcasing how easily Rust code can be transformed into cross-platform WebAssembly that runs in both browsers and Python environments.

## https://simonwillison.net/2026/Feb/6/an-update-on-heroku/#atom-everything

# Summary

Heroku announced it is transitioning to a "sustaining engineering model" focused on stability and maintenance rather than new features, signaling that parent company Salesforce is deprioritizing the platform. The company's vague explanation that they're "focusing investments on areas where we can deliver the greatest long-term customer value" (including enterprise AI) provides little clarity about what this actually means for existing customers, frustrating observers with its lack of transparency.

The author, who still runs his blog on Heroku, interprets this as a sign the platform is being neglected and plans to migrate away from it to an alternative service like Fly, suggesting that customers should similarly prepare for Heroku's potential decline under Salesforce's ownership.
