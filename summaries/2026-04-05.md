### Absurd In Production - (Armin Ronacher's Thoughts and Writings)

https://lucumr.pocoo.org/2026/4/4/absurd-in-production/

# Summary

Armin Ronacher reports on five months of running Absurd in production, a durable execution system built entirely on PostgreSQL that requires only a SQL file and thin SDKs rather than separate services or complex infrastructure. The core design has proven sound, with improvements focused on practical production needs like better step handling, task result inspection, a CLI tool (absurdctl), and a web dashboard (Habitat). Key strengths include the minimal SDK size (1,400-1,900 lines vs. competitors' 170,000+), a checkpoint-based replay model that doesn't require deterministic code, and pull-based task scheduling that eliminates coordinator complexity.

The main gaps remaining are a built-in scheduler for cron-like behavior, a push model for webhooks, and PostgreSQL table partitioning support for efficient data cleanup. Ronacher also reflects on the broader question of open-source durable execution libraries in an era of AI agents, concluding that while such libraries may not support standalone companies, they're complex enough to warrant quality open-source implementations with proper tooling and ecosystem support.

### Can JavaScript Escape a CSP Meta Tag Inside an Iframe? - (Simon Willison's Weblog)

https://simonwillison.net/2026/Apr/3/test-csp-iframe-escape/#atom-everything

# Summary

A researcher testing sandbox security for a Claude Artifacts clone discovered that Content Security Policy (CSP) meta tags injected into sandboxed iframes cannot be escaped or disabled by JavaScript running within them. Through extensive testing on Chromium and Firefox, they confirmed that CSP policies defined via meta tags are enforced at parse time and persist even when JavaScript attempts to remove, modify, or replace them—or when the iframe navigates to a data URI.

This finding is practically significant because it means developers can apply CSP protections to untrusted content in sandboxed iframes without needing to host files on a separate domain. The meta tag approach provides a secure alternative for enforcing security policies on dynamically injected iframe content.

### The Axios supply chain attack used individually targeted social engineering - (Simon Willison's Weblog)

https://simonwillison.net/2026/Apr/3/supply-chain-social-engineering/#atom-everything

# Summary

The Axios team experienced a sophisticated supply chain attack where malware was distributed through a compromised dependency. The attack involved a highly coordinated social engineering campaign targeting a specific maintainer, Jason Saayman, who was deceived by attackers impersonating a company founder. The attackers created a convincing fake Slack workspace with cloned branding, fake team profiles of other OSS maintainers, and scheduled a Microsoft Teams meeting where they tricked Saayman into installing malware disguised as a missing software update needed for the meeting—ultimately giving attackers remote access to steal his credentials.

This attack demonstrates the vulnerability of open source maintainers to well-planned social engineering schemes that exploit time pressure and trust. Simon Willison emphasizes that this tactic mirrors documented threat strategies and that all maintainers of widely-used open source software should be aware of this attack vector, as the combination of professional execution, compelling fake credentials, and meeting-time urgency makes it particularly effective.

### Highlights from my conversation about agentic engineering on Lenny's Podcast - (Simon Willison's Weblog)

https://simonwillison.net/2026/Apr/2/lennys-podcast/#atom-everything

# Summary

Simon Willison discusses how AI coding agents have reached an inflection point in late 2024, with models like GPT-5.1 and Claude Opus 4.5 now producing working code reliably enough to enable autonomous agents. This shift fundamentally changes software engineering work: code generation that previously took weeks now takes hours, testing has become the new bottleneck, and rapid prototyping is now accessible to anyone. However, this transformation is mentally exhausting and requires significant skill to manage effectively, with concerns about burnout as developers continuously offload more work to agents.

The broader implications extend beyond software engineering. Willison argues that developers are "bellwethers" for other information workers since code has clear right/wrong outcomes, making it easier to evaluate AI quality than in fields like law or journalism. As productivity explodes, the field must grapple with new challenges: estimation becomes impossible, prototyping loses its value as a differentiator, and mid-level engineers face particular challenges since AI benefits both senior engineers (who leverage it with experience) and juniors (who learn from it), while potentially displacing the middle tier.

### Gemma 4: Byte for byte, the most capable open models - (Simon Willison's Weblog)

https://simonwillison.net/2026/Apr/2/gemma-4/#atom-everything

# Summary

Google DeepMind has released Gemma 4, a new suite of open-source, vision-capable reasoning models in sizes of 2B, 4B, 31B, and a 26B-A4B Mixture-of-Experts variant, all under the Apache 2.0 license. The models are designed for exceptional "intelligence-per-parameter" efficiency, with the smaller E2B and E4B models using Per-Layer Embeddings technology to maximize on-device deployment capabilities. Beyond vision, these models uniquely support multi-modal inputs including video, images, and audio for speech recognition.

The author tested the models locally using LM Studio and via Google's API, finding that most performed well, though the 31B local version had issues. The models demonstrate strong visual understanding capabilities across tasks like OCR and chart comprehension. Google is providing API access to the larger models through AI Studio, making this a significant release in the growing trend of creating highly efficient, capable small language models.

### Supply Chain Attack on Axios Pulls Malicious Dependency from npm - (Simon Willison's Weblog)

https://simonwillison.net/2026/Mar/31/supply-chain-attack-on-axios/#atom-everything

# Summary

A supply chain attack targeted Axios, a popular HTTP client library with 101 million weekly downloads, by injecting a malicious dependency called plain-crypto-js into versions 1.14.1 and 0.30.4. The malware was designed to steal credentials and install a remote access trojan (RAT), and appears to have been deployed using a compromised npm authentication token.

The attack highlights the importance of npm's trusted publishing feature, which would restrict package publication to verified GitHub Actions workflows. Security researchers noted that the malicious packages lacked accompanying GitHub releases—a red flag also observed in the recent LiteLLM attack—suggesting this could serve as a useful heuristic for identifying potentially malicious releases.
