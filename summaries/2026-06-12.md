# Summary

A senior staff engineer writes to advice columnist Charity Majors asking how to convince leadership to stop overloading engineering teams with unsustainable workloads. Majors explains that "speed" is merely a symptom of deeper problems and recommends replacing vague complaints about slowness with specific, textured vocabulary like "saturation" and "clock rate" to enable more nuanced conversations about tradeoffs. She emphasizes that urgency and emotional pressure are poor long-term strategies, and that systems running at 100% capacity actually slow down—like rush-hour traffic—because they lack the flexibility needed to handle unexpected issues and crises.

Majors advises the engineer to stop arguing for "slowing down" and instead present concrete evidence of what works: the engineer already has proof that when workload decreases slightly, team velocity actually *increases* and people do more interesting work. Rather than making familiar complaints that leaders tune out, presenting this data-backed solution would be far more persuasive. She acknowledges the problem may never be fully solved, but strategic communication and demonstrating structural improvements offer the best path forward.

As AI tools make code generation nearly instantaneous and affordable, engineering leadership must fundamentally shift focus from coding speed to human judgment and customer value. ### CRISPR tech selectively shreds cancer cells, including "undruggable" cancers - (Hacker News)

https://innovativegenomics.org/news/crispr-technique-selectively-shreds-cancer-cells/

# Summary

Researchers at UC Berkeley's Innovative Genomics Institute have developed a novel CRISPR-based technique that selectively destroys cancer cells carrying mutations in the p53 tumor suppressor gene, which is found in nearly half of all cancers and up to 70-90% of difficult-to-treat cancers like ovarian, pancreatic, and lung cancer. Rather than attempting to fix the broken tumor suppressor protein—a challenge that has eluded drug developers—the team engineered a CRISPR-Cas12a2 system that detects cancer-specific RNA signatures and triggers "chromatin shredding," which demolishes the genetic material of mutated cells while leaving healthy cells untouched.

The innovation returns CRISPR to its natural defensive function as a cellular destroyer rather than a fixer, and lab tests confirmed the system can precisely distinguish between healthy and cancerous cells, initiating cell death only when the specific mutation is present. The approach is also programmable, allowing researchers to quickly adapt it to target new cancer mutations faster than traditional drug development methods. However, the team must still overcome delivery challenges to make the therapy viable for real-world clinical use.

### I Am Not a Reverse Centaur - (Hacker News)

https://blog.miguelgrinberg.com/post/i-am-not-a-reverse-centaur

# Summary

Miguel Grinberg, an open-source developer, refuses to become a "reverse centaur"—a term Cory Doctorow uses to describe people reduced to merely reviewing machine-generated code. Grinberg has noticed an influx of low-effort, LLM-generated pull requests to his projects and argues they waste his time since contributors often lack genuine interest in improving the projects. To combat this trend, he now requires all contributors to first discuss their proposed changes in an issue before submitting pull requests, ensuring human engagement and reducing meaningless code reviews.

Beyond this policy change, Grinberg expresses deeper concerns about open source's future and whether coding itself matters anymore. He worries that as LLMs become prevalent, fewer people will engage in the creative challenge of coding, potentially creating a world where machines generate all code and humans merely supervise—a future he firmly opposes. While he continues maintaining his existing projects, he's become reluctant to make new projects public, reflecting a broader disengagement with open-source culture.

### A PDF that changes based on how its read - (Hacker News)

https://sgaud.com/texts/pdf

# Summary

This article introduces "Adaptive PDFs," a technique that allows a single PDF file to present differently to humans and machines. By leveraging an existing PDF specification property (available since 2001) that enables replacement text for marked content, the author embeds clean markdown structure into PDFs that renderers ignore but text extractors recognize. The result is a file that looks like a normal, formatted PDF to human readers while machines like language models extract perfectly structured markdown with proper headings, lists, and tables—eliminating the guesswork currently required when parsing untagged PDFs.

The approach adds minimal file size overhead (typically single-digit percentages) and works with existing PDF tools without requiring new file formats. Benchmark testing shows that while token counts remain roughly equivalent, the information density per token increases significantly because structure is now explicit rather than inferred. This creates truly adaptive documents that automatically serve the appropriate format based on who is reading them, eliminating the need for separate conversion steps or multiple file versions.

### Looking Forward to Postgres 19: It's About Time - (Hacker News)

https://www.pgedge.com/blog/looking-forward-to-postgres-19-its-about-time

# Summary

Postgres 19 is introducing native temporal table support, finally implementing the SQL:2011 standard that other databases adopted years ago. This feature allows developers to easily track how data changes over time by using range types with a `WITHOUT OVERLAPS` constraint in primary keys, eliminating the need for complex workarounds like the btree_gist extension and exclusion constraints that previously required manual row splitting and stitching.

The new temporal syntax is intuitive and built directly into the database engine. Instead of managing separate `valid_from` and `valid_to` columns, developers can now use a single range column and leverage the `FOR PORTION OF` syntax for updates, allowing precise time-based modifications without manual data manipulation. While Postgres 19 currently supports application-time temporal tables (tracking when facts were true in the real world), full bitemporal support that also tracks transaction-time (when the database recorded changes) remains for future enhancement.
