### Pattern Screamer: Subnet Discovery in Networks with Unknown Addressing - (Python Weekly)

https://ifritnoises.org/articles/pattern-screamer/

# Summary

This article introduces "Pattern Screamer," a tool for discovering network subnets and routing infrastructure when network addressing is unknown. Rather than exhaustively scanning large address spaces (which is slow and disruptive), the tool uses TTL-based traceroute techniques with heuristic sampling: it divides large CIDR blocks into /24 subnets and probes only the most likely gateway addresses (.1 and .254) within each subnet. This reduces probe targets significantly (e.g., from 16+ million addresses in 10.0.0.0/8 to manageable numbers) while building a network topology graph.

The tool supports multiple protocols—ICMP Echo, ICMP Timestamp, and TCP—to bypass filtering mechanisms that commonly block certain traffic types. By using TTL expiration to trigger ICMP Time Exceeded responses from routers along the path, the approach reveals network infrastructure without excessive scanning. The article demonstrates the technique on a lab network and discusses how different protocols behave under various filtering conditions, making the multi-protocol approach more effective for infrastructure reconnaissance in real-world scenarios.

### Agentic RL: Token-In, Token-Out Done Right - (Python Weekly)

https://qgallouedec-tito.hf.space/

# Summary

This article discusses a critical bug in multi-turn reinforcement learning (RL) for language models with tool-use capabilities. The core problem is the **Token-In, Token-Out (TITO) invariant violation**: when training code decodes model-generated tokens to detect tool calls, then re-encodes them for the next turn, it can produce different token sequences than what the model originally sampled—because tokenization isn't perfectly reversible. This causes gradients to be computed on tokens the model never actually produced, breaking the math silently and creating training instability.

The solution is elegantly simple: maintain a single buffer of sampled tokens throughout the rollout without ever re-encoding them. Parse the tokens only for routing purposes (detecting tool calls), but never feed the parsed information back into prompts. This preserves token boundaries automatically and ensures gradients are always computed on the exact sequences the policy generated. The article notes this fix only requires chat templates to be "prefix-preserving for tool messages"—a property most existing templates already satisfy.

### Breaking Circular Imports in Python Without Losing Type Safety - (Python Weekly)

https://www.orcaset.com/blog/breaking-circular-imports-in-python-without-losing-type-safety

# Summary

Orcaset is a Python framework for building financial models where line items are represented as functions that map dates to values. While the framework supports direct referencing of dependencies between line items (like interest depending on debt), this can create circular import problems when line items are defined in separate files. To solve this without losing type safety, Orcaset uses **local imports within function bodies**—importing dependencies only when the function is called, after all modules are fully defined. This breaks the circular dependency chain while maintaining static type checking capabilities and clear dependency tracing.

Though local imports are generally discouraged in Python, Orcaset finds them preferable to alternatives because they keep dependencies explicit and co-located with their usage, avoid runtime errors by catching incomplete models at write time, and remain terser than manifest-based systems. The overhead is negligible since Python caches imports and Orcaset caches generators, making this approach practical despite its unconventional appearance.

### Django: introducing django-integrity-policy - (Python Weekly)

https://adamj.eu/tech/2026/05/31/introducing-django-integrity-policy/

# Summary

Adam Johnson introduces django-integrity-policy, a new Django middleware package that implements the Integrity-Policy security header—a browser feature (supported in Firefox 145+ and Chrome 138+) that requires all scripts and stylesheets to include integrity attributes containing content hashes. This header helps prevent supply chain attacks by blocking resources that don't match their expected hash values. The package allows developers to easily configure which resource types require integrity attributes through simple Django settings, similar to Johnson's earlier django-permissions-policy package.

To use the header effectively, developers must add integrity attributes to all their static files, including first-party resources—a task made easier with the django-sri package's template tags. Johnson notes that he created the package using Claude AI with just two prompts, leveraging an existing repository as a template, which he found to be an efficient approach for this straightforward, specification-driven task. The Integrity-Policy header is part of the broader Web Application Integrity, Consistency and Transparency (WAICT) initiative to bring code-signing-level security to the web.

### How Servers Work: A Hands-On Introduction to TCP Sockets - (Python Weekly)

https://labs.iximiuz.com/tutorials/how-servers-work-tcp-sockets

# Summary

This tutorial introduces TCP sockets and network programming fundamentals for backend and DevOps engineers. It explains that a TCP server is an operating system process that waits for client connections and exchanges data over a network using the TCP protocol. Rather than understanding specific application protocols like HTTP or SSH, the article emphasizes that TCP operates at a lower level, providing a reliable, ordered byte stream between two endpoints—meaning all servers (HTTP, database, cache, etc.) follow a similar underlying workflow: create a socket, bind it to an address and port, listen for connections, accept clients, and read/write data.

The article further covers how network communication works through chunking (breaking large data into smaller packets for transmission), packet reassembly (reassembling out-of-order packets at the destination), and TCP's role in ensuring reliable, ordered delivery. It uses the analogy of real-world sockets to explain this operating system abstraction for inter-process communication, and notes that while socket families vary (IPv4, IPv6, Unix domain), the fundamental workflow remains consistent regardless of the socket type used.

### PyTorch Custom Operation - (Python Weekly)

https://leimao.github.io/blog/PyTorch-Custom-Operation/

# Summary

This article explains how to implement custom PyTorch operations in C++ and CUDA for use in both Python and compiled inference programs. It covers two main approaches: **custom functions** (stateless operations registered with `TORCH_LIBRARY_IMPL`) and **custom classes** (stateful classes with parameters that inherit from `torch::CustomClassHolder`). The author demonstrates both using an identity convolution example, showing how to implement CPU and CUDA versions that PyTorch automatically dispatches to based on tensor device type.

The article also covers the practical implementation details: compiling C++/CUDA code into a shared library (`libidentity_conv_ops.so`), loading it with `torch.ops.load_library`, and registering "fake" abstract versions of the custom operations using `@register_fake_class` and `@torch.library.register_fake`. These fake versions enable compatibility with `torch.compile` and `torch.export` by allowing symbolic tracing with FakeTensors without executing actual C++/CUDA code during tracing.

### Django security releases issued: 6.0.6 and 5.2.15 - (Python Weekly)

https://www.djangoproject.com/weblog/2026/jun/03/security-releases/

# Summary

The Django team has released security updates for versions 6.0.6 and 5.2.15 to address five low-severity vulnerabilities. These issues include: a signed cookie salt namespace collision that could allow cookies to be accepted in unintended contexts; potential unencrypted email transmission when STARTTLS handshakes fail; improper caching of responses marked with private Cache-Control directives when using mixed case; missing Authorization header variance in cache keys; and incorrect caching of responses with whitespace-padded Vary headers. All users are encouraged to upgrade immediately.

The patches have been applied to Django's main, 6.1 (alpha), 6.0, and 5.2 branches, with downloadable releases available for versions 6.0.6 and 5.2.15. The vulnerabilities were reported by external security researchers and disclosed responsibly through Django's security reporting process.

### AI Engineering for Developers - (Python Weekly)

https://www.lucavall.in/blog/ai-engineering-for-developers

# Summary

This article is a guide to AI engineering for experienced software developers, written by Luca Cavallin in May 2026. It introduces the three-layer AI engineering stack (application, model development, and infrastructure layers) and explains that AI engineering differs from ML engineering because it focuses on building systems around pretrained foundation models rather than training models from scratch. The author argues that most teams should start with prompt engineering and retrieval-augmented generation (RAG) before considering more costly approaches like finetuning, and provides practical guidance on selecting models, planning AI features with explicit quality checkpoints, and understanding the unique challenges of developing, deploying, and maintaining AI systems.

The article emphasizes that foundation models work best for tasks involving unstructured-to-structured transformations with noise tolerance—such as customer support, code assistance, document automation, and search—but are unreliable for tasks requiring precision or real-time accuracy. Key challenges include non-deterministic outputs that can't be unit-tested, expensive inference requiring optimization through caching and routing, and ongoing maintenance as models and tokenizers evolve. The core insight is that the product isn't the model itself but the engineered system around it, requiring backend engineers to add three new responsibilities: keeping systems grounded through tools and retrieval, evaluating quality continuously, and optimizing for cost and speed.
