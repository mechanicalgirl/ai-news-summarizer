### TurboQuant: Redefining AI efficiency with extreme compression - (Hacker News)

https://research.google/blog/turboquant-redefining-ai-efficiency-with-extreme-compression/

# Summary

TurboQuant is a new compression algorithm that dramatically reduces the size of high-dimensional vectors used in AI models while maintaining accuracy. The technique addresses a critical bottleneck in large language models by compressing key-value caches—high-speed memory structures that store frequently accessed information—without the memory overhead that typically accompanies traditional vector quantization methods.

TurboQuant works in two stages: first, it randomly rotates data vectors and applies a high-quality quantizer (called PolarQuant) that converts standard coordinates into polar coordinates to eliminate normalization overhead; second, it uses a 1-bit error-correction algorithm (QJL) based on the Johnson-Lindenstrauss Transform to eliminate any remaining bias with virtually no additional memory cost. Testing across multiple benchmarks shows that all three techniques successfully reduce memory requirements for vector search and AI model inference without sacrificing performance.

### VitruvianOS – Desktop Linux Inspired by the BeOS - (Hacker News)

https://v-os.dev

# VitruvianOS Summary

VitruvianOS (V\OS) is a Linux-based operating system inspired by BeOS that prioritizes user experience, simplicity, and control. Built on the KISS principle, it features a fast and intuitive desktop environment that works seamlessly out-of-the-box without requiring configuration. The OS is completely free and open source, emphasizes user privacy by collecting no data, and includes a custom "Nexus Kernel Bridge" that enables BeOS/Haiku applications to run on standard Linux kernels.

The project is community-driven and distinguishes itself by putting users in control while maintaining the elegance and performance of classic operating systems alongside modern Linux capabilities. V\OS supports multiple file systems (XFS and SquashFS) and ships with real-time kernel patches for optimal responsiveness, with plans for future features like file system indexing and multiuser support.

### Show HN: I took back Video.js after 16 years and we rewrote it to be 88% smaller - (Hacker News)

https://videojs.org/blog/videojs-v10-beta-hello-world-again

# Summary

Video.js v10 Beta represents a major ground-up rewrite of the popular open-source video player, developed collaboratively with other leading projects like Plyr, Vidstack, and Media Chrome. The update modernizes the codebase for contemporary web development practices and introduces dramatically reduced bundle sizes—88% smaller than v8 by default, with even greater reductions when using the new Streaming Processor Framework (SPF) for adaptive bitrate streaming. The rewrite emphasizes deep customization support for React, TypeScript, and Tailwind CSS, while maintaining beautiful defaults and improved performance.

The new architecture enables unprecedented flexibility and modularity, allowing developers to build lightweight players with only required features included in the bundle—a simple React "hello world" example weighs just under 5KB gzipped. V10 splits the monolithic player into optional, swappable State, UI, and Media components connected through API contracts, and introduces "eject" functionality similar to shadcn/ui that lets developers access and customize full source code. The player maintains compatibility with existing streaming engines while offering purpose-built, smaller alternatives for common use cases.

### Tell HN: Litellm 1.82.7 and 1.82.8 on PyPI are compromised - (Hacker News)

https://github.com/BerriAI/litellm/issues/24512

# Summary

**litellm version 1.82.8 on PyPI contains a critical supply chain compromise: a malicious `.pth` file (litellm_init.pth) that automatically executes on Python startup without requiring any import statement. The payload is a credential-stealing script that collects sensitive data including environment variables, SSH keys, cloud credentials (AWS, GCP, Azure), Kubernetes secrets, Docker configs, crypto wallets, and shell history, then encrypts and exfiltrates it to an attacker-controlled server at `https://models.litellm.cloud/`.**

**The attack impacts anyone who installed litellm 1.82.8, affecting local machines, CI/CD pipelines, Docker containers, and production servers. Immediate actions include yanking the package from PyPI, checking for the malicious file in site-packages directories, and rotating all potentially compromised credentials across affected systems. The BerriAI team must audit their PyPI publishing credentials and CI/CD pipeline for evidence of compromise.**

### Social media bans and digital curfews to be trialled on UK teenagers - (Hacker News)

https://www.bbc.com/news/articles/cn89g3ngkyzo

# Summary

The UK government is launching a pilot program in which 300 teenagers will trial various social media restrictions, including complete app bans, overnight digital curfews, and one-hour daily limits, with a control group experiencing no changes. The trial aims to gather real-world evidence on the practical effects and feasibility of these interventions before the government decides whether to follow Australia's lead in banning social media access for under-16s. Technology Secretary Liz Kendall stated the pilots will provide evidence to inform future policy decisions, with participants and families interviewed before and after to assess impacts on family life, sleep, and schoolwork.

The initiative is supported by some children's charities and has broad political backing, though experts remain divided—some warn restrictions could be easily circumvented or drive youth to harmful online spaces, while others argue tech companies should simply be required to make platforms safer rather than banning them outright. The government's parallel consultation on a potential social media ban will continue until May 26, and the results will be complemented by an independent scientific study funded by the Wellcome Trust involving 4,000 students aged 12-15 to assess social media's actual impact on adolescent wellbeing.

### Arm AGI CPU - (Hacker News)

https://newsroom.arm.com/blog/introducing-arm-agi-cpu

# Summary

Arm has announced the Arm AGI CPU, its first production-ready silicon product designed specifically for agentic AI infrastructure in data centers. Built on the Arm Neoverse platform, the processor is engineered to handle continuous, large-scale AI workloads where software agents coordinate tasks in real time. A single rack configuration can deliver over 8,160 cores with more than 2x the performance per rack compared to latest x86 systems, thanks to Arm's superior memory bandwidth and efficient core architecture.

The announcement marks a significant milestone as Arm extends beyond licensing its IP to become a silicon manufacturer. Major tech companies including Meta (lead co-developer), OpenAI, Google, Microsoft, and others are already deploying or planning to deploy the Arm AGI CPU, with commercial systems available for order from ASRockRack, Lenovo, and Supermicro. Arm plans to contribute the reference server design to the Open Compute Project to accelerate ecosystem adoption and standardization.
