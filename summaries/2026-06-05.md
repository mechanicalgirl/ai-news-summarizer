### Fine-tuning an LLM to write docs like it's 1995 - (Hacker News)

https://passo.uno/fine-tuning-docs-llm/

# Summary

The author experiments with fine-tuning local language models to write technical documentation in the style of 1990s Microsoft manuals. Using a corpus of 37 million words from Bitsavers (a digital archive of old computer manuals), they trained multiple models (Llama and Qwen) with varying parameters using QLoRA (a memory-efficient fine-tuning technique) on Runpod's GPU infrastructure, spending approximately $50 total.

The results demonstrate successful style transfer: fine-tuned models produced period-appropriate documentation structures with 90s terminology and formatting conventions, rather than modern markdown. The most impressive result was a 7B Qwen model trained on 192k examples that generated a convincing REST API explanation resembling genuine Windows 2000 Resource Kit material. However, the effectiveness varied by base model—Qwen performed better than Llama at maintaining the historical style, while base (non-instruct) models failed completely, confirming that fine-tuning works best on instruction-trained models.

### Anthropic's open-source framework for AI-powered vulnerability discovery - (Hacker News)

https://github.com/anthropics/defending-code-reference-harness

# Summary

This GitHub repository contains a **reference implementation for autonomous vulnerability discovery and remediation using Claude AI**. It provides both interactive skills (threat modeling, vulnerability scanning, triage, and patching) and an autonomous pipeline for finding and fixing security vulnerabilities in code. The tools are designed to work with various programming languages and vulnerability types, with a current focus on C/C++ memory vulnerabilities.

The repository includes a structured ramp-up plan for security teams, starting with Day 1 interactive skills for threat modeling and static scanning, progressing to Day 2 autonomous pipeline runs on known vulnerable code, and extending to customization for specific tech stacks. All interactive skills are safe to run unsandboxed, while the autonomous pipeline executes in a gVisor sandbox for isolation. The authors emphasize that teams should start small and iterate quickly rather than spending months designing perfect pipelines.

### Leap in DNA synthesis slashes time to build new genetic sequences - (Hacker News)

https://spectrum.ieee.org/faster-dna-synthesis-sidewinder

# Summary

Researchers have developed a new DNA synthesis method called Sidewinder that dramatically speeds up the construction of genetic sequences designed by AI. The technique can assemble dozens of DNA sequences simultaneously in a single test tube with unprecedented accuracy (one error per 10 million assembly events versus one per 10-30 for conventional methods), while using cheap, readily available raw materials. This addresses a critical bottleneck in modern genomics where AI tools like Evo 2 can design novel DNA sequences far faster than scientists can physically build them.

Sidewinder works by tagging DNA fragments with unique molecular barcodes that act like "page numbers," ensuring each piece links with the correct partner regardless of sequence similarity. The breakthrough includes a new algorithm (PyWinder) that generates these barcodes in minutes on a standard laptop rather than hours. The founders have launched a company called Genyro to commercialize the technology while keeping it accessible to academic researchers, potentially enabling applications in drug discovery, synthetic organisms, and data storage.

### Open Code Review – An AI-powered code review CLI tool - (Hacker News)

https://github.com/alibaba/open-code-review

# Summary

Open Code Review is an AI-powered code review CLI tool developed by Alibaba and open-sourced after being battle-tested at scale over two years. It combines deterministic engineering with LLM agents to deliver precise, line-level code review comments. The tool reads Git diffs, sends them to configurable LLMs (OpenAI or Anthropic compatible), and generates structured reviews while supporting fine-tuned rulesets for common vulnerabilities like NPE, thread-safety, XSS, and SQL injection.

The tool addresses key limitations of general-purpose AI agents used for code review—incomplete coverage, position drift, and unstable quality—by using a hybrid architecture. It employs hard constraints for critical steps (precise file selection, smart bundling, rule matching) handled by deterministic engineering, while reserving dynamic decision-making and context retrieval for the LLM agent. Users can install via NPM or GitHub releases, configure an LLM endpoint with API credentials, and begin reviewing code through CLI commands or integration with AI coding agents.

### Do transformers need three projections? Systematic study of QKV variants - (Hacker News)

https://arxiv.org/abs/2606.04032

# Summary

This paper investigates whether transformer models require three separate projections (query, key, value) in their attention mechanism, systematically evaluating variants that share projections. The researchers tested three constraint schemes: shared key-value (Q-K=V), shared query-key (Q=K-V), and single projection (Q=K=V), supplemented with 2D positional encodings to address symmetry issues. Their experiments across synthetic tasks, computer vision, and language modeling revealed that shared projections perform comparably to or better than standard three-projection transformers.

The key findings show that Q-K=V projection sharing achieves a 50% reduction in KV cache with minimal performance loss (3.1% perplexity degradation in language modeling), and when combined with other cache-reduction techniques like group query attention, yields up to 96.9% cache reduction—enabling practical on-device inference. The authors attribute this success to keys and values occupying similar representational spaces and attention operating in low-rank regimes, positioning projection sharing as an underexplored weight-tying opportunity with direct memory benefits for edge deployment.

### Changing How We Develop Ladybird - (Hacker News)

https://ladybird.org/posts/changing-how-we-develop-ladybird/

# Summary

Ladybird is closing its public pull request process and will only accept code contributions from project maintainers moving forward. Project lead Andreas Kling explains that as the browser moves toward its first alpha release, the project requires tighter development controls, clearer security measures, and a smaller group of people responsible for code quality. While the team is grateful for past external contributions, the rise of AI-generated code has fundamentally changed the trust dynamics—a substantial patch no longer reliably indicates good faith effort, making it impossible to assess contributor trustworthiness as before.

For a browser that executes untrusted internet content on user machines, security vulnerabilities pose critical risks, and every line of code entering the project becomes the maintainers' responsibility. Ladybird will remain open source with publicly available code, and external involvement through bug reports, testing, standards discussion, and security reports will continue to be valued—but code contributions will be restricted to maintainers who can be held accountable for the consequences of their changes.

### Watching a Z80 from an RP2350 - (Hacker News)

https://emalliab.wordpress.com/2026/05/26/watching-a-z80-from-an-rp2350/

# Summary

Kevin's blog post describes his experimental project to interface a Raspberry Pi Pico RP2350 microcontroller with a Z80 CPU using the RP2350's PIO (Programmable I/O) capabilities. He covers the Z80 bus architecture (16-bit address, 8-bit data, and control lines), explores the Z80's clock requirements (finding it can operate at very low frequencies, as slow as ~5KHz), and details the RP2350's GPIO programming challenges, particularly managing its 48 GPIO pins across two bases.

To demonstrate the interface works, Kevin develops test code that captures data from the Z80 bus and displays it on LEDs. He progresses from simply mirroring the data bus to LEDs to more sophisticated logic that detects specific IO write operations (like writes to address 0) and only then updates the LED display. His final working example uses the powerful RP2350 to essentially emulate simple TTL logic chips, allowing a Z80 running BASIC code to control LEDs through IO port operations.
