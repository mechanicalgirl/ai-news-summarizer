### What types of exceptions should you catch? - (Python Weekly)

https://www.pythonmorsels.com/what-types-of-exceptions-should-you-catch/

# Summary

This article explains best practices for catching exceptions in Python. The key principle is to catch only specific exceptions you understand and expect, rather than broadly catching multiple exception types. The author demonstrates this with a CSV parsing example, showing why catching `NameError` and `KeyError` is problematic—`NameError` masks bugs in your own code, while `KeyError` produces unclear error messages that are actually more informative when left unhandled. A better approach is to handle potential issues preemptively, such as validating CSV headers before processing rows.

The one exception to this rule is "mission-critical" code where you want to catch broad exception types to prevent the entire program from crashing. For example, when processing multiple files, catching all exceptions allows you to log errors and continue processing rather than halting on the first failure. In summary, be specific and deliberate about which exceptions you catch, understanding when each would occur and why you need to handle it.

### Macro Evals for Agentic Systems - (Python Weekly)

https://developers.openai.com/cookbook/examples/partners/macro_evals_for_agentic_systems/macro_evals_for_agentic_systems

# Summary

This article introduces a cookbook guide for conducting macro-evaluations (macro-evals) on agentic systems—AI systems that use multiple specialized agents working together. Rather than evaluating individual agent responses in isolation, macro-evals examine recurring patterns and system-wide behaviors across large populations of traces to identify systemic issues like late handoffs, missed signals, or incorrect review triggers.

The guide demonstrates a practical workflow using a synthetic electric vehicle (EV) order system where specialized agents handle different functions (pricing, compliance, supply, scheduling, etc.). This approach helps development teams understand how their multi-agent systems perform at scale and identify improvements needed across the entire system rather than just fixing individual responses.

### Opaque Types in Python - (Python Weekly)

https://blog.glyph.im/2026/05/opaque-types-in-python.html

# Summary

This article introduces the "opaque data type" design pattern for Python libraries, which allows developers to expose complex configuration objects while maintaining a minimal, controlled public interface. The pattern uses `typing.NewType` to create a public type that wraps a private dataclass with private attributes, combined with public constructor functions that are the only way to create instances. This approach provides type-checking benefits while preventing users from directly accessing internal implementation details.

The key advantage of this pattern is that it preserves flexibility for future evolution. As demonstrated through a shipping library example, the implementation can grow from simple string-based options to complex carrier and conveyance enumerations without breaking the public API—clients continue using the same constructor functions (like `shipFast()`) regardless of how the underlying `_RealShipOpts` class changes internally. This allows library developers to avoid committing prematurely to an expansive public API while building and refining their solution.

### Shrink Your Python Container in One Command with SlimToolkit - (Python Weekly)

https://codecut.ai/shrink-python-container-slimtoolkit/

# Summary

SlimToolkit is a command-line tool that reduces Docker container image sizes by analyzing runtime behavior to identify and strip unused files. It works through static analysis (examining image contents) and dynamic analysis (running the image to see which files are actually used), enabling significant size reductions without modifying the Dockerfile.

The article demonstrates SlimToolkit's effectiveness by slimming a Chainlit LLM chatbot image from 308 MB to 163 MB (about 47% reduction). The process involves building the initial image, running `slim build` with appropriate flags like `--continue-after enter` to trace actual application usage, and using `--include-path` to preserve directories needed at startup but not probed during runtime. The tool successfully removes OS cruft (perl, apt libraries, bash), Python build leftovers, and unused dependency files while maintaining full application functionality.

### Build a Live Object Detection App for the Reachy Mini With TensorFlow and PyCharm - (Python Weekly)

https://blog.jetbrains.com/pycharm/2026/05/build-a-live-object-detection-app-for-reachy-mini-with-tensorflow-and-pycharm/

# Summary

This tutorial demonstrates how to build a live object detection application using TensorFlow and PyCharm, then deploy it on the Reachy Mini open-source robot. The Reachy Mini is a compact, AI-focused robot developed by Pollen Robotics, Hugging Face, and Seeed Studio that features a camera, microphone, and speaker, making it ideal for conversational and vision-based applications.

The project is split into two stages: Stage 1 creates a standalone TensorFlow object detection pipeline using the SSD MobileNet V2 model from TensorFlow Hub, which can be tested on a laptop with just a webcam using PyCharm's notebook integration. Stage 2 integrates this detection model with the Reachy Mini robot, enabling real-time object tracking where the robot's head follows detected objects, its antennas respond to new detections, and a live web dashboard displays annotated camera feeds. The tutorial includes helper functions for running inference and drawing bounding boxes, along with code for both single-frame testing and continuous real-time detection.
