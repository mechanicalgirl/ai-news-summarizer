### Python Type Checker Comparison: Speed and Memory Usage - (Python Weekly)

https://pyrefly.org/blog/speed-and-memory-comparison/

# Summary

This article compares the performance of four Python type checkers—Pyrefly, Ty, Pyright, and Mypy—across 53 popular open-source packages. The newer checkers written in Rust (Pyrefly and Ty) significantly outperform the older ones: they complete small packages in under 100ms versus 2-20x longer for Pyright and Mypy, and handle large packages like numpy and pandas in seconds while Pyright takes over a minute. Pyrefly is particularly efficient on memory usage, consuming less RAM than Ty on most packages tested.

The performance differences stem from implementation languages (Rust vs. Python/TypeScript), architectural choices, inference features, and maturity levels. Scientific computing packages like scipy and numpy are the most challenging to check due to complex overloads and unions. Pyrefly's design prioritizes both speed and memory efficiency while maintaining advanced inference features for better type safety, and the team believes further optimization improvements are still possible in future releases.

### Why Aren’t We uv Yet? - (Python Weekly)

https://aleyan.com/blog/2026-why-arent-we-uv-yet/

# Summary

Since uv's launch in February 2024, it has rapidly gained popularity despite still trailing traditional pip/requirements.txt in overall adoption. While uv was the most "admired" tool in Stack Overflow's 2025 survey, only 10% of Python repositories in the top 100,000 use it. However, adoption is much stronger in newer projects—30-32% of repositories created in 2025-2026 use uv, indicating a clear generational shift in Python tooling preferences.

The adoption gap appears largely driven by LLM coding agents, which continue recommending pip and requirements.txt due to their prevalence in training data, despite many developers preferring uv themselves. The author recommends that library authors mention uv in installation instructions, application authors switch to uv with pyproject.toml, and script authors adopt PEP 723 inline dependencies instead of requirements.txt files—changes that would help close the gap between tool preference and actual usage.

### Tracking Celery Task Failures in Python - (Python Weekly)

https://blog.appsignal.com/2026/04/09/tracking-celery-task-failures-in-python.html

# Summary

This article explains the challenges of tracking Celery task failures in Python applications. Celery is widely used for handling background jobs in Python, but its default behavior can hide reliability issues—tasks often fail silently, retry automatically, and eventually succeed, making failures invisible unless developers actively monitor worker logs. The article demonstrates how failures get buried in logs across distributed systems, showing a practical example of a task with a 70% failure rate that retries before succeeding, illustrating why teams often only discover problems when users complain.

The article then introduces AppSignal as a solution to this problem. AppSignal automatically instruments Celery tasks and provides structured, centralized visibility into task executions, failures, and retries—eliminating the need to manually sift through worker logs. It captures complete tracebacks and contextual information about each task, making it easy to identify and debug issues before they impact users rather than discovering them after the fact.
