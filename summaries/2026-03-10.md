### Netflix Automates RDS PostgreSQL to Aurora PostgreSQL Migration Across 400 Production Clusters - (InfoQ)

https://www.infoq.com/news/2026/03/netflix-automates-rds-aurora/

# Netflix Automates RDS to Aurora PostgreSQL Migration

Netflix has developed an internal automation platform that successfully migrated nearly 400 production PostgreSQL databases from Amazon RDS to Aurora PostgreSQL with minimal downtime and operational risk. The system enables self-service migrations by coordinating replication validation, controlled cutover, change data capture handling, and rollback safeguards entirely at the infrastructure level, abstracting complexity from application teams who access databases through a platform-managed Envoy-based data access layer.

The migration workflow creates an Aurora cluster as a physical read replica of the source RDS instance, validates replication health and lag under production traffic, coordinates change data capture slot states to prevent WAL buildup, and performs a controlled quiescence phase where the source is rebooted to block new connections before promoting the Aurora replica. Netflix prioritized rollback as a first-class concern, keeping the original RDS instance intact until promotion is finalized, allowing traffic to be redirected back through the data access layer if anomalies are detected post-migration, with no application redeployment required.

### Article: Change as Metrics: Measuring System Reliability Through Change Delivery Signals - (InfoQ)

https://www.infoq.com/articles/change-metrics-system-reliability/

# Summary

This article argues that system changes are the primary driver of production incidents (60-80% of cases) and should therefore be treated as first-class reliability signals. The author proposes a minimal metric framework consisting of three business-level indicators—Change Lead Time, Change Success Rate, and Incident Leakage Rate—to assess both delivery efficiency and system reliability. These metrics align with industry-standard DORA metrics and provide comprehensive insight into change delivery performance.

To operationalize this framework, the article introduces three actionable technical metrics—Change Approval Rate, Progressive Rollout Rate, and Change Monitor Time—that identify friction points in the deployment pipeline. The author emphasizes that modern systems require an event-centric data warehouse architecture to collect and standardize change events across heterogeneous platforms and distributed teams, enabling unified observability and risk-based prioritization of improvements that simultaneously reduce incident risk and improve delivery throughput.

### Google BigQuery Previews Cross-Region SQL Queries for Distributed Data - (InfoQ)

https://www.infoq.com/news/2026/03/google-bigquery-cross-region-sql/

# Summary

Google Cloud has announced a preview of global queries for BigQuery, enabling developers to run SQL queries across data stored in different geographic regions without requiring ETL pipelines to move or centralize the data first. The feature automatically handles data movement between regions, identifies which parts of queries must execute in different locations, and combines the results—simplifying multi-location analytics while allowing developers to control where queries are processed for compliance purposes.

However, global queries come with trade-offs including higher latency than single-region queries, additional costs (for compute, data replication, and temporary storage), and potential regulatory restrictions on data movement. Developers must explicitly opt in and configure the feature, and while similar capabilities exist in competing services like AWS Redshift and Athena, BigQuery's automatic coordination of distributed execution across regions sets it apart.

### Scaling Human Judgment: How Dropbox Uses LLMs to Improve Labeling for RAG Systems - (InfoQ)

https://www.infoq.com/news/2026/03/dropbox-scaling-human-judgement/

# Summary

Dropbox engineers developed a "human-calibrated LLM labeling" approach to improve their RAG system (Dropbox Dash) by automating the labor-intensive process of labeling document relevance. The method combines human expertise with LLM capabilities: humans label a small, high-quality dataset to calibrate an LLM evaluator, which then generates hundreds of thousands or millions of labels—amplifying human effort by roughly 100x. This addresses the limitations of purely manual labeling, which is expensive, slow, and inconsistent.

The key to this approach's success is validation and context awareness. LLM-generated labels are evaluated against human judgments and user behavior signals (like click-through data), with special attention to the hardest mistakes. Additionally, LLMs are given access to search capabilities and internal knowledge to understand domain-specific terminology and context, significantly improving labeling accuracy. This hybrid human-LLM method proves effective for scaling RAG systems without sacrificing quality.

### AWS Introduces Nested Virtualization on EC2 Instances - (InfoQ)

https://www.infoq.com/news/2026/03/aws-ec2-nested-virtualization/

# AWS Introduces Nested Virtualization on EC2 Instances

AWS has announced support for nested virtualization on EC2 instances, allowing developers to run virtual machines within virtualized instances using KVM or Hyper-V on C8i, M8i, and R8i instance types. This long-requested feature enables use cases such as mobile app emulation, automotive hardware simulation, and Windows Subsystem for Linux (WSL) on Windows workstations. The three-layer architecture consists of AWS physical infrastructure and Nitro hypervisor, an EC2 instance running its own hypervisor, and virtual machines running inside that instance.

Previously, nested virtualization on AWS required expensive bare-metal instances, so developers welcomed this feature as a major improvement. The capability can now be enabled via API at instance launch, making it more accessible and cost-effective than previous workarounds. AWS still recommends bare-metal instances for performance-critical workloads requiring low latency, and the feature is available across all regions, though Graviton instances are not currently supported.

### Standardizing Post-Quantum IPsec: Cloudflare Adopts Hybrid ML-KEM to Replace Ciphersuite Bloat - (InfoQ)

https://www.infoq.com/news/2026/03/cloudflare-post-quantum-ipsec/

# Summary

Cloudflare has implemented a standardized post-quantum IPsec approach using hybrid ML-KEM (Module-Lattice-based Key-Encapsulation Mechanism) to replace previous "ciphersuite bloat" and protect networks against future quantum computing threats. This hybrid method runs ML-KEM alongside classical Diffie-Hellman encryption in parallel, providing dual protection against both quantum and classical attacks without requiring specialized hardware or additional costs. The implementation aligns with NIST's 2030 deadline for transitioning away from RSA and Elliptic Curve Cryptography.

The move standardizes IPsec's post-quantum security in a way similar to how TLS already operates, resolving earlier inefficient approaches that relied on pre-shared keys or specialized quantum key distribution hardware. Cloudflare has already integrated this protection across its platform, with the Cloudflare One Appliance receiving the update automatically in February 2026, and over 60% of TLS traffic on Cloudflare's network already using hybrid ML-KEM. The company's IPsec implementation is currently in closed beta while working on third-party vendor interoperability.

### New Research Reassesses the Value of AGENTS.md Files for AI Coding - (InfoQ)

https://www.infoq.com/news/2026/03/agents-context-file-value-review/

# Summary

Recent research from ETH Zurich challenges the widespread industry practice of using AGENTS.md files to guide AI coding agents. The study, which tested four different AI models on 138 real-world Python tasks, found that LLM-generated context files actually *reduced* task success rates by 3% and increased inference costs by over 20%, while human-written files provided only marginal 4% improvements despite increasing costs by up to 19%. The researchers concluded that these context files force AI agents to perform unnecessary exploration and testing rather than serving as effective repository overviews.

The findings contradict current best practices, as developers are widely encouraged to create AGENTS.md files to help AI agents understand codebases. However, some developers have questioned the study's conclusions, arguing that the research was conducted on GitHub projects—which tend to be small, recent, and well-documented—rather than the complex, closed-source codebases where domain-specific knowledge in AGENTS.md files would be most valuable. The research highlights a gap between current recommendations and observed outcomes, suggesting that context files should be carefully curated to include only truly non-inferable details like specialized tooling or custom build commands.

### Soft Forks: How Agent Skills Create Specialized AI Without Training - (Radar)

https://www.oreilly.com/radar/soft-forks-how-agent-skills-create-specialized-ai-without-training/

# Summary

This O'Reilly article explains "soft forks" as a method for creating specialized AI agent behaviors without retraining models. Drawing from open-source terminology, soft forks modify agent behavior at runtime through context injection rather than changing underlying model weights. The article uses a computing analogy: models are CPUs providing raw intelligence, agent harnesses (like Claude Code) are operating systems managing resources, and agent skills are applications that specialize the system for specific tasks—similar to installing new software without recompiling the kernel.

Agent skills are implemented through simple skill packages containing Markdown files with metadata and instructions, reference documents, and executable scripts. This approach offers significant economic advantages over traditional fine-tuning, which requires substantial investment in talent, compute, and data every time the base model updates. Skills instead require only Markdown files and resource bundles, making them versionable, reviewable, and deployable artifacts that can be managed with standard software engineering tools like Git.
