### Google Open-Sources the Common Expression Language for Python - (InfoQ)

https://www.infoq.com/news/2026/03/google-cel-expr-python/

# Summary

Google has open-sourced CEL-expr-python, a Python implementation of the Common Expression Language (CEL)—a non-Turing complete embedded language designed for policy enforcement, data validation, and dynamic configuration. The implementation wraps Google's official C++ core to ensure consistency with CEL semantics while allowing Python developers to extend functionality directly in Python. This provides Python developers with an officially maintained, production-ready CEL runtime rather than relying on community-supported alternatives.

CEL is particularly valuable for applications requiring fast, secure, and flexible rule evaluation. Its key features include compile-once-evaluate-many efficiency, guaranteed termination, and side-effect-free execution with evaluation speeds ranging from nanoseconds to microseconds. The open-source release enables developers to compile expressions using Go or Java toolchains and evaluate them in Python, making CEL more accessible across different programming ecosystems and use cases.

### QCon London 2026: How To Run on Three Clouds at Once, and When Not To - (InfoQ)

https://www.infoq.com/news/2026/03/form3-triple-active-multicloud/

# Summary

At QCon London 2026, Form3 shared their experience running a payments platform across AWS, Google Cloud, and Azure simultaneously in response to regulatory pressure around cloud concentration risk. Their multi-cloud architecture uses Kubernetes clusters connected via private networks, NATS JetStream for messaging, and CockroachDB for distributed data storage, along with custom operators to handle cross-cloud challenges like database quorum management and node pool updates. The approach proved valuable during a major Google Cloud outage, when payments continued flowing through other clouds uninterrupted.

However, the speakers cautioned that this triple-active strategy isn't universally applicable. When Form3 expanded to the US market, they discovered that American customers prioritized geographic resilience and low latency over multi-cloud redundancy, forcing them to adopt a simpler active-standby architecture instead. The key takeaway: multi-cloud complexity only makes sense if your market demands it, your budget supports it, and you have strong platform engineering capabilities to maintain it.

### QCon London 2026: The Hidden Power of Boring Problems - (InfoQ)

https://www.infoq.com/news/2026/03/engineering-boring-problems/

# Summary

At QCon London 2026, Yinka Omole presented a talk arguing that engineers should prioritize mastering fundamental, "boring" problems over constantly chasing new technologies and frameworks. Despite decades of predictions that programming would become obsolete—from FORTRAN in the 1950s to AI code generation today—the developer population has grown from 14 million in 2019 to 21 million by 2025. This growth persists because underlying engineering challenges like data modeling, reliability, and distributed systems repeatedly resurface across different technologies and industries.

The talk illustrated how investing in foundational knowledge compounds over time through examples like PostgreSQL, which gradually surpassed MySQL due to its early focus on correctness and extensibility, enabling it to adapt to new requirements like JSON and vector support. Similarly, WhatsApp's architecture leveraged Erlang—a language designed for distributed communication systems—allowing a 32-engineer team to handle billions of messages daily. The presentation warned against rewriting systems from scratch, as this discards accumulated operational knowledge, and showed how sometimes simpler architectures outperform complex distributed systems, emphasizing that deep understanding of core problems yields greater long-term value than constantly adopting new tools.

### DoorDash Builds DashCLIP to Align Images, Text, and Queries for Semantic Search Using 32M Labels - (InfoQ)

https://www.infoq.com/news/2026/03/doordash-semantic-search/

# Summary

DoorDash has developed DashCLIP, a multimodal machine learning system that aligns product images, text descriptions, and user queries into a shared semantic embedding space to improve product discovery and ranking across its diverse CPG marketplace. The system uses separate encoders for images, text, and queries, employing contrastive learning to place semantically related items close together in the embedding space. DoorDash trained DashCLIP using a two-stage pipeline: first with continual pretraining on 400,000 product images and titles, then with a Query Catalog Contrastive loss using 32 million labeled query-product pairs (700,000 human-annotated and expanded via GPT-based labeling).

The deployed system outperformed baseline vision-language models like CLIP and BLIP in offline experiments and demonstrated improved engagement metrics in online A/B tests for sponsored product recommendations. Beyond advertising and ranking, the embeddings proved generalizable to other tasks like category prediction and query relevance classification, establishing DashCLIP as a foundational multimodal representation layer across DoorDash's marketplace platform.

### Presentation: Image Processing for Automated Tests - (InfoQ)

https://www.infoq.com/presentations/image-processing-automated-tests/

# Summary

Stefan Dirnstorfer discusses the evolution of automated UI testing from DOM-based approaches to visual image processing powered by AI. He explains that while traditional testing relies on internal component structures, image-based testing becomes necessary when developers lack access to internals or work with obfuscated systems. The challenge lies in the precision required—LLMs often struggle with subtle visual discrepancies like one-pixel shifts—making a hybrid approach combining advanced image registration techniques, "Chain-of-Thought" vision processing, and classical algorithms essential for reliable QA automation.

Dirnstorfer demonstrates practical applications using Claude's computer use API as a visual UI agent, showing how modern LLMs can interpret screenshots, locate UI elements, and validate visual states. He emphasizes that while AI vision capabilities are advancing rapidly, they have inherent limitations (similar to human optical illusions), making the combination of generative AI with classical image processing algorithms the most promising approach for automating complex visual testing tasks in real-world applications.

### Article: Evaluating AI Agents in Practice: Benchmarks, Frameworks, and Lessons Learned - (InfoQ)

https://www.infoq.com/articles/evaluating-ai-agents-lessons-learned/

# Summary

This article provides a comprehensive framework for evaluating AI agents in production environments, emphasizing that agents are complex systems requiring evaluation beyond traditional NLP metrics. The key insight is that classical benchmarks like BLEU and ROUGE fail to capture agent failures in real-world scenarios—such as when an agent silently skips a step after a tool failure. The article advocates for a hybrid evaluation approach combining automated scoring (LLM-as-a-judge, trace analysis) with human judgment, while prioritizing behavioral metrics like task success rates, graceful error recovery, and consistency over sandbox performance.

The evaluation framework addresses five critical dimensions: treating agents as full systems rather than just models, measuring actual behavior and task success, implementing continuous hybrid evaluation pipelines, assessing operational constraints (latency, cost, token efficiency), and ensuring safety and governance compliance. The article emphasizes that production-grade evaluation requires attention to reliability, version management, and data protection, providing practical tools and examples (MLflow, TruLens, LangChain Evals) for teams transitioning AI agents from prototypes to reliable enterprise systems.

### How Grab Optimizes Image Caching on Android with Time-Aware LRU - (InfoQ)

https://www.infoq.com/news/2026/03/grab-tlru-image-cache/

# Summary

Grab engineers improved their Android app's image caching by upgrading from a standard Least Recently Used (LRU) cache to a Time-Aware LRU (TLRU) cache. The original 100 MB LRU cache had significant limitations: it filled quickly for some users causing performance issues, while for others images remained cached for months, wasting storage. TLRU adds time-based expiration parameters—Time To Live (TTL), minimum cache size threshold, and maximum cache size—to reclaim storage more effectively.

Rather than building TLRU from scratch, Grab forked and extended Glide's proven DiskLruCache implementation, which already handled complex edge cases like crash recovery and thread safety. The implementation required tracking last-access times, implementing time-based eviction logic, and migrating existing user caches by assigning migration timestamps to all LRU entries. Through controlled experiments balancing cache hit ratio with storage optimization, Grab achieved a 50 MB reduction in cache size for 95% of users while maintaining acceptable performance, ultimately reclaiming terabytes of storage across devices without increasing server costs.

### Google Researchers Propose Bayesian Teaching Method for Large Language Models - (InfoQ)

https://www.infoq.com/news/2026/03/google-bayesian-llm/

# Summary

Google researchers have proposed a "Bayesian teaching" method to improve how large language models update their beliefs during multi-step interactions with users. Rather than training models only on correct answers, the team trains them to imitate a Bayesian system that uses probabilistic reasoning to update user preference estimates based on new feedback. In experiments using a simulated flight recommendation task, this approach outperformed both untrained models and those trained on perfect oracle responses, with models better matching Bayesian decision-making across multiple interaction rounds.

The research addresses a key limitation of current LLMs: they struggle to effectively revise internal assumptions when new information arrives. Community feedback was largely positive, with developers noting the potential for improved multi-turn adaptation in long-running AI agents, though some questioned whether reinforcement learning might be more suitable than the supervised fine-tuning method used.

### Cloudflare Introduces Support for ASPA, an Emerging Internet Routing Security Standard - (InfoQ)

https://www.infoq.com/news/2026/03/cloudflare-aspa-standard/

# Summary

Cloudflare has announced support for ASPA (Autonomous System Provider Authorization), a new cryptographic security standard that strengthens Internet routing by validating the path data takes across networks. ASPA works within the RPKI framework to prevent route leaks and hijacks by allowing networks to publish and verify their authorized upstream providers, ensuring traffic only travels through approved network chains. This addresses a key vulnerability in the Border Gateway Protocol (BGP), which lacks native path validation.

While ASPA represents a significant cryptographic upgrade for Internet security—as demonstrated in recent incidents like the Venezuela BGP route leak that ASPA would have detected—widespread adoption will require substantial updates to RPKI infrastructure, router implementations, and BGP software. Though still in draft status at IETF, major providers like AWS and Cloudflare are committed to the standard, and some regional registries and routing software already support ASPA object creation and validation.

https://www.oreilly.com/radar/capability-architecture-for-ai-native-engineering/

# Summary

This O'Reilly article argues that the primary challenge facing engineering teams in adopting AI is not talent, but rather the lack of a shared reference model and common language for integrating AI into everyday work. While some teams have successfully moved beyond one-off experiments to build repeatable AI workflows, many struggle due to information overload and confusion about best practices. The author contends that establishing this collective framework is essential for moving from scattered experimentation to coordinated, scalable practice across the engineering industry.
