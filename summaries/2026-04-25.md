### Django to Browser Push - Without WebSockets, Channels, or Redis - (Python Weekly)

https://usman.it/django-realtime-updates-without-websockets/

# Summary

DjangoRealtime is a new open-source Django package that enables real-time browser updates without the complexity of WebSockets, Channels, Redis, or Celery. It leverages Server-Sent Events (SSE) and PostgreSQL's NOTIFY/LISTEN functionality to push events from the server to connected browsers. The implementation is remarkably simple: install the package, add it to Django settings, and use straightforward API calls to publish events from the backend and listen for them on the frontend.

The author, Muhammad Usman, created DjangoRealtime based on lessons learned from building Socketize, a realtime platform startup. He chose SSE over WebSockets because most real-time web use cases only require one-way server-to-browser communication, and SSE works seamlessly over HTTP without requiring additional infrastructure. The solution requires only Django and PostgreSQL—both likely already present in most Django applications—making it ideal for use cases like task notifications, live dashboards, multi-tab synchronization, and in-app notifications.

### Decoupling Your Business Logic from the Django ORM - (Python Weekly)

https://buttondown.com/carlton/archive/decoupling-your-business-logic-from-the-django-orm/

# Summary

This article discusses the challenge of managing business logic in Django applications as they grow. The author argues that while Django's Active Record pattern (putting logic in models) works well initially, it becomes problematic as applications scale—models accumulate logic from multiple overlapping concerns, becoming difficult to maintain, test, and understand. The article also highlights related ORM performance issues, such as unnecessary field fetching and the N+1 query problem caused by lazy related lookups.

The solution proposed is to decouple business logic from the ORM by creating separate classes (like simple data transfer objects) that contain only the exact data needed for specific tasks. This separation allows developers to manage complexity more effectively by keeping database persistence concerns distinct from application logic, making code easier to test, maintain, and evolve without worrying about premature optimization.

### Django: fixing a memory “leak” from Python 3.14’s incremental garbage collection - (Python Weekly)

https://adamj.eu/tech/2026/04/20/django-python-3.14-incremental-gc/

# Summary

When upgrading a Django project to Python 3.14, the author encountered out-of-memory errors during database migrations on a resource-limited server. The issue was caused by Python 3.14's new incremental garbage collection algorithm, which spreads garbage collection work across multiple smaller operations to reduce pause times. However, this change inadvertently caused temporary model classes created during migrations to persist in memory longer than expected, creating what appeared to be a memory leak. The author used Memray profiling and custom debugging hooks to identify that over 50% of memory was being consumed by these temporary classes that should have been quickly garbage-collected.

The article documents the author's investigation and workaround for this issue, but notes that the problem has since been validated by the Python community—the Python 3.14 release manager announced that the incremental garbage collection algorithm will be reverted in Python 3.14.5 and the next Python 3.15 alpha release due to reports of increased memory usage, making this one of several cases proving the reversion was the right decision.

### Your Models Know Their Own Schema. Let Them Show You. - (Python Weekly)

https://jeffield.net/blog/your-models-know-their-own-schema-let-them-show-you/

# Summary

**django-schematic** is a Django tool that automatically generates live, interactive diagrams of your database schema directly from your model definitions, eliminating the need for manual schema documentation. Instead of spending hours drawing boxes and arrows in diagramming tools—only to have them become outdated with the next migration—the tool reads your actual Django models and displays them as an always-accurate, interactive graph that updates whenever your code changes.

The tool also enables smart PNG exports that embed layout information invisibly in the image file, allowing developers to share schema diagrams in pull requests and GitHub issues. When colleagues import these PNGs back into django-schematic, it restores the exact view while pulling current model definitions from their own codebase, ensuring diagrams stay accurate even as the schema evolves. The tool stays hidden in production and offers filtering and layout options to keep complex schemas manageable.

### Implementing MikroTik's Binary API Protocol in Python from Scratch - (Python Weekly)

https://www.joekarlsson.com/blog/implementing-mikrotik-binary-api-protocol-in-python/

# Summary

The author implemented MikroTik's proprietary binary API protocol in Python from scratch (~137 lines) to programmatically manage network infrastructure on his homelab, rather than relying on existing libraries. The protocol uses sentence-based messaging over raw TCP, with an elegant variable-length encoding scheme (similar to UTF-8) for word lengths, allowing short commands to use minimal bytes while supporting arbitrarily large payloads. Authentication is straightforward plain-text username/password, and once connected, commands mirror MikroTik's CLI structure, making it simple to manage bonding configurations, DHCP settings, and firewall rules without any external dependencies.

The author chose to implement from scratch to deeply understand the protocol and create a lightweight tool suitable for shell scripts and automation. He emphasizes that binary protocols aren't complex, variable-length encoding appears throughout modern systems (UTF-8, Protocol Buffers, Git), and sometimes lightweight custom solutions outperform heavy library dependencies—particularly for homelab tools that need zero external dependencies and efficient communication over persistent TCP connections.
