### Can you get root with only a cigarette lighter? (2024) - (Hacker News)

https://www.da.vidbuchanan.co.uk/blog/dram-emfi.html

# Summary

This article describes how the author successfully achieved root access on a Linux system using hardware fault injection with minimal equipment—specifically, a piezoelectric BBQ lighter as an electromagnetic fault injection (EMFI) tool. Rather than requiring expensive specialized equipment, the author created a simple setup consisting of just a resistor and wire soldered to a memory pin on a 2011 Samsung laptop. By triggering the lighter near an antenna wire connected to the DDR memory bus, they could reliably induce bit-flips in specific memory locations.

The author demonstrates two proof-of-concept exploits: first, a "sandbox escape" from CPython by manipulating object pointers through controlled bit-flips, and second, a full privilege escalation attack on Linux itself. The key insight is that by saturating the memory bus with repeated accesses to target data, they could ensure a randomly-timed fault would hit the desired location with high probability. This work shows that serious security vulnerabilities can be exploited with surprisingly simple and inexpensive hardware tools, challenging assumptions about the cost and complexity barriers to hardware-level attacks.

### Tin Can, a 'landline' for kids - (Hacker News)

https://www.businessinsider.com/tin-can-landline-kids-cellphone-cell-alternative-how-2025-9

# Summary

Tin Can is a new WiFi-based "landline" phone designed for children that aims to delay their need for a full smartphone. Founded in 2025 by three Seattle-area parents, the device functions like a traditional corded landline but operates over home WiFi with parental controls that restrict calls to approved contacts during set hours. The company has already sold "tens of thousands" of units since launch and is backordered until December, with $3.5 million in funding raised.

The product addresses a growing movement among parents seeking to postpone giving kids smartphones until high school, while still allowing them social independence and communication with friends. By offering a closed network where only other Tin Can users can call, the device eliminates exposure to social media, excessive screen time, and associated anxiety while providing an alternative to smartwatches or heavily-restricted Android phones. The concept has resonated strongly with parents on social media who appreciate both the phone etiquette lessons and the autonomy it gives children without the risks of traditional smartphones.

### The gold standard of optimization: A look under the hood of RollerCoaster Tycoon - (Hacker News)

https://larstofus.com/2026/03/22/the-gold-standard-of-optimization-a-look-under-the-hood-of-rollercoaster-tycoon/

# Summary

RollerCoaster Tycoon (1999) is renowned as one of the most optimized games ever made, achieving impressive performance through multiple layers of optimization beyond just its famous Assembly language implementation. Chris Sawyer employed aggressive optimization techniques throughout the codebase, including using different data types for different money values based on their expected ranges, replacing multiplication and division operations with faster bitshift operations (when dealing with powers of two), and designing formulas specifically to work with CPU-friendly numbers.

What makes RCT's optimization particularly remarkable is how Sawyer's dual role as both programmer and game designer allowed him to make fundamental design decisions informed by technical performance constraints. Rather than implementing expensive pathfinding for thousands of guests, the game uses a simpler "random wandering" system where guests accidentally discover attractions, with true pathfinding reserved only for critical cases like mechanics reaching broken rides. Even these necessary pathfinding operations include built-in traversal limits to prevent performance spikes, turning technical limitations into actual gameplay features—something nearly impossible in modern game development where design and programming roles are strictly separated.

### The way CTRL-C in Postgres CLI cancels queries is incredibly hack-y - (Hacker News)

https://neon.com/blog/ctrl-c-in-psql-gives-me-the-heebie-jeebies

# Summary

This article examines a longstanding security vulnerability in PostgreSQL's query cancellation mechanism. When users press Ctrl-C in psql to cancel a query, the cancellation request (CancelRequest) is sent unencrypted over the network, even when the original database connection uses strict TLS encryption. Although PostgreSQL 17 added support for encrypted cancellation requests in its client library (libpq), psql itself still sends these messages in plaintext due to architectural constraints—a situation that has persisted for over 30 years.

The article highlights two main concerns: a potential denial-of-service attack where network eavesdroppers could replay intercepted cancellation requests to disrupt future queries, and practical challenges for Postgres proxy tools like Elephantshark that rely on TLS's Server Name Indication (SNI) to route connections, since unencrypted CancelRequests lack this identifying information. The author recommends using Postgres 18 with explicit protocol version settings, VPNs, and checking that third-party Postgres drivers support encrypted cancellations until psql itself is updated to use the newer secure functions.

### The future of version control - (Hacker News)

https://bramcohen.com/p/manyana

# Summary

Bram Cohen has released Manyana, a proof-of-concept version control system based on CRDTs (Conflict-Free Replicated Data Types) that addresses fundamental limitations of traditional systems like Git. By using CRDTs, merges never fail by definition—instead, conflicts are flagged when changes touch each other and presented in a far more informative way, allowing developers to see exactly what each branch did rather than staring at opaque blob markers.

The system offers several advantages over conventional version control: permanent line ordering prevents merge inconsistencies, conflicts become informative rather than blocking, and history is preserved structurally rather than reconstructed from the DAG. Notably, Manyana enables "rebase without the nightmare" by replaying commits onto new bases while keeping full history intact. Currently a 470-line Python demo operating on individual files rather than a full system, Manyana demonstrates that CRDT-based version control can solve hard UX problems and provide better solutions than existing tools.

### Reports of code's death are greatly exaggerated - (Hacker News)

https://stevekrouse.com/precision

# Summary

The article argues that despite widespread claims that AI will kill coding, programming will remain essential because good code serves two purposes: it produces functioning software AND it serves as a precise artifact for understanding complexity. While AI can help with "vibe coding"—iteratively refining English-language specifications into working code—this approach breaks down at scale when unexpected bugs emerge from unmastered complexity. The solution lies in abstraction: compressing multiple concepts into simpler semantic levels, which is what great programmers do and what separates quality code from "slop."

Looking toward AGI, the author contends that as AI becomes more capable, we won't use it to skip writing better code—we'll use it to solve our hardest abstraction problems and create clearer, more elegant systems. Just as ChatGPT hasn't killed writing because great writers do more than produce text, AGI won't kill coding because great programmers do more than produce functioning programs. Code, like writing, is fundamentally an art of precision and expression that will only become more valuable as AI advances.

### Why I love NixOS - (Hacker News)

https://www.birkey.co/2026-03-22-why-i-love-nixos.html

# Summary

The author loves NixOS primarily for its deterministic and reproducible Nix package manager, which allows users to define entire operating systems declaratively in a single configuration file rather than accumulating unpredictable system state over time. This approach enables easy system rebuilding, rollbacks, and portability across machines without manual setup steps or scattered configuration files.

Beyond system management, NixOS excels in modern development workflows—particularly with AI coding agents—by providing isolated, reversible environments for experimenting with different tool versions and dependencies without polluting the base system. The same functional package management model extends across platforms (macOS, Linux, FreeBSD) and deployment scenarios (Docker images, CI pipelines), offering the author a coherent, stable system that is "declarative, reproducible, reversible and stable" while minimizing system drift and upgrade anxiety.

# Project NOMAD Summary

Project NOMAD is a free, open-source offline server that bundles Wikipedia, AI language models, maps, and educational content into a single system that runs completely without internet. It's built on proven open-source tools (Kiwix, Ollama, OpenStreetMap, and Kolibri) and installs in seconds on Linux systems, requiring no subscriptions or paywalls.

### A Copy-Paste Bug That Broke PSpice AES-256 Encryption - (Hacker News)

https://jtsylve.blog/post/2026/03/18/PSpice-Encryption-Weakness

# Summary

A critical vulnerability in PSpice's AES-256 encryption (Mode 4) was discovered due to a copy-paste bug in the key derivation code. The encryption scheme uses only a 4-byte user-supplied key along with known constants and zero padding to create the full 32-byte AES key, reducing the effective keyspace from 2^256 to just 2^32 (~4 billion possibilities). Since the first encrypted block contains a known plaintext prefix ("0001.0000 "), attackers can brute-force the correct key in seconds using modern hardware, compromising all SPICE model files encrypted with that user key.

The root cause stems from PSpice supporting both DES and AES encryption: the code mistakenly passes the short 8-byte DES-sized key to the AES engine instead of the intended 31-byte extended key, which was likely an unintended copy-paste error from the DES implementation. The bug has existed since AES support was introduced in 2014, but fixing it now would break compatibility with twelve years of encrypted files. The researcher released SpiceCrypt, an open-source tool that decrypts PSpice and LTspice files to restore simulator interoperability for lawfully obtained models.
