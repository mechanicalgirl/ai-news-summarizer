### Uber’s Hive Federation Decentralizes 16K Datasets and 10+ PB for Zero-Downtime Analytics at Scale - (InfoQ)

https://www.infoq.com/news/2026/04/uber-hive-decentralized-data/

# Summary

Uber has redesigned its Hive data warehouse to decentralize over 16,000 datasets totaling 10+ petabytes, moving away from a monolithic architecture that created scalability, operational, and security challenges. Using a pointer-based migration approach with the Hive Metastore, Uber redirects datasets to new HDFS locations without duplicating data, enabling zero-downtime migration through split-second pointer updates. The system includes four key components—Bootstrap Migrator, Realtime Synchronizer, Batch Synchronizer, and Recovery Orchestrator—that manage initial migration, maintain metadata alignment during transitions, and enable safe rollbacks if needed.

This federated architecture addresses critical limitations of the previous system by eliminating resource contention between teams, enforcing fine-grained access controls, and distributing dataset ownership across domains. The migration successfully moved thousands of datasets, performed over 7 million metadata syncs, and reclaimed 1+ PB of storage while maintaining data integrity and regulatory compliance. By decentralizing responsibility, teams gain operational autonomy, improved observability, and reduced dependency on centralized operations, ultimately enhancing the resilience and scalability of Uber's analytics ecosystem.

### Article: Building Hierarchical Agentic RAG Systems: Multi-Modal Reasoning with Autonomous Error Recovery - (InfoQ)

https://www.infoq.com/articles/building-hierarchical-agentic-rag-systems/

# Summary

This article addresses the "modality gap" problem in enterprise RAG systems—the challenge of simultaneously querying both structured databases (SQL) and unstructured documents to provide complete answers. Traditional linear RAG pipelines fail because they commit to specific data sources upfront and lack mechanisms to iteratively refine incomplete results, leading to hallucinations and missed context. The author demonstrates how hierarchical multi-agent orchestration with a supervisor-worker topology can decompose complex queries into specialized sub-tasks, achieving 84.5% accuracy compared to 62.8% for flat-agent approaches.

The proposed solution combines three key architectural patterns: hierarchical agent orchestration for task decomposition, autonomous error recovery through reflective retry mechanisms that detect and correct failures (reducing hallucinations by 60%), and cloud-agnostic database adapters that work across multiple enterprise data warehouses. The approach emphasizes deterministic control flow with explicit state management and schema awareness to enable production-grade deployment while maintaining auditability and compliance—addressing a real problem where approximately 30% of multi-hop queries in financial services fail silently by omitting over 20% of relevant data points.

### Cloudflare and ETH Zurich Outline Approaches for AI-Driven Cache Optimization - (InfoQ)

https://www.infoq.com/news/2026/04/cloudflare-ai-caching-strategies/

# Summary

Cloudflare and ETH Zurich have identified significant operational challenges created by AI-driven crawler traffic, which now accounts for over 10 billion requests per week on Cloudflare's network. AI bots exhibit fundamentally different behavior than human users—maintaining high unique URL ratios, issuing parallel requests, and accessing rarely-visited pages without leveraging browser caching. This creates "cache churn" that displaces frequently-accessed human content, reducing cache hit rates and increasing origin server load across CDNs and backend infrastructure.

To address these challenges, the organizations propose AI-aware caching strategies that separate human and AI traffic into distinct cache tiers, implement alternative cache eviction algorithms (such as least frequently used or first-in-first-out), and employ machine learning-driven policies that adapt dynamically. They also recommend complementary measures like structured feeds or pay-per-crawl models to control AI access. The proposals underscore that modern cache architectures must be redesigned to efficiently serve both human users and AI agents simultaneously.

### GitHub Actions Custom Runner Images Reach General Availability - (InfoQ)

https://www.infoq.com/news/2026/04/github-actions-custom-runners/

# Summary

GitHub has announced that custom runner images for GitHub Actions are now generally available after leaving public preview in October. This feature allows teams to create pre-configured virtual machine images with all necessary tools, dependencies, SDKs, and certificates baked in, eliminating the need to install them from scratch on every workflow run. The process involves setting up an image-generation runner, using the snapshot keyword to create images, and configuring runners to use those images—with GitHub automatically managing versioning and teams able to pin to specific versions or automatically receive updates.

The feature is exclusively available on larger runners (GitHub Team or Enterprise Cloud plans) and images must be built from GitHub-approved base images, remaining within the GitHub ecosystem rather than integrating external container registries. While this approach offers less flexibility than competitors like GitLab (which allows any container image) or CircleCI (which supports custom Docker images), it provides a more managed, integrated solution with reduced operational overhead—similar to self-hosted runners with pre-baked AMIs but fully integrated into the GitHub platform.

### Article: Stateful Continuation for AI Agents: Why Transport Layers Now Matter - (InfoQ)

https://www.infoq.com/articles/ai-agent-transport-layer/

# Summary

As AI agents become more sophisticated, transport layers are emerging as a critical performance concern for multi-turn agentic workflows. Unlike simple single-turn LLM interactions, AI coding agents perform 10-50+ sequential turns involving tool calls, file reads, edits, and test iterations. With HTTP-based stateless APIs, the entire conversation history must be resent with each turn, causing request payloads to grow linearly and creating significant overhead—particularly problematic over bandwidth-constrained connections.

OpenAI's new WebSocket mode addresses this by implementing stateful continuation, caching conversation history server-side and allowing clients to send only a reference ID plus new data. This approach can reduce client-sent data by over 80% and improve execution time by 15-29%, demonstrating that architectural decisions around state management have become first-order concerns for agent performance. However, the article notes that these stateful designs introduce new trade-offs in reliability, observability, and portability that must be carefully evaluated.

### Architecture as Code to Teach Humans and Agents About Architecture - (Radar)

https://www.oreilly.com/radar/architecture-as-code-to-teach-humans-and-agents-about-architecture/

# Summary

This O'Reilly article discusses "Architecture as Code," a practice where software architects define system structure, constraints, and relationships through code rather than just diagrams. This approach creates a feedback loop that helps architects verify implementation aligns with design, allows developers to signal when components change, and enables architects to assess whether those changes are improvements or deviations from the original plan.

The article highlights how this practice has become increasingly valuable with the rise of agentic AI. Since AI agents need deterministic constraints to operate effectively, Architecture as Code provides a way to embed architectural guardrails—such as limits on complexity or coupling—directly into the code generation process. This allows architects to establish inviolate rules that guide agents toward architecturally sound solutions rather than brute-force approaches.

Without developing proper governance structures and management practices specifically designed for AI agents, enterprises will continue to struggle with agentic AI despite having built the agents themselves.

### Engineering Storefronts for Agentic Commerce - (Radar)

https://www.oreilly.com/radar/engineering-storefronts-for-agentic-commerce/

# Summary

This article explores how AI shopping agents are fundamentally changing ecommerce design by prioritizing structured data over marketing persuasion. The author demonstrates through an experiment that autonomous agents purchasing hiking jackets consistently chose a more expensive option with clean JSON data over a cheaper jacket with attractive marketing copy. This occurs because agents use a "Sandwich Architecture"—a three-layer system where an LLM translator converts user requests into structured queries, a deterministic middle layer validates data against strict type requirements (filtering out unstructured marketing language), and a judge layer makes final decisions based on validated data.

The key insight is that traditional ecommerce strategies built around visual design and persuasive copy become irrelevant when the "buyer" is a machine with no eyes. Merchants must now engineer their storefronts with precise, machine-readable data infrastructure instead of relying on emotional appeals and marketing techniques refined over years. This shift represents a fundamental change in how businesses should optimize their online presence for the era of autonomous shopping agents.
