### The Coming Loop - (Armin Ronacher's Thoughts and Writings)

https://lucumr.pocoo.org/2026/6/23/the-coming-loop/

# Summary

Armin Ronacher discusses the emerging pattern of "loops"—automated systems that queue work for AI agents, evaluate results, and iterate beyond where the model would normally stop. While agent loops (tools calling tools within a single session) are familiar, the outer "harness loop" is becoming increasingly dominant in software development. However, Ronacher expresses concern that current LLM-generated code tends to be overly defensive, complex, and lacking in strong invariants—problems that amplify when loops run unattended for extended periods.

Ronacher acknowledges that loops work well for specific domains like code porting, performance exploration, and security scanning—tasks that produce temporary artifacts or mechanical transformations rather than long-lived code requiring deep understanding. Yet he remains uncomfortable with applying this approach to production software, as it represents a shift from deterministic, comprehensible systems toward "software as organism" that requires treatment and monitoring rather than full comprehension. The uncomfortable reality, he concludes, is that opting out may not be possible, as adversaries and security researchers will increasingly use automated loops against systems regardless of whether developers choose to adopt them.
