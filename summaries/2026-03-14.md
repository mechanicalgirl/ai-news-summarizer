### Two React Design Choices Developers Don’t Like—But Can’t Avoid - (Dev.To)

https://dev.to/playfulprogramming/two-react-design-choices-developers-dont-like-but-cant-avoid-d6g

# Summary

This article argues that two widely criticized React design choices—deferred state commits and dependency arrays on effects—were actually correct solutions to fundamental problems that all UI frameworks eventually encounter. The author, who has been developing Solid 2.0, initially believed these React patterns were flawed but discovered that React had correctly identified deeper constraints that signals-based frameworks like Solid 1.x glossed over.

The core issue is that async operations create unavoidable challenges for UI consistency. React's deferred state commits ensure that derived values stay synchronized with their sources, preventing the UI from displaying intermediate, speculative states that don't reflect actual data. Similarly, dependency arrays allow React to know all effect dependencies before rendering or executing side effects, preventing effects from running with incomplete information. While these patterns feel awkward compared to synchronous signals, they are necessary architectural decisions when dealing with asynchronous data that cannot be immediately resolved—a reality inherent to web development's networked, client-server model.

### ascend : run Python functions on Kubernetes - (Python Weekly)

https://ocramz.github.io/posts/2026-03-05-ascend.html

# Summary

**ascend** is an open-source Python library that allows developers to easily run functions on remote Kubernetes clusters with minimal code changes. By using a simple `@ascend` decorator, users can specify hardware requirements (like GPUs), timeouts, and dependencies, and the function is automatically serialized, sent to a remote node, executed, and results returned—all without requiring Docker, CI/CD pipelines, or infrastructure management knowledge.

The library leverages Kubernetes as a job queue and resource allocator, uses Kaniko for containerized builds without Docker on the local machine, and stores serialized functions in cloud object storage. While designed for high-trust environments with shared storage access, it offers a middle ground between developing locally and using expensive all-in-one MLOps platforms, providing a seamless cloud computing experience through minimal infrastructure overhead.

### Thinnings: Sublist Witnesses and de Bruijn Index Shift Clumping - (Python Weekly)

https://www.philipzucker.com/thin1/

# Summary

This article explains **thinnings**, a mathematical concept from dependent type theory that can be useful in any programming language. Thinnings are witness data that prove one list is a sublist of another, representable as boolean lists or bitvectors. The author demonstrates how to compute and verify thinnings using Python and Prolog examples, showing that while finding a sublist certificate takes O(|ys|) time, verifying it takes only O(|xs|) time. Thinnings can be composed and form a mathematical category, with various representations (bitvectors, prefix sums, index sets) and operations.

The article then connects thinnings to **de Bruijn indices**, a technique for representing variable bindings in lambda calculus by counting binders rather than using names. When moving terms under new binders during substitution, indices must be "shifted" to account for the new scoping level. The author positions thinnings as a way to systematically handle these shift operations, setting up later discussion of how thinnings can optimize lambda egraph implementations and generalized union-find operations.

### Channel Surfer lets you watch YouTube like it’s old-school cable TV - (TechCrunch)

https://techcrunch.com/2026/03/12/channel-surfer-watch-youtube-retro-cable-tv-guide/

# Summary

Channel Surfer is a new web app by London-based developer Steven Irby that transforms YouTube into a retro cable TV experience. Users can browse 40 curated topic-focused channels—covering news, politics, sports, gaming, AI, and music—and "tune in" to videos as if watching live TV. The app displays an interactive guide showing upcoming content and allows users to join videos mid-stream, while a counter shows how many others are watching simultaneously.

The project launched to immediate success with over 10,000 views on day one. Irby created it to combat "algorithm fatigue" and the decision paralysis of modern streaming, missing the simplicity of traditional channel surfing. Users can also import their own YouTube subscriptions via a bookmarklet. Built with Next.js, PartyKit, and Cloudflare, the free service currently offers 175 YouTube channels and 25 music playlists, with Irby planning future expansion to TV platforms like Fire TV and Google TV.
