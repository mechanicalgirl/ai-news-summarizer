### Figma Builds In-House Redis Proxy to Hit Six Nines Uptime - (InfoQ)

https://www.infoq.com/news/2026/05/figma-redis-figcache/

# Summary

Figma has developed FigCache, an in-house Redis proxy service that has achieved six nines of uptime since entering production in late 2025. The system was built to replace a fragmented caching infrastructure that threatened site availability due to connection volume limits, inefficient client libraries, and operational complexity. Rather than adopting existing open-source solutions, Figma built a custom proxy because available alternatives lacked the semantic awareness needed to implement runtime guardrails and custom commands, and couldn't transparently handle the company's diverse client base.

FigCache is a stateless service built on ResPC, Figma's custom Go RPC framework, that sits between applications and AWS ElastiCache Redis clusters. Its architecture separates connection management from backend command routing, making it highly extensible. Notable features include using Starlark programs for runtime configuration management, a fanout filter engine that handles multi-shard operations that would normally cause Redis Cluster errors, and compatible client libraries in Go, Ruby, and TypeScript that require minimal code changes to adopt. This proxy-based approach mirrors Figma's 2024 database rearchitecture and abstracts operational complexity away from application code.

### Cloudflare Introduces Flagship: an Edge-Native Feature Flag Service Built on OpenFeature - (InfoQ)

https://www.infoq.com/news/2026/05/cloudflare-flagship-openfeature/

# Summary

Cloudflare has announced Flagship, a new feature flag service built on its edge platform and based on the OpenFeature open standard. The service allows developers to control feature rollouts and run experiments without redeploying code by evaluating flags directly in Cloudflare Workers at the edge, eliminating network latency and enabling sub-millisecond evaluation. Flagship supports targeting rules, percentage-based rollouts, and various flag types (boolean, strings, numbers, JSON objects), making it particularly useful for AI-driven workloads requiring rapid iteration.

The service addresses a gap in the market by providing native feature flag capabilities to Cloudflare users, potentially commoditizing the feature flag platform space. Community feedback highlights the zero-latency edge evaluation as the main advantage over third-party services like LaunchDarkly and Split.io, with some predicting that cloud platforms will increasingly offer native flag evaluation, reducing the market for expensive standalone feature flag SaaS products. Flagship is currently available in closed beta.

### Presentation: The Human Scalability Problem: Why Your Teams Don’t Scale Like Your Code - (InfoQ)

https://www.infoq.com/presentations/human-scalability/

# Summary

Charlotte de Jong Schouwenburg discusses how companies face critical "human scalability" challenges during rapid growth—a problem distinct from technical scalability. While software systems can scale efficiently, human cooperation often breaks down due to communication overload, unclear roles, and lost context. She illustrates this with the LeanIX case study, which showed how a 12-person startup scaling to 500 employees experienced slower decision-making, role ambiguity, and coordination failures despite sound technical infrastructure.

To address these challenges, Schouwenburg emphasizes the importance of proactive planning around communication architecture and "engineering trust" to maintain high-performing, autonomous teams. Her key insight is that leaders must intentionally design organizational structures and communication patterns early in growth, rather than attempting to fix coordination problems after they become entrenched at scale. This requires treating human scalability with the same rigor typically applied to technical systems.

### Article: From Batch to Micro-Batch Streaming: Lessons Learned the Hard Way in a Delta Index Pipeline - (InfoQ)

https://www.infoq.com/articles/micro-batch-streaming-lessons-learned/

# Summary

This article describes a production data pipeline's migration from scheduled batch processing to micro-batch Spark Structured Streaming to reduce latency in a delta index pipeline used for search and ads retrieval. Rather than adopting record-level streaming, the team implemented a simpler approach using time-driven execution, external watermark-based progress tracking on time-partitioned S3 data, and partition-level completeness checks. The key insight was that scheduling delays—not computational cost—were the primary bottleneck, and micro-batch streaming eliminated this overhead without the operational complexity of true event streaming.

The article shares hard-won lessons about designing reliable streaming systems on object storage, including: avoiding unreliable completion markers in favor of deterministic, rate-based progress tracking; treating job restarts as normal operational events; and prioritizing freshness over exhaustive historical replay when using overlapping window semantics. This pragmatic approach proved more effective than record-level streaming for batch-oriented systems where partition-level correctness and freshness matter more than per-record processing guarantees.

### Podcast: Roq: Leveraging Quarkus to Build Static Sites at the Speed of Go - (InfoQ)

https://www.infoq.com/podcasts/leveraging-quarkus-build-static-sites/

# Summary

Roq is a static site generator built on top of Quarkus that allows developers to create static websites from Markdown files without needing to know Java or Quarkus. Andy Damevin, a long-time Quarkus contributor, created Roq as an experiment to demonstrate that Java could be used efficiently for tasks traditionally associated with other languages like Go. The tool features a simple architecture—essentially a thin layer translating Markdown into static HTML—with fast bootstrapping and a small footprint thanks to Quarkus's efficiency.

The project reflects Java's resurgence as a developer-friendly language, particularly since Quarkus introduced native compilation capabilities that address Java's previous performance concerns. Damevin explains that while he initially explored Go, Quarkus's native speed combined with Java's superior code organization and runtime optimization capabilities made Java competitive again. Future development plans include adding more user-friendly features for non-technical users, and migration to Roq is straightforward—simply extract content into Markdown files and add them to the platform.

### DoorDash Used Copilot to Convert Its XCTest-Based iOS Test Suite to Swift Testing - (InfoQ)

https://www.infoq.com/news/2026/05/doordash-copilot-swift-testing/

# Summary

DoorDash successfully migrated its iOS test suite from XCTest to Swift Testing using AI coding assistant Copilot, achieving significant performance and efficiency gains. The migration was automated using custom "migration rules" that converted XCTest syntax to Swift Testing's modern format, while developers focused on correctness. The company implemented strict reliability safeguards, including a requirement that each migrated test pass ten consecutive runs to eliminate flakiness.

The migration delivered substantial results: ~60% faster test execution in CI, ~40% faster overall builds, reduced infrastructure costs, and improved developer productivity. DoorDash's success was enabled by a structured approach that combined automation with clear engineering principles, team ownership of migrations, and developer-friendly tooling including the Cursor AI assistant and custom integration with their build system. The faster parallel execution of Swift Testing also exposed hidden issues in tests that were previously masked by XCTest's sequential model.

### Cloudflare Builds High-Performance Infrastructure for Running LLMs - (InfoQ)

https://www.infoq.com/news/2026/05/cloudflare-llm-infrastructure/

# Summary

Cloudflare has developed new infrastructure optimized for running large language models (LLMs) across its global network. The company's key innovations include disaggregated prefill—splitting model processing into two stages on different machines where one handles input preparation (compute-bound) and the other generates output (memory-bound)—and a custom inference engine called Infire that manages multiple GPUs more efficiently, reduces memory usage, and improves response times. Additionally, Cloudflare introduced Unweight, a compression system that reduces model weights by 15-22% without accuracy loss.

These optimizations address the significant challenges of running massive models in production, such as Kimi K2.5 (over 1 trillion parameters), which requires splitting across multiple GPUs. By using pipeline and tensor parallelism while minimizing cross-GPU communication, Cloudflare's approach enables faster inference and more efficient resource utilization. The infrastructure improvements reflect broader industry recognition that traditional systems weren't designed to handle the scale and reliability demands of production AI workloads.

### DuckLake 1.0: Data Lake Format with SQL Catalog Metadata - (InfoQ)

https://www.infoq.com/news/2026/05/ducklake-sql-catalog/

# Summary

DuckDB Labs released DuckLake 1.0, a new data lake format that fundamentally differs from existing solutions like Apache Iceberg and Delta Lake by storing table metadata in a SQL database rather than distributed across object storage files. This approach addresses common problems such as complex coordination, slow metadata operations, and the proliferation of small files. DuckLake 1.0 includes features like data inlining for efficient small updates/deletes, sorted tables, bucket partitioning, and deletion vectors, with clients available for Apache Spark, Trino, Pandas, and other platforms.

The release marks the production-ready version of the specification first outlined in the "DuckLake manifesto" a year prior and guarantees backward compatibility. A key innovation is data inlining, which handles insert, update, and delete operations directly in the catalog database by default (with a 10-row threshold), eliminating the "small file problem." Future versions will introduce Git-like branching for datasets and role-based permissions, while MotherDuck provides a hosted service for managing the catalog database and storage infrastructure.

### JobRunr Introduces ClawRunr, an Open-Source Java AI Agent - (InfoQ)

https://www.infoq.com/news/2026/05/clawrunr/

# Summary

JobRunr has released ClawRunr, an open-source Java AI agent designed to run on users' own hardware and manage scheduled, recurring, and one-off background tasks. Built on Java 25 and Spring frameworks, ClawRunr combines conversational AI interactions with persistent task execution, supporting multiple LLM providers (OpenAI, Anthropic, Ollama) and communication channels (web chat, Telegram, Discord). The system addresses key challenges in AI agent design by making retries, persistence, scheduling, and monitoring core features, with tasks stored as Markdown files and managed through JobRunr's job scheduling infrastructure.

Users configure the agent through a guided onboarding flow, and optional Playwright integration adds browser automation capabilities. ClawRunr is released under the LGPL-3.0 license and evolved from a proof-of-concept demonstrating JobRunr's application to AI agents, with future plans including additional channels and improved memory management.

### Confluent Moves Schema IDs to Kafka Headers to Simplify Schema Governance - (InfoQ)

https://www.infoq.com/news/2026/05/confluent-kafka-header-schema-id/

# Summary

Confluent has introduced a new feature that moves schema IDs from Kafka message payloads to message headers, simplifying schema governance in event-driven architectures. This change decouples schema metadata from data itself, allowing organizations to adopt schema validation without modifying existing event formats and enabling more flexible schema evolution across distributed systems.

The header-based approach offers several advantages: it reduces coordination overhead between producers and consumers, supports incremental adoption of schema governance practices, and improves interoperability with downstream tools like Apache Flink and analytics platforms. While the transition may require updates to some Kafka connectors and tools, the feature enables zero-downtime adoption and maintains backward compatibility with formats like Avro, Protobuf, and JSON Schema.

Vitest 4.1, a Vite-native JavaScript testing framework, has been released with several key features designed to improve test organization and production parity. The update introduces test tags (inspired by pytest markers) for filtering and configuring groups of tests with shared settings like timeouts; an experimental native Node.js execution mode that bypasses Vite's module runner for faster startup and closer-to-production behavior; new lifecycle hooks (aroundEach and aroundAll); and an AI agent reporter that reduces token usage by suppressing output for passing tests. The framework remains API-compatible with Jest, making migration straightforward for existing users. Some minor issues were reported post-release, including regressions with coverage ignore hints and compatibility issues with Yarn Classic, though these have since been addressed.

# Summary

The article describes how artificial intelligence swarms are being weaponized to disrupt democracy through coordinated disinformation campaigns. Traditional troll farms—expensive operations requiring human workers—have evolved into "AI farms" that deploy hundreds of thousands of autonomous agents to generate synthetic content at industrial scale. Groups like Russia's CopyCop use uncensored language models to transform articles into political propaganda and distribute them across fake websites, leaving no trace. These "malicious AI swarms" create persistent, adaptive networks that simulate authentic communities and build synthetic consensus by exploiting human cognitive biases.

The swarms exploit psychological vulnerabilities, particularly the "bandwagon effect" and "illusory truth" effect—humans tend to believe information that appears repeated across multiple independent sources. By generating thousands of unique variations of the same message across different platforms and fake identities, AI swarms create the false impression of widespread public opinion, making propaganda appear credible and leading people to align with manipulated narratives. This represents an unprecedented threat to democratic discourse.

### Local AI - (Radar)

https://www.oreilly.com/radar/local-ai/

# Summary

This O'Reilly article discusses the growing viability of running large language models (LLMs) locally on personal or organizational hardware rather than relying on cloud-based APIs from major providers. Recent releases like Gemma 4 have made local models competitive with frontier models, with the quality now suitable for production use. The article outlines four primary reasons organizations are shifting to local AI: cost savings (developers can recoup hardware investments within months compared to monthly API spending of $500–$1,000+), privacy and data sovereignty concerns (particularly for regulated industries and non-US companies), performance control, and operational autonomy.

The economics favor local deployment for many use cases, with a capable GPU setup costing $500–$1,500 initially but then incurring minimal ongoing costs ($20–$40 monthly for electricity). While enterprise-scale implementations require additional infrastructure and support overhead, individual developers and small teams can easily run local models using tools like Ollama. The article emphasizes that privacy motivations go beyond security fears to encompass legal compliance requirements (GDPR, healthcare, financial services regulations) that may prohibit sending sensitive data to external cloud providers.
